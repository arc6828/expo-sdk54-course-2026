# แบบทดสอบท้ายบทเรียน บทที่ 1: ความรู้เบื้องต้นและการเตรียมสภาพแวดล้อม (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความรู้และความเข้าใจเกี่ยวกับความรู้พื้นฐาน การตั้งค่าระบบ และโครงสร้างเบื้องต้นของ React Native Expo SDK 54

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: ข้อใดอธิบายความแตกต่างที่สำคัญที่สุดระหว่าง React Native และเฟรมเวิร์กประเภท Hybrid App ยุคเก่า (เช่น Cordova หรือ PhoneGap) ได้ถูกต้องที่สุด?
ก. React Native เขียนด้วยภาษา Swift แต่ Hybrid App เขียนด้วยภาษา HTML/CSS  
ข. React Native จะแปลงคอมโพเนนต์ไปเป็น Native UI component จริง ๆ ของอุปกรณ์ ส่วน Hybrid App จะแสดงผลผ่าน Web View  
ค. React Native รันได้เฉพาะบนระบบปฏิบัติการ iOS เท่านั้น ในขณะที่ Hybrid App รันได้ทุกระบบ  
ง. React Native ไม่สามารถเชื่อมต่อกับฮาร์ดแวร์ของอุปกรณ์ได้ แต่ Hybrid App ทำได้  

### ข้อ 2: เมื่อเราใช้เทมเพลตเริ่มต้นของ Expo SDK 54 และรันคำสั่ง `npm run reset-project` จะเกิดผลลัพธ์ใดขึ้นกับโครงการ?
ก. ลบการเชื่อมต่อเครือข่ายของแอปและล้างข้อมูลแคชทั้งหมด  
ข. ลบโปรเจคทิ้งอย่างถาวรและสร้างโฟลเดอร์เปล่าขึ้นมาใหม่  
ค. ย้ายไฟล์ตัวอย่างและโครงสร้างเดิมไปไว้ในโฟลเดอร์ `app-example/` และสร้างโฟลเดอร์ `app/` ใหม่พร้อมไฟล์เริ่มต้นที่ว่างเปล่า  
ง. ทำการคอมไพล์โปรเจคให้กลายเป็นไฟล์สำหรับติดตั้ง (.apk และ .ipa) ทันที  

### ข้อ 3: ข้อจำกัดที่สำคัญที่สุดของการใช้เครื่องมือ "Expo Go" ในการทดสอบแอปพลิเคชันคืออะไร?
ก. ไม่สามารถทดสอบบนสมาร์ทโฟนระบบปฏิบัติการ iOS ได้  
ข. ไม่สามารถทดสอบโค้ดขณะแก้ไขแบบเรียลไทม์ได้ ต้องคอมไพล์ใหม่ทุกครั้ง  
ค. ไม่สามารถทดสอบแอปพลิเคชันที่ติดตั้งไลบรารีหรือเขียนโค้ดภาษา Native เสริมเองภายนอกมาตรฐานของ Expo ได้  
ง. ต้องเสียค่าบริการรายเดือนในการสแกน QR Code เพื่อเปิดใช้งาน  

### ข้อ 4: ไฟล์ใดในโปรเจค React Native Expo ที่ทำหน้าที่เก็บข้อมูลคุณสมบัติแอปพลิเคชัน (App Metadata) เช่น ชื่อแอป โลโก้ (Icon) หน้าต้อนรับ (Splash Screen) และเวอร์ชันของแอป?
ก. `package.json`  
ข. `app.json`  
ค. `tsconfig.json`  
ง. `_layout.tsx`  

### ข้อ 5: คุณสมบัติ "Fast Refresh" ในสภาพแวดล้อมการพัฒนาของ Expo ทำหน้าที่อะไร?
ก. ช่วยเพิ่มความเร็วในการดาวน์โหลดไลบรารีจากเซิร์ฟเวอร์ npm  
ข. อัปเดตอินเทอร์เฟซของแอปบนอุปกรณ์ทดสอบทันทีที่กดบันทึกโค้ด (Save) โดยไม่ต้องเริ่มแอปพลิเคชันใหม่ทั้งหมด  
ค. ช่วยเคลียร์แคชและรีสตาร์ตระบบปฏิบัติการของมือถือโดยอัตโนมัติ  
ง. การอัปเดตแอปพลิเคชันขึ้นสู่ Google Play Store และ App Store อย่างรวดเร็ว  

### ข้อ 6: คำสั่งมาตรฐานใดที่แนะนำให้ใช้รันใน Terminal เพื่อเปิดเซิร์ฟเวอร์นักพัฒนา Metro Bundler ของ Expo?
ก. `npm run ios`  
ข. `npx expo run:android`  
ค. `npx expo start`  
ง. `npx react-native start`  

### ข้อ 7: หากหน้าจอแอปพลิเคชันบนตัวจำลอง (Simulator) ไม่ตอบสนอง หรือ Fast Refresh ไม่ทำงาน คุณสามารถกดปุ่มใดบนคีย์บอร์ดใน Terminal ของเซิร์ฟเวอร์นักพัฒนาเพื่อสั่งโหลดหน้าจอแอปใหม่ด้วยตนเอง (Manual Reload)?
ก. กดปุ่ม `r`  
ข. กดปุ่ม `i`  
ค. กดปุ่ม `a`  
ง. กดปุ่ม `d`  

### ข้อ 8: การทดสอบแอปพลิเคชันด้วยสมาร์ทโฟนจริงผ่านแอป Expo Go มีเงื่อนไขการเชื่อมต่อเครือข่ายอินเทอร์เน็ตที่จำเป็นอย่างไร?
ก. สมาร์ทโฟนต้องเชื่อมต่อ Cellular 5G ส่วนเครื่องคอมพิวเตอร์เชื่อมต่อ Wi-Fi บ้าน  
ข. ทั้งสมาร์ทโฟนและเครื่องคอมพิวเตอร์ต้องเชื่อมต่อผ่านสาย USB ตลอดเวลาเท่านั้น  
ค. สมาร์ทโฟนและเครื่องคอมพิวเตอร์ต้องเชื่อมต่อในเครือข่ายวงแลนเดียวกัน (เช่น Wi-Fi วงเดียวกัน)  
ง. เครื่องคอมพิวเตอร์และมือถือต้องเปิดสัญญาณ Bluetooth เชื่อมต่อกันก่อน  

### ข้อ 9: โครงสร้างโฟลเดอร์ของ Expo SDK 54 ที่ใช้ระบบนำทางตามโครงสร้างไฟล์ (File-based Routing) จะมองหาไฟล์เริ่มต้นสำหรับหน้าหลักของแต่ละโฟลเดอร์จากชื่อไฟล์ใด?

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:** 
  * **ก. ผิด:** ทั้ง React Native และ Hybrid App ในแง่เขียนโค้ดจะใช้เทคโนโลยีเว็บ (โดยเฉพาะ JavaScript/TypeScript) ไม่ใช่เขียนด้วย Swift เป็นหลัก
  * **ข. ถูก:** นี่คือคุณลักษณะเด่นของ React Native ที่ทำให้ประสิทธิภาพ (Performance) ใกล้เคียงกับ Native App แท้ ๆ เพราะคอมโพเนนต์อย่าง `<View>` หรือ `<Text>` จะถูกจับคู่และสร้างเป็นคอมโพเนนต์ระดับ OS จริง ๆ ของมือถือตัวนั้น ส่วน Hybrid App ยุคเก่าจะรันเว็บแอปจำลองภายในเบราว์เซอร์ครอบ (Web View)
  * **ค. ผิด:** React Native เป็น Cross-platform สามารถรันได้ทั้ง iOS, Android และ Web
  * **ง. ผิด:** React Native มีความสามารถสูงในการเข้าถึงฮาร์ดแวร์ผ่าน API ส่วนติดต่อของภาษา Native

### เฉลย ข้อ 2: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** คำสั่งนี้ไม่เกี่ยวข้องกับการล้างแคชหรือลบข้อมูลเครื่องจำลอง
  * **ข. ผิด:** โปรเจคและไดเรกทอรีหลักยังคงอยู่ไม่ได้ถูกลบทั้งหมด
  * **ค. ถูก:** การรัน `npm run reset-project` จะรันสคริปต์เพื่อแยกโค้ดตัวอย่างของระบบ (เช่น explore screen หรือ components ต่าง ๆ) ไปเก็บไว้ที่ `app-example/` และสร้างโฟลเดอร์ `app/` สำหรับเขียนโค้ดที่สะอาดใหม่ขึ้นมา เพื่อเริ่มต้นโครงสร้างของคุณเอง
  * **ง. ผิด:** การคอมไพล์โปรเจคเป็นไฟล์ติดตั้งทำโดยใช้บริการของ EAS Build ไม่ใช่สคริปต์ตัวนี้

### เฉลย ข้อ 3: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** Expo Go รองรับทั้ง iOS และ Android อย่างสมบูรณ์
  * **ข. ผิด:** Expo Go รองรับ Fast Refresh สามารถอัปเดตหน้าจอแบบเรียลไทม์ได้ทันที
  * **ค. ถูก:** เนื่องจาก Expo Go เป็นแอปสำเร็จรูปที่มีไลบรารีมาตรฐานติดตั้งมาให้แล้ว หากคุณต้องการเขียนโค้ด Native (เช่น Swift, Kotlin) หรือใช้แพ็กเกจที่จำเพาะเจาะจงที่แก้ไขส่วนประกอบ OS ดั้งเดิม คุณจะต้องเปลี่ยนไปรันผ่าน Expo Development Builds แทน
  * **ง. ผิด:** เครื่องมือ Expo Go และการแชร์หน้าจอผ่าน Metro Bundler ใช้งานได้ฟรี ไม่มีค่าใช้จ่าย

### เฉลย ข้อ 4: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `package.json` มีหน้าที่เก็บรายชื่อ Dependencies และสคริปต์คำสั่งรันระบบ
  * **ข. ถูก:** `app.json` เป็นไฟล์คอนฟิกูเรชันหลักของ Expo (Expo config) ที่เอาไว้ตั้งค่าข้อมูลทั่วไปของแอปพลิเคชัน โครงสร้างสีเริ่มต้น รูปภาพไอคอน ภาพหน้าต้อนรับ และการกำหนดค่าเฉพาะของแพลตฟอร์มปลายทาง
  * **ค. ผิด:** `tsconfig.json` เป็นไฟล์สำหรับตั้งค่าคอมไพเลอร์ของภาษา TypeScript
  * **ง. ผิด:** `_layout.tsx` เป็นไฟล์กำหนดโครงสร้างการนำทางและการแสดงผลของหน้าจอ (Layout) ของ Expo Router

### เฉลย ข้อ 5: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** การจัดการแพ็กเกจไลบรารีเป็นหน้าที่ของ npm / yarn
  * **ข. ถูก:** Fast Refresh เป็นฟีเจอร์เด่นของการพัฒนาแอปพลิเคชันด้วย React Native ที่จะคอยเฝ้าดูการเปลี่ยนแปลงของไฟล์โค้ด (Watch Mode) เมื่อมีการบันทึกไฟล์ โค้ดที่เปลี่ยนจะถูกส่งไปทำงานบนเครื่องที่เปิดจำลองแอปอยู่ทันที ช่วยลดระยะเวลาที่นักพัฒนาต้องรอรันระบบใหม่ลงอย่างมหาศาล
  * **ค. ผิด:** ฟังก์ชันนี้ไม่มีความสามารถในการยุ่งเกี่ยวกับระบบปฏิบัติการหลักของสมาร์ทโฟน
  * **ง. ผิด:** การอัปโหลดแอปพลิเคชันขึ้นสโตร์ใช้ EAS Submit หรือส่งไฟล์ Build โดยตรง

### เฉลย ข้อ 6: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `npm run ios` ใช้สำหรับสั่งรันแอปบน iOS Simulator ทันที (ซึ่งจะสตาร์ท Metro ให้อยู่แล้ว แต่เป็นคำสั่งจำเพาะเจาะจงแพลตฟอร์ม)
  * **ข. ผิด:** `npx expo run:android` ใช้สำหรับคอมไพล์โปรเจคแบบ Development Builds รันบนเครื่องจำลอง Android ซึ่งช้ากว่าและยุ่งยากกว่าการทดสอบด้วย Expo Go ในขั้นเริ่มต้น
  * **ค. ถูก:** `npx expo start` (หรือ alias `npm start` ใน package.json) คือคำสั่งพื้นฐานและแนะนำที่สุดของ Expo ในการเปิดระบบเซิร์ฟเวอร์ Metro Bundler และสร้าง QR Code เพื่อสแกนรันแอป
  * **ง. ผิด:** `npx react-native start` เป็นคำสั่งรันเซิร์ฟเวอร์สำหรับโปรเจคที่เป็น React Native CLI บริสุทธิ์

### เฉลย ข้อ 7: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** ในหน้าต่าง Terminal ของเซิร์ฟเวอร์ Metro การกดปุ่ม `r` ย่อมาจาก **Reload** จะส่งสัญญาณบอกแอปทดสอบบนเครื่องจำลองหรือโทรศัพท์จริงให้รีโหลดหน้าจอใหม่ทั้งหมดแบบแมนนวล
  * **ข. ผิด:** การกดปุ่ม `i` ย่อมาจาก **iOS** ใช้สำหรับสั่งเปิดแอปบน iOS Simulator
  * **ค. ผิด:** การกดปุ่ม `a` ย่อมาจาก **Android** ใช้สำหรับสั่งเปิดแอปบน Android Emulator
  * **ง. ผิด:** การกดปุ่ม `d` ย่อมาจาก **Dev Menu** ใช้สำหรับสั่งเปิดเมนูนึกพัฒนา (Developer Menu) บนแอปจำลอง

### เฉลย ข้อ 8: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** การเปิดข้ามเครือข่ายผ่าน Cellular อาจทำให้แอปหาเครื่องคอมพิวเตอร์ในวงแลนของเครือข่ายนักพัฒนาไม่เจอ ยกเว้นมีการตั้งค่าอุโมงค์การเชื่อมต่อแบบ Tunnel (ซึ่งทำได้ยากและช้ากว่าปกติมาก)
  * **ข. ผิด:** ไม่จำเป็นต้องต่อสาย USB เสมอไปเมื่อใช้งาน Expo Go เพราะระบบสามารถส่งผ่านข้อมูลทาง Wi-Fi วงแลนได้แบบไร้สาย
  * **ค. ถูก:** หัวใจสำคัญของการสื่อสารระหว่าง Metro Bundler บน PC และมือถือที่ใช้รันแอปคือต้องอยู่ในเครือข่ายภายในบ้าน/ที่ทำงานเดียวกัน (Subnet วงเดียวกัน) เพื่อให้เครื่องมือถือสามารถค้นหา IP Address และ Port 8081 ของ PC พบ
  * **ง. ผิด:** Expo CLI และ Expo Go ไม่สื่อสารโค้ดโปรแกรมหลักผ่านบลูทูธ

### เฉลย ข้อ 9: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `App.tsx` เป็นไฟล์เริ่มต้นของโปรเจค React Native แบบดั้งเดิม แต่ไม่ใช่โครงสร้างของ Expo Router ที่เปิดตัวขึ้นมาในรุ่นหลัง
  * **ข. ถูก:** โครงสร้าง File-based Routing (Expo Router) จะทำงานคล้ายกับการจัดการหน้าเว็บ (Web Router) โดยไฟล์ที่ทำหน้าที่เป็นจุดเริ่มต้นหรือหน้าแรกของโฟลเดอร์ใด ๆ จะต้องตั้งชื่อว่า `index.tsx` (หรือ `index.js`, `index.ts`) เสมอ
  * **ค. ผิด:** `main.tsx` ไม่เกี่ยวข้องกับการหาเส้นทางเริ่มต้นของโฟลเดอร์
  * **ง. ผิด:** `root.tsx` ไม่ได้ใช้ระบุตำแหน่งของ Layout เริ่มต้นในระดับโฟลเดอร์
