# บทที่ 1: ความรู้เบื้องต้นเกี่ยวกับ React Native Expo และการเตรียมสภาพแวดล้อม (Content)

ยินดีต้อนรับสู่หลักสูตรพัฒนาแอปพลิเคชันมือถือด้วย **React Native และ Expo SDK 54** ในบทนี้เราจะเรียนรู้แนวคิดพื้นฐาน ติดตั้งเครื่องมือ และเริ่มต้นรันแอปพลิเคชันแรกของคุณ

---

## 1.1 ทำความรู้จัก React Native และ Expo

### React Native คืออะไร?
**React Native** คือโอเพนซอร์สเฟรมเวิร์กที่พัฒนาโดย Meta (Facebook) ช่วยให้ผู้พัฒนาสามารถสร้างแอปพลิเคชันสำหรับ iOS และ Android ได้พร้อมกันด้วยการเขียนโค้ดภาษา JavaScript และเฟรมเวิร์ก React แทนที่จะต้องแยกกันเขียนด้วย Swift/Objective-C (iOS) และ Kotlin/Java (Android)

* **Native Performance:** แตกต่างจาก Hybrid App ยุคเก่าที่รันผ่าน Web View เพราะ React Native จะแปลงคอมโพเนนต์ JavaScript ไปเป็นคอมโพเนนต์ Native จริง ๆ ของอุปกรณ์ ทำให้ได้ประสิทธิภาพที่ไหลลื่นและรูปลักษณ์ที่เป็นธรรมชาติ
* **Cross-Platform:** เขียนโค้ดชุดเดียว (Single Codebase) สามารถแชร์การทำงานได้มากกว่า 80-90% ระหว่างแพลตฟอร์ม

### Expo คืออะไร?
**Expo** คือชุดเครื่องมือ (Suite of Tools) และบริการเสริมที่ครอบทับอยู่บน React Native อีกชั้นหนึ่ง เพื่อช่วยลดความซับซ้อนในการตั้งค่าและเขียนโค้ด Native โดยในปัจจุบัน Expo เป็นเครื่องมือมาตรฐานที่ทางทีมงาน React Native แนะนำอย่างเป็นทางการในการเริ่มต้นโครงการใหม่

> [!NOTE]
> **Expo SDK 54** นำเสนอฟีเจอร์ความปลอดภัย ประสิทธิภาพที่สูงขึ้น การใช้ React 19 และมาพร้อมกับโครงสร้างการจัดเส้นทางโฟลเดอร์แบบไฟล์ (File-based Routing) เวอร์ชันล่าสุดผ่าน **Expo Router v6**

## 1.2 การเปรียบเทียบการเลือกพัฒนาแอปพลิเคชัน

ก่อนเริ่มต้น เรามาทำความเข้าใจแนวทางในการรันแอปพลิเคชัน 3 แบบหลัก ๆ:

| หัวข้อเปรียบเทียบ | Expo Go | Expo Development Builds | React Native CLI |
| :--- | :--- | :--- | :--- |
| **ความง่ายในการเริ่ม** | ง่ายที่สุด (ติดตั้งแอปจาก Store) | ปานกลาง (ต้องติดตั้ง SDK ท้องถิ่น) | ยากที่สุด (ต้องตั้งค่า SDK และตัวคอมไพล์) |
| **โค้ดภาษา Native** | ใช้ได้เฉพาะคอมโพเนนต์มาตรฐานที่แถมมา | เขียนโค้ด Native (Swift/Kotlin) เสริมได้ | เขียนโค้ด Native เสริมได้อย่างอิสระ |
| **การสร้างและติดตั้ง** | รันผ่านแอปแวดล้อมแชร์ร่วมกัน | สร้างไฟล์แอปเฉพาะกิจ (.apk / .ipa) | ต้องคอมไพล์ทุกครั้งที่ทดสอบโค้ด |
| **เหมาะสำหรับ** | ผู้เริ่มต้น, โปรเจคขนาดเล็ก, ทำ UI | โปรเจคจริงขนาดกลาง-ใหญ่, ใช้ปลั๊กอินภายนอก | โปรเจคขนาดใหญ่ที่ต้องการควบคุมระดับ OS |

*สำหรับหลักสูตรนี้ เราจะเริ่มต้นด้วย **Expo Go** เพื่อให้ผู้เรียนได้ทดลองเขียนและรันแอปบนอุปกรณ์จริงอย่างรวดเร็วที่สุด จากนั้นจะขยายไปสู่หัวข้อสร้างแอปพลิเคชันแบบกำหนดเองในบทหลัง*

## 1.3 การเตรียมความพร้อมและติดตั้งสภาพแวดล้อม (Environment Setup)

เพื่อให้เครื่องคอมพิวเตอร์ของคุณพร้อมสำหรับการพัฒนาแอปพลิเคชัน ให้ติดตั้งโปรแกรมต่าง ๆ ตามขั้นตอนดังนี้:

### ขั้นที่ 1: ติดตั้ง Node.js
Node.js เป็นโปรแกรมจำเป็นสำหรับรันสคริปต์ JavaScript และตัวติดตั้งแพ็กเกจ (npm)
1. ไปที่เว็บไซต์ [Node.js Official Website](https://nodejs.org/)
2. ดาวน์โหลดและติดตั้งเวอร์ชัน **LTS** (แนะนำเวอร์ชัน 20 หรือ 22 ขึ้นไป)
3. ตรวจสอบความสำเร็จของการติดตั้งใน Terminal/PowerShell:
   ```bash
   node -v
   npm -v
   ```

### ขั้นที่ 2: ติดตั้ง Visual Studio Code (VS Code)
VS Code เป็นโปรแกรมเขียนโค้ดและแก้ไขข้อความยอดนิยม
1. ดาวน์โหลดที่ [VS Code Website](https://code.visualstudio.com/)
2. ติดตั้งส่วนขยาย (Extensions) ที่แนะนำสำหรับการเขียน React Native:
   * **ES7+ React/Redux/React-Native snippets** (ช่วยคีย์ลัดสร้างโค้ด)
   * **Prettier - Code formatter** (ช่วยจัดรูปแบบโค้ดอัตโนมัติ)
   * **Expo Tools** (ช่วยตรวจสอบไฟล์ `app.json`)

### ขั้นที่ 3: ติดตั้ง Git
ใช้สำหรับการจัดการเวอร์ชันโค้ด (Version Control) ดาวน์โหลดได้จาก [Git SCM](https://git-scm.com/)

### ขั้นที่ 4: ติดตั้งแอปพลิเคชัน Expo Go บนสมาร์ทโฟน
1. **iOS:** ดาวน์โหลดแอปพลิเคชัน **Expo Go** จาก App Store
2. **Android:** ดาวน์โหลดแอปพลิเคชัน **Expo Go** จาก Google Play Store
3. ตรวจสอบให้มั่นใจว่าสมาร์ทโฟนและเครื่องคอมพิวเตอร์ที่ใช้เขียนโค้ด เชื่อมต่อวงเครือข่ายอินเทอร์เน็ต (Wi-Fi) วงเดียวกัน

## 1.4 การสร้างโปรเจคใหม่ด้วย create-expo-app

เปิด Terminal (ใน macOS/Linux) หรือ PowerShell (ใน Windows) และย้ายไดเรกทอรีไปยังตำแหน่งที่ต้องการสร้างโปรเจค จากนั้นรันคำสั่ง:

```bash
npx create-expo-app@latest my-first-app
```

* **`npx create-expo-app@latest`**: คำสั่งดาวน์โหลดสคริปต์สร้างแอปเวอร์ชันล่าสุด
* **`my-first-app`**: ชื่อโปรเจคและโฟลเดอร์ที่เราต้องการสร้าง

เมื่อการดาวน์โหลดเสร็จสมบูรณ์ ให้ใช้คำสั่งเปลี่ยนโฟลเดอร์เข้าไปในโปรเจค:
```bash
cd my-first-app
```

### การรีเซ็ตโปรเจคเริ่มต้น (Reset Project Layout)
สำหรับเทมเพลตยุคใหม่ของ Expo ที่สร้างผ่านคำสั่งเริ่มต้น หากผู้เรียนไม่ต้องการหน้าจอตัวอย่างหรือ UI ที่ติดมากับระบบ (เช่น แถบ Tabs ด้านล่าง และหน้าจอ Explore) ทาง Expo มีสคริปต์ล้างโครงสร้างเพื่อให้คุณเริ่มต้นทำแอปจากศูนย์ได้อย่างรวดเร็ว:

```bash
npm run reset-project
```

> [!IMPORTANT]
> คำสั่งนี้จะทำการย้ายโค้ดเทมเพลตเริ่มต้นทั้งหมดไปเก็บไว้ที่โฟลเดอร์ `app-example/` เพื่อใช้ในการศึกษาภายหลัง และระบบจะทำการสร้างโฟลเดอร์ `app/` ขึ้นมาใหม่พร้อมไฟล์ `index.tsx` แบบเปล่า ๆ เพื่อให้เราพร้อมเขียนหน้าจอแอปพลิเคชันจากศูนย์ทันที

## 1.5 โครงสร้างของโปรเจค React Native Expo SDK 54

เมื่อเราเปิดโฟลเดอร์โปรเจคด้วย VS Code จะพบโครงสร้างไฟล์หลัก ๆ ดังนี้:

```text
my-first-app/
├── app/                  # โฟลเดอร์สำหรับเก็บโค้ดหน้าจอและการนำทาง (Expo Router)
│   ├── _layout.tsx       # เลย์เอาต์หลักครอบทุกหน้าจอ
│   └── index.tsx         # หน้าจอแรกสุดของแอปพลิเคชัน (Home Screen)
├── assets/               # ที่เก็บรูปภาพ ฟอนต์ และสัญลักษณ์ของแอป
├── app.json              # ไฟล์การตั้งค่าคุณสมบัติแอปพลิเคชัน (App Metadata)
├── package.json          # รายชื่อ Library และชุดคำสั่งสคริปต์ในการรันแอป
├── tsconfig.json         # การตั้งค่าพฤติกรรมสำหรับภาษา TypeScript
└── node_modules/         # โฟลเดอร์ที่เก็บแพ็กเกจ Library ทั้งหมด (ห้ามแก้ไขโดยตรง)
```

### รายละเอียดไฟล์ตั้งค่าที่สำคัญ:
1. **`app.json`**: ใช้กำหนดชื่อแอปพลิเคชัน โลโก้ (icon) ภาพตอนเข้าแอป (splash image) สิทธิ์การเข้าถึงอุปกรณ์ (permissions) และเวอร์ชันของแอป
2. **`app/index.tsx`**: จุดเริ่มต้นการเขียนหน้าตาแอปพลิเคชัน เช่น:

```tsx
import { StyleSheet, Text, View } from 'react-native';

export default function HomeScreen() {
  return (
    <View style={styles.container}>
      <Text style={styles.title}>Hello, Expo SDK 54! 👋</Text>
      <Text style={styles.subtitle}>นี่คือแอปพลิเคชันแรกของฉัน</Text>
    </View>
  );
}

const styles = StyleSheet.create({
  container: {
    flex: 1,
    backgroundColor: '#0f172a', // สีพื้นหลัง Slate เข้มสไตล์โมเดิร์น
    alignItems: 'center',
    justifyContent: 'center',
  },
  title: {
    fontSize: 24,
    fontWeight: 'bold',
    color: '#f8fafc',
    marginBottom: 8,
  },
  subtitle: {
    fontSize: 16,
    color: '#94a3b8',
  },
});
```

## 1.6 การรันโปรเจคและการทดสอบแอปพลิเคชัน

ในโฟลเดอร์โปรเจค ให้รันคำสั่งสตาร์ทเซิร์ฟเวอร์นักพัฒนา:

```bash
npm start
```
หรือรันผ่านคำสั่งเฉพาะ:
```bash
npx expo start
```

ตัวจัดการการรัน (Expo CLI) จะเริ่มต้นเซิร์ฟเวอร์ **Metro Bundler** และแสดงหน้าอินเทอร์เฟซควบคุมขึ้นมา รวมถึงสร้าง **QR Code** บนหน้าจอคอนโซล:

```text
› Metro waiting on exp://192.168.1.33:8081
› Scan the QR code above with Expo Go (Android) or the Camera app (iOS)

› Press a │ open on Android
› Press i │ open on iOS simulator
› Press w │ open on web
› Press r │ reload app
```

### ขั้นตอนการดูผลลัพธ์บนอุปกรณ์ต่าง ๆ:

#### A. สำหรับสมาร์ทโฟนส่วนตัว (แนะนำ)
1. **ระบบ Android:** เปิดแอป **Expo Go** -> แตะเมนู "Scan QR Code" -> สแกนโค้ดบนคอมพิวเตอร์ของคุณ
2. **ระบบ iOS:** เปิดแอป **Camera** (กล้องมาตรฐาน) -> ส่อง QR Code -> แตะลิงก์เปิดด้วยแอป Expo Go
3. เมโทรบันด์เลอร์จะทำงานส่งโค้ด JavaScript มาแปลงร่างเป็นคอมโพเนนต์ Native แสดงผลบนหน้าจอมือถือของคุณทันที!

#### B. สำหรับตัวจำลองคอมพิวเตอร์ (Simulators/Emulators)
* **iOS Simulator:** ต้องติดตั้ง Xcode (เฉพาะ macOS) จากนั้นกดปุ่ม `i` บนคีย์บอร์ดใน Terminal
* **Android Emulator:** ต้องติดตั้ง Android Studio และสร้าง Virtual Device (AVD) จากนั้นกดปุ่ม `a` บนคีย์บอร์ดใน Terminal

> [!TIP]
> **Fast Refresh:** เมื่อคุณทำการแก้ไขข้อความใด ๆ ในไฟล์ `app/index.tsx` แล้วกดบันทึก (Save) หน้าจอแอปพลิเคชันบนมือถือหรือตัวจำลองของคุณจะอัปเดตผลลัพธ์ใหม่ทันทีในเวลาไม่ถึงวินาทีโดยไม่ต้องเริ่มรันแอปใหม่ทั้งหมด

## 1.7 สรุปท้ายบทเรียน

**ทำความรู้จัก React Native และ Expo:** React Native เป็นเครื่องมือพัฒนาแอปพลิเคชันแบบครอสแพลตฟอร์มที่ใช้ JavaScript/React ในการควบคุมคอมโพเนนต์ Native จริงของอุปกรณ์เพื่อให้ได้ประสิทธิภาพสูง โดยทำงานร่วมกับ Expo SDK 54 ซึ่งเป็นชุดเครื่องมืออำนวยความสะดวกในการจัดสรรสถาปัตยกรรมและไลบรารีมาตรฐานระดับระบบปฏิบัติการอย่างครบถ้วน

**การเปรียบเทียบการเลือกพัฒนาแอปพลิเคชัน:** แนวทางการพัฒนาแอปพลิเคชันแบ่งออกเป็น 3 รูปแบบหลัก ได้แก่ Expo Go ซึ่งติดตั้งและเริ่มรันได้ง่ายที่สุดเหมาะแก่การศึกษาและทดลองใช้เครื่องมือมาตรฐาน, Expo Development Builds ที่เหมาะสมสำหรับแอปพลิเคชันขนาดกลางถึงใหญ่เนื่องจากรองรับการปรับแต่งเนทีฟโมดูลเพิ่มเติมได้เอง และ React Native CLI ที่ให้ความอิสระและระดับสิทธิ์การควบคุมระดับลึกที่สุดแก่ผู้พัฒนารวมถึงการเขียนโค้ด Native โดยตรง

**การเตรียมความพร้อมและติดตั้งสภาพแวดล้อม:** การเริ่มต้นพัฒนาแอปพลิเคชันต้องการสภาพแวดล้อมการเขียนโค้ดที่ประกอบไปด้วย Node.js เวอร์ชัน LTS สำหรับการรัน JavaScript นอกเบราว์เซอร์, VS Code พร้อมส่วนขยายอำนวยความสะดวก เช่น Expo Tools และ Prettier, ระบบควบคุมเวอร์ชันด้วย Git และแอปพลิเคชัน Expo Go บนระบบปฏิบัติการ iOS หรือ Android เพื่อทำหน้าที่แปลงผลและรันแอปบนอุปกรณ์จริงในวงเน็ตเวิร์กเดียวกัน

**การสร้างโปรเจคใหม่ด้วย create-expo-app:** การสร้างโครงสร้างโฟลเดอร์สำหรับแอปพลิเคชันใหม่ใช้คำสั่ง `npx create-expo-app@latest` ตามด้วยชื่อโปรเจกต์ ซึ่งจะได้โครงสร้างการทำงานแบบสมบูรณ์พร้อมใช้ และผู้พัฒนาสามารถรันคำสั่ง `npm run reset-project` เพื่อล้างหน้าต่างจำลองของเทมเพลตเริ่มต้นและปรับโครงสร้างโปรเจกต์ให้เหลือเพียงหน้ากระดาษเปล่าสำหรับการเขียนแอปพลิเคชันตั้งแต่เริ่มต้น

**โครงสร้างของโปรเจค React Native Expo SDK 54:** โฟลเดอร์ของโปรเจกต์ที่สร้างขึ้นจะแบ่งสัดส่วนการทำงานออกเป็น โฟลเดอร์ `app/` สำหรับเก็บหน้าจอและระบบการเปลี่ยนหน้าผ่านไฟล์ (Expo Router), โฟลเดอร์ `assets/` เก็บภาพไอคอนและส่วนประกอบด้านมีเดีย, ไฟล์ `app.json` ทำหน้าที่ควบคุมการตั้งค่าคุณสมบัติ (Metadata) ของแอป และไฟล์ `app/index.tsx` ซึ่งเป็นคอมโพเนนต์แรกที่แอปเรียกมาแสดงผลและทำหน้าที่เป็นจุดเริ่มต้นของโค้ดหลัก

**การรันโปรเจคและการทดสอบแอปพลิเคชัน:** การรันแอปทำได้โดยใช้คำสั่ง `npm start` หรือ `npx expo start` เพื่อเริ่มการทำงานของ Metro Bundler ซึ่งจะทำการรวบรวมไฟล์เขียนโค้ดและส่งผลผ่าน QR Code ให้สแกนดูพฤติกรรมจริงผ่านแอป Expo Go บนอุปกรณ์สมาร์ทโฟน หรือทดสอบผ่านโปรแกรมจำลอง Emulator/Simulator โดยมีการสนับสนุนฟีเจอร์ Fast Refresh ที่ช่วยสะท้อนผลการแก้ไขโค้ดสู่หน้าจอทดสอบโดยทันที
