# บทที่ 8: การเก็บข้อมูลในเครื่องและการเชื่อมต่อเครือข่าย API (Overview)

ในบทเรียนนี้ ผู้เรียนจะได้ศึกษาแนวคิดการเชื่อมโยงระบบฐานข้อมูลภายนอกและระบบจัดเก็บข้อมูลถาวรภายในเครื่องสมาร์ทโฟน ซึ่งเป็นส่วนประกอบสำคัญของสถาปัตยกรรมโมบายแอปพลิเคชันยุคใหม่ โดยเน้นการจัดเก็บข้อมูลผ่าน AsyncStorage แบบ Key-Value, การแปลงข้อมูล JSON Serialization และเรียกสั่งการรับส่งข้อมูลผ่านเครือข่ายอินเทอร์เน็ต (GET & POST Request) ด้วยคำสั่ง Fetch API ที่เป็นมาตรฐานประสิทธิภาพสูงของระบบปฏิบัติการ พร้อมวิธีหนุนการแจ้งข้อมูลให้ผู้ใช้ทราบระหว่างการดึงข้อมูลและกรณีฉุกเฉินอินเทอร์เน็ตขาดหาย

1. 📋 **[แผนการสอนประจำบท (01-lesson-plan.md)](./01-lesson-plan.md)**  
   หัวข้อการสอน วัตถุประสงค์เชิงพฤติกรรม สื่อประกอบการสอน และหัวข้อการวัดประเมินทักษะจัดเก็บข้อมูลและติดต่อเครือข่าย
2. 📖 **[เนื้อหาหลักการเรียนรู้ (02-content.md)](./02-content.md)**  
   เนื้อหาหลักทางทฤษฎีอธิบายการเรียกใช้ AsyncStorage (CRUD), การทำ JSON stringify/parse, การยิง GET/POST ด้วย Fetch, การออกแบบ loading indicators, การจัดการข้อผิดพลาดด้วย try-catch ร่วมกับ `response.ok` และระบบจดจำสิทธิ์ Auth Token Session
3. 📝 **[แบบทดสอบท้ายบทเรียน (03-quiz.md)](./03-quiz.md)**  
   แบบทดสอบปรนัยทบทวนความเข้าใจ 10 ข้อ ภาษาไทยครอบคลุมพฤติกรรมและเมธอดของ AsyncStorage, การจัดการแปลงออบเจกต์ซับซ้อน, ความปลอดภัยการตรวจสอบผลลัพธ์ Fetch, และวงรอบชีวิตการเรียกใช้เช็คสถานะตอนเริ่มเปิดแอป
4. 💻 **[คู่มือปฏิบัติการขั้นพื้นฐาน (04-lab.md)](./04-lab.md)**  
   โจทย์ปฏิบัติการสร้าง **สมุดโน้ตบันทึกและซิงก์ความคิดเห็นออนไลน์ (Synchronized Note Keeper)** เพื่อฝึกฝนเขียนแอปติดต่อ API จากภายนอก (GET/POST) พร้อมกลไกสร้างประวัติกู้คืนสำรองกรณีออฟไลน์สัญญาณขาดหายผ่านหน่วยความจำ AsyncStorage ในตัวเครื่องปลายทาง พร้อมโค้ดเฉลยฉบับสมบูรณ์ (Complete Solution)

## 🛠️ สรุปโปรเจคปฏิบัติการท้ายบทเรียน (Lab Project Summary)
ในปฏิบัติการของบทนี้ ผู้เรียนจะได้ฝึกฝนสร้างแอปพลิเคชันที่มีระบบต้านทานเครือข่ายขัดข้อง:
* **Fetch Remote GET/POST API:** ดึงข้อมูลโน้ต 5 โพสต์ล่าสุด และส่งโน้ตเขียนใหม่ไปบันทึกที่เซิร์ฟเวอร์โดยตรง
* **Offline Caching:** ทุกรอบที่โหลดผ่านเกณฑ์สำเร็จ ให้ทำการเซฟประวัติเป็น JSON string เก็บที่ AsyncStorage
* **Fallback recovery:** ดักจับความล้มเหลวเครือข่ายด้วย try-catch และไปกู้คืนประวัติในเครื่องขึ้นมานำเสนอทันที
* **Interactive Loading UI:** คุมสเตตัสการหมุน ActivityIndicator ในระหว่างทำเน็ตเวิร์กรันไทม์
