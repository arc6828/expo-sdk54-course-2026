# บทที่ 8: การเก็บข้อมูลในเครื่องและการเชื่อมต่อเครือข่าย API (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- ความเข้าใจเกี่ยวกับหน่วยความจำถาวรแบบ Key-Value ด้วยไลบรารี `@react-native-async-storage/async-storage`
- ขั้นตอนการติดตั้งและการเขียนฟังก์ชันบันทึก ดึงข้อมูล และลบข้อมูลผ่าน AsyncStorage
- การจัดการบันทึกและแปลงออบเจกต์โครงสร้างซับซ้อนด้วยคำสั่ง `JSON.stringify()` และ `JSON.parse()`
- สถาปัตยกรรมการดึงและส่งข้อมูลผ่านเครือข่ายภายนอก (REST API) โดยใช้ Fetch API
- การเขียนแบบฟอร์มส่งข้อมูล (HTTP POST Method) เพื่อสร้างหรือแก้ไขข้อมูลบนฐานข้อมูลเซิร์ฟเวอร์
- ระบบจัดการสถานะการทำงานขณะรอคอยข้อมูล (Loading) และการรับมือข้อผิดพลาดอินเทอร์เน็ต (Error Handling)
- การประยุกต์ใช้งาน AsyncStorage ในการจดจำสถานะการล็อกอินเซสชันของผู้ใช้ (User Session Token Cache)

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถอธิบายหลักการบันทึกข้อมูลถาวรบนอุปกรณ์ด้วย AsyncStorage และเขียนโค้ดบันทึก-อ่านประเภทออบเจกต์ได้ถูกต้อง
2. ผู้เรียนสามารถใช้คำสั่ง Fetch API ในการยิง HTTP Request ทั้งรูปแบบ GET และ POST ได้สำเร็จ
3. ผู้เรียนสามารถออกแบบและเขียนสเตตคุมการแสดงผลหน้าจอระหว่างรอรับสายข้อมูล (Loading/ActivityIndicator) ได้ถูกต้อง
4. ผู้เรียนสามารถดักจับข้อผิดพลาดกรณีเครือข่ายล้มเหลว (Try-Catch Error handling) และเขียนกล่องป๊อปอัพแจ้งเตือนผู้ใช้งานได้อย่างเหมาะสม
5. ผู้เรียนสามารถเขียนจำลองจำการจำเซสชันบันทึกโทเค็นเมื่อล็อกอิน และลบเซสชันเมื่อล็อกเอาต์ออกจากแอปได้ถูกต้อง

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายเปรียบเทียบกลไกจัดเก็บข้อมูลแบบถาวร (Storage & Client-Server Architecture Lecture):** บรรยายการเก็บข้อมูลแบบ State (หายไปเมื่อปิดแอป) เทียบกับแบบ AsyncStorage (คงอยู่แม้เปิดแอปใหม่) และอธิบายสเต็ปการติดต่อหลังบ้านผ่าน REST API
2. **การทำ Live Coding Demonstration หัวข้อ Login Session Cache and Posts Submission:** ผู้สอนสาธิตการทำหน้าจอล็อกอินจำลองที่เก็บข้อมูลคีย์โทเค็นลงในหน่วยความจำเครื่อง และเขียนหน้าเขียนบอร์ดกระทู้ส่งข้อมูลใหม่ขึ้นคลาวด์ผ่าน HTTP POST
3. **กิจกรรมแฮกเกอร์สายกู้ข้อมูล (Storage Recovery and Validation Challenge):** ให้ผู้เรียนทดลองดึงข้อมูลประวัติเดิมมาแสดงผลตอนเริ่มต้นเปิดแอปใหม่ โดยทำการสลักแปลงกลับจาก JSON string พร้อมดักจับเงื่อนไขข้อมูลว่างเปล่า
4. **กิจกรรมประลองระบบทนทานเครือข่าย (Offline & Error Boundaries Simulation):** ให้กลุ่มผู้เรียนช่วยกันแก้ไขโค้ดแอปให้ทำงานได้โดยไม่พังเมื่ออินเทอร์เน็ตหลุดกระทันหัน โดยต้องแสดงข้อความเตือนที่เป็นมิตรพร้อมปุ่ม "ลองใหม่อีกครั้ง (Retry)"

## สื่อการเรียนการสอน
1. เอกสารสไลด์เรื่อง "Persistent Local Storage and Remote API Integration in Expo"
2. คลังข้อมูลทดสอบออนไลน์แบบเปิดสาธารณะ (JSONPlaceholder API)
3. รหัสตัวอย่างการดีไซน์ Wrapper Helper สำหรับอ่านและบันทึก AsyncStorage ปลอดภัยแบบเรียลไทม์
4. สภาพแวดล้อมทดสอบเครือข่ายสำหรับจำลองสัญญาณเน็ตขัดข้องบนอุปกรณ์ Simulator

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 ตรวจสอบคะแนนสอบแบบทดสอบปรนัยท้ายบทเรียน 10 ข้อ
1.2 ตรวจสอบโครงสร้างความถูกต้องของการเรียกใช้บล็อก異步 (`async/await`) ในการควบคุมอินเทอร์เฟซ AsyncStorage
1.3 สังเกตการแสดงโครงสร้างสไตล์ตัวหมุนโหลดข้อมูล (Spinner) และการใช้คำสั่งจัดการ JSON ในแอปพลิเคชัน

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถสร้างหน้าแอปบันทึกบันทึกโพสต์และดึง API จากเซิร์ฟเวอร์มาแคชเก็บในโทรศัพท์ได้โดยรันสำเร็จไม่มี Error ผ่านเกณฑ์ (ผ่าน/ไม่ผ่าน)
2.3 สังเกตพฤติกรรมความอดทนในการทดสอบระบบความเสถียรของข้อมูลยามสัญญาณขาดหาย และความใส่ใจในการประมวลผลเซสชันล็อกอิน
