# บทที่ 2: การจัดเลย์เอาต์หน้าจอด้วย Flexbox (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- แนวคิดระบบเลย์เอาต์ใน React Native (ใช้ Flexbox เป็นหลัก แตกต่างจาก CSS Web ทั่วไป)
- การเปรียบเทียบพฤติกรรมการเขียนสไตล์แบบ Inline Styles และการสร้างคลาสสไตล์ด้วย `StyleSheet.create`
- คุณสมบัติพื้นฐานสำหรับการจัดขนาดและขอบเขต: `width`, `height`, `margin`, `padding`, `border`
- ทิศทางการจัดเรียงแกนหลักและแกนรอง: `flex`, `flexDirection` (ค่าเริ่มต้นคือ `column` แตกต่างจากเว็บบราวเซอร์)
- การกระจายพื้นที่และการจัดวางแนวแกน: `justifyContent` และ `alignItems`
- การจัดการแถวที่เกินขนาดและช่องว่างระหว่างคอมโพเนนต์: `flexWrap` และ `gap`

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถอธิบายความแตกต่างของค่าเริ่มต้นของ `flexDirection` ใน React Native เปรียบเทียบกับ CSS บนเว็บได้ถูกต้อง
2. ผู้เรียนสามารถใช้คำสั่งสไตล์พื้นฐานในการปรับแต่งความกว้าง ความสูง ระยะขอบ และเส้นขอบของคอมโพเนนต์ได้ถูกต้อง
3. ผู้เรียนสามารถเขียนสไตล์การจัดแนวทางคอมโพเนนต์โดยใช้ `justifyContent` และ `alignItems` เพื่อจัดโครงสร้างหน้าจอตามที่กำหนดได้
4. ผู้เรียนสามารถประกอบชุดกล่องสไตล์ชีทในรูปแบบรังผึ้ง หรือตาราง (Grid) โดยประยุกต์ใช้คอมโพเนนต์ `View` และ `Text` ได้สำเร็จโดยไม่มีข้อผิดพลาด

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายแบบโต้ตอบ (Interactive Lecture):** นำเสนอภาพสไลด์และกล่องคอนเทนเนอร์เพื่อให้เห็นภาพของการทำงานแบบ Flexbox
2. **กิจกรรมจับผิดเลย์เอาต์ (Visual Layout Quiz):** แสดงภาพ UI ที่บิดเบี้ยวหรือจัดตำแหน่งผิด และให้ผู้เรียนวิเคราะห์ร่วมกันว่าขาดสไตล์ใด
3. **การลงมือปฏิบัติสร้างกล่องสี (Layout Coding Practice):** ให้ผู้เรียนเปิดคอมไพเลอร์ลองจัดหน้าจอเป็นธงชาติ หรือกล่องแม่สี 3 สีในทิศทางต่าง ๆ
4. **การสรุปผลและสะท้อนคิด (Reflection):** ร่วมสรุปความท้าทายในการจัดเลย์เอาต์ที่มีขนาดหน้าจอแตกต่างกัน (Cross-Device Layouts)

## สื่อการเรียนการสอน
1. เอกสารสไลด์เรื่อง "Flexbox Layout System in React Native"
2. บอร์ดเกมจำลองคุณสมบัติ Flexbox เพื่อช่วยในการตัดสินใจเลือกคำสั่ง
3. โจทย์ตัวอย่างการจัดวาง Layout แบบต่าง ๆ (เช่น Header, Sidebar, Grid)
4. ไฟล์เริ่มต้นโปรเจคที่มีการ์ดกล่องสีจำลองเตรียมไว้

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 วัดความรู้จากแบบทดสอบปรนัยท้ายบทเรียน 10 ข้อ
1.2 วัดทักษะจากการตรวจผลงานการต่อบล็อกสี่สีและการเรียงลำดับเนื้อหาในปฏิบัติการท้ายบทเรียน
1.3 ตรวจสอบโครงสร้างความถูกต้องของการเขียนคลาสสไตล์ผ่าน `StyleSheet`

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถเขียนโค้ดจัดหน้าจอจำลองเมนูสี่เหลี่ยมจัดแนว Flexbox ได้ถูกต้องตามต้นแบบผ่านเกณฑ์ (ผ่าน/ไม่ผ่าน)
2.3 พฤติกรรมการกระตือรือร้นในการทำงานและการให้ความร่วมมือแก้ไขโจทย์ร่วมกับผู้อื่นในชั่วโมงเรียน
