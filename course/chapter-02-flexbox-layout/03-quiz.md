# แบบทดสอบท้ายบทเรียน บทที่ 2: การจัดเลย์เอาต์หน้าจอด้วย Flexbox (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความเข้าใจเกี่ยวกับสไตล์ชีท (StyleSheet), กล่องสเกลขนาด (Box Model) และคุณสมบัติของระบบจัดตำแหน่งแบบ Flexbox ใน React Native Expo

---

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: ใน React Native ข้อใดคือทิศทางการจัดเรียงแกนหลักเริ่มต้น (Default flexDirection) ของคอมโพเนนต์ลูกภายในกล่องคอนเทนเนอร์?
ก. `row`  
ข. `column`  
ค. `row-reverse`  
ง. `column-reverse`  

### ข้อ 2: การระบุตัวเลขโดด ๆ ในการกำหนดขนาด เช่น `{ width: 150 }` ในสไตล์ของ React Native มีหน่วยวัดเป็นอะไร?
ก. Pixels (px)  
ข. Percentage (%)  
ค. Points / Density-independent Pixels (dp)  
ง. Inches (in)  

### ข้อ 3: หากต้องการจัดเลย์เอาต์ให้กล่อง View ขนาดกว้าง 100 และสูง 100 มีหน้าตาเป็นวงกลมสมบูรณ์ จะต้องกำหนดคุณสมบัติ `borderRadius` ในสไตล์อย่างไร?
ก. `borderRadius: 100`  
ข. `borderRadius: 50`  
ค. `borderRadius: 25`  
ง. `borderRadius: 10`  

### ข้อ 4: ข้อใดอธิบายความแตกต่างระหว่างคุณสมบัติ `margin` และ `padding` ได้อย่างถูกต้อง?
ก. `margin` คือขนาดเส้นขอบของวัตถุ ส่วน `padding` คือความโค้งมนของมุมกล่อง  
ข. `margin` คือพื้นที่เว้นระยะรอบนอกขอบกล่อง ส่วน `padding` คือพื้นที่เว้นระยะรอบในขอบกล่องไปยังเนื้อหา  
ค. `margin` มีผลกับทิศทางแนวตั้งเท่านั้น ส่วน `padding` มีผลเฉพาะทิศทางแนวนอน  
ง. `margin` ใช้สำหรับข้อความ ส่วน `padding` ใช้สำหรับรูปภาพ  

### ข้อ 5: คุณสมบัติ `alignItems` ใน React Native ทำงานและควบคุมการจัดตำแหน่งในแนวแกนใด?
ก. ควบคุมการจัดเรียงในทิศทางแกนหลัก (Main Axis)  
ข. ควบคุมการจัดเรียงในทิศทางแกนรอง (Cross Axis)  
ค. ควบคุมขนาดของเส้นขอบกล่อง  
ง. ควบคุมการหดตัวของขนาดตัวอักษร  

### ข้อ 6: หากเราเปิดกล่องคอนเทนเนอร์แนวนอนด้วย `flexDirection: 'row'` คุณสมบัติ `justifyContent` จะควบคุมการจัดตำแหน่งและจัดระเบียบข้อมูลตามทิศทางใด?
ก. แนวนอน (ซ้ายไปขวา)  
ข. แนวตั้ง (บนลงล่าง)  
ค. แนวทะแยงมุม  
ง. หมุนตามเข็มนาฬิกา  

### ข้อ 7: ตัวเลือกข้อใดของ `justifyContent` ที่ช่วยกระจายกล่องตัวเลือกให้อยู่ห่างกันมากที่สุด โดยให้บล็อกแรกและบล็อกสุดท้ายชิดขอบข้างสุดของคอนเทนเนอร์พอดี?
ก. `space-around`  
ข. `space-evenly`  
ค. `space-between`  
ง. `center`  

### ข้อ 8: เมื่อความกว้างรวมของบล็อกคอมโพเนนต์ลูกมีมากกว่าขนาดพื้นที่ของกล่องแม่ คุณสมบัติใดที่จะต้องตั้งค่าเพิ่มเติมเป็น `'wrap'` เพื่อจัดให้บล็อกที่ล้นหน้าจอตัดขึ้นแถวใหม่โดยอัตโนมัติ?
ก. `flexWrap`  
ข. `flexDirection`  
ค. `alignContent`  
ง. `flexGrow`  

### ข้อ 9: ใน Expo SDK 54 คุณสมบัติข้อใดที่เพิ่มความสะดวกในการกำหนดระยะห่างระหว่างบล็อกวัตถุในคอนเทนเนอร์แนวตั้งและแนวนอนพร้อมกันโดยไม่ต้องเขียนระยะขอบแยกทีละชิ้น?
ก. `marginHorizontal`  
ข. `gap`  
ค. `spacing`  
ง. `paddings`  

### ข้อ 10: หากมีบล็อกลูกสองกล่องอยู่ในคอนเทนเนอร์เดียวกัน และกล่องแรกถูกกำหนดให้มีสไตล์ `flex: 1` ในขณะที่กล่องที่สองกำหนดเป็น `flex: 2` ข้อใดอธิบายการจัดสรรสัดส่วนพื้นที่ว่างได้ถูกต้อง?
ก. กล่องแรกจะได้ความยาว 100px และกล่องที่สองได้ 200px  
ข. กล่องแรกและกล่องที่สองจะกินพื้นที่เท่ากันโดยสลับกัน  
ค. ระบบจะแบ่งพื้นที่เป็น 3 ส่วนเท่า ๆ กัน โดยกล่องแรกจะได้ 1 ส่วน และกล่องที่สองจะได้ 2 ส่วนที่เหลือ  
ง. กล่องแรกจะซ้อนอยู่บนกล่องที่สองเนื่องจากค่าดัชนีน้อยกว่า

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:** 
  * **ก. ผิด:** `row` เป็นค่าเริ่มต้นสำหรับ CSS บนบราวเซอร์ทั่วไป แต่ไม่ใช่ของ React Native
  * **ข. ถูก:** React Native ออกแบบมาสำหรับมือถือซึ่งเน้นการอ่านและการเลื่อนลงแนวตั้งเป็นหลัก ดังนั้นทิศทางแกนหลักเริ่มต้น (`flexDirection`) จึงถูกกำหนดเป็น `column` (แนวตั้ง) โดยอัตโนมัติ
  * **ค. และ ง. ผิด:** เป็นรูปแบบการสลับย้อนทางแกน (Reverse) ซึ่งไม่ใช่ค่ามาตรฐานเริ่มต้น

### เฉลย ข้อ 2: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** React Native ไม่ได้กำหนดสไตล์ด้วยพิกเซลกายภาพตรงตัวเพราะขนาดจอและปริมาณพิกเซลแต่ละเครื่องต่างกันมาก
  * **ข. ผิด:** เปอร์เซ็นต์จะทำงานเฉพาะเมื่อใส่ค่าเป็นข้อความสตริง เช่น `'50%'` เท่านั้น
  * **ค. ถูก:** ค่าตัวเลขในสไตล์ชีทของ React Native คือหน่วยจุดวัดเชิงพื้นที่ที่ไม่ขึ้นกับความหนาแน่นพิกเซลของจอแต่ละรุ่น (เรียกว่า Points ใน iOS หรือ Density-independent Pixels - dp ใน Android) ซึ่งจะสเกลพิกเซลจริงขึ้นตามหน้าจออุปกรณ์ปลายทางให้อัตโนมัติ
  * **ง. ผิด:** นิ้วไม่ใช่หน่วยวัดของเลย์เอาต์หน้าจอแอปพลิเคชัน

### เฉลย ข้อ 3: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** การตั้งค่าเท่าความกว้างจะทำให้โค้งมนล้นหรืออาจกลายเป็นรูปแคปซูลหากสัดส่วนอื่นเบี้ยว
  * **ข. ถูก:** ตามสูตรเลขคณิตการตัดมุมกล่องสี่เหลี่ยมจัตุรัสให้กลายเป็นวงกลมสมบูรณ์ ขอบโค้งมน (`borderRadius`) จะต้องมีขนาดเท่ากับ **ครึ่งหนึ่ง** ของความยาวกว้างและสูง (กว้าง 100, สูง 100 -> ครึ่งหนึ่งคือ 50)
  * **ค. และ ง. ผิด:** จะได้มุมกล่องโค้งงอเฉย ๆ แต่ขอบยังเป็นทรงสี่เหลี่ยมด้านเท่า

### เฉลย ข้อ 4: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ขอบวัตถุและมุมโค้งใช้อีกคลาสคือ `borderWidth` และ `borderRadius`
  * **ข. ถูก:** ในสถาปัตยกรรม Box Model ของการวางเลย์เอาต์ `margin` (ระยะขอบภายนอก) จะช่วยสร้างช่องไฟว่าง ๆ ผลักตัวบล็อกให้ออกห่างจากกล่องอื่น ส่วน `padding` (ระยะขอบภายใน) จะใช้เว้นช่องไฟภายในกล่องสีของตนเองเพื่อไม่ให้ตัวหนังสือชิดขอบในเกินไป
  * **ค. และ ง. ผิด:** ทั้งสองคำสั่งทำงานได้ทุกแนวทิศทางและควบคุมช่องว่างไม่ได้จำกัดประเภทเนื้อหา

### เฉลย ข้อ 5: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** แกนหลักจะถูกควบคุมและเรียงบล็อกผ่านคุณสมบัติ `justifyContent`
  * **ข. ถูก:** `alignItems` จะทำหน้าที่ควบคุมและจัดแนวระเบียบให้กับคอมโพเนนต์ย่อยตามทิศทางแนวแกนรอง (Cross Axis) เสมอ (เช่น หากแถวเรียงตัวแนวนอน แกนรองจะเป็นแนวตั้ง)
  * **ค. และ ง. ผิด:** ควบคุมขนาดเส้นขอบและตัวอักษรจะใช้ `borderWidth` และ `fontSize`

### เฉลย ข้อ 6: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** หากมีการกำหนด `flexDirection: 'row'` (เรียงแนวนอน) ทิศทางแกนหลัก (Main Axis) จะสลับมาอยู่ทางแนวนอนทันที ส่งผลให้ `justifyContent` ทำหน้าที่ผลักหรือจัดบล็อกเรียงชิดตามแนวนอน (ซ้าย-ขวา) ไปด้วย
  * **ข. ผิด:** แกนแนวตั้งจะกลายเป็นแกนรอง (Cross Axis) และถูกควบคุมโดย `alignItems` แทน
  * **ค. และ ง. ผิด:** ระบบ Flexbox ไม่มีการจัดมุมหรือหมุนทิศทางทะแยงในการทำงานหลัก

### เฉลย ข้อ 7: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `space-around` จะแบ่งช่องว่างระหว่างบล็อกเท่า ๆ กันและเหลือช่องว่างข้างหน้า/ท้ายครึ่งหนึ่ง
  * **ข. ผิด:** `space-evenly` จะแบ่งช่องว่างให้กว้างเท่ากันทั้งหมดอย่างเท่าเทียมรวมถึงขอบริมด้วย
  * **ค. ถูก:** `space-between` จะกระจายบล็อกลูกชิดซ้ายและชิดขวาสุดริมขอบหน้าจอพอดี และกระจายสัดส่วนสเปซช่องว่างที่เหลือไว้คั่นตรงกึ่งกลางระว่างกล่องทั้งหมดเท่ากัน
  * **ง. ผิด:** `center` จะรวมกล่องทั้งหมดชิดกึ่งกลางแกน

### เฉลย ข้อ 8: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** คุณสมบัติ `flexWrap: 'wrap'` จะคอยสั่งการตัว Bunder เลย์เอาต์ว่า หากความกว้างสะสมของคอมโพเนนต์ลูกเกินพื้นที่คอนเทนเนอร์แม่ ให้ผลักบล็อกถัดไปลงมาขึ้นแถวใหม่ด้านล่างทันที
  * **ข. ผิด:** `flexDirection` ใช้กำหนดเพียงทิศทางการไหลหลักของแกนว่าวางตัวแนวตั้งหรือนอน
  * **ค. และ ง. ผิด:** ใช้สำหรับคำสั่งสัดส่วนการหดและจัดแนวย่อยเมื่อมีหลายแถว

### เฉลย ข้อ 9: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `marginHorizontal` จะเว้นขอบริมภายนอกทั้งซ้ายและขวาของกล่องแต่ละตัว แต่ไม่มีผลกึ่งกลางระหว่างบล็อกย่อย
  * **ข. ถูก:** คำสั่ง `gap` (มีพร้อมใช้งานใน SDK ใหม่) ช่วยให้นักพัฒนาสร้างช่องไฟเว้นระหว่างบล็อกย่อยภายในคอนเทนเนอร์ Flexbox ได้ง่ายที่สุด โดยจะคั่นระหว่างบล็อกเท่านั้น โดยไม่ต้องคอยระบุหักลบขอบริมด้านซ้ายขวาของแต่ละกล่อง
  * **ค. และ ง. ผิด:** ไม่ใช่ชื่อคุณสมบัติมาตรฐานที่จัดทำในระบบสไตล์ React Native

### เฉลย ข้อ 10: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ค่าตัวเลขใน `flex` ไม่ได้หมายความถึงพิกเซล (px) แบบตายตัว แต่เป็นหน่วยอัตราส่วน
  * **ข. ผิด:** จะกินสัดส่วนขนาดที่ไม่เท่ากันเพราะตัวเลขต่างกัน
  * **ค. ถูก:** คุณสมบัติ `flex` ทำงานโดยหาผลรวมค่าสะสมทั้งหมด (ในโจทย์ 1 + 2 = 3 ส่วน) แล้วแบ่งสเกลสัดส่วนพื้นที่ว่างออกเป็นจำนวนส่วนรวมนั้น จากนั้นจึงนำไปปันสัดส่วนให้กับกล่องตามค่าที่ระบุ (กล่องแรกได้ 1/3 ส่วน และกล่องที่สองได้ 2/3 ส่วน)
  * **ง. ผิด:** การซ้อนทับภาพขึ้นอยู่กับลำดับการโค้ดหรือคุณสมบัติ `zIndex` ไม่ใช่ `flex`
