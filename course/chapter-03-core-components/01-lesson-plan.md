# บทที่ 3: คอมโพเนนต์พื้นฐานและการจัดการปฏิสัมพันธ์กับผู้ใช้ (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- คอมโพเนนต์แสดงผลรูปภาพ `Image` (การดึงภาพจาก Local Assets และเครือข่าย Network, การใช้ `resizeMode`)
- คอมโพเนนต์รับข้อมูลข้อความ `TextInput` (การจัดการคีย์บอร์ด, ชนิดตัวแปรป้อนค่า, พร็อพส์ `placeholder`, `onChangeText` และ `secureTextEntry` สำหรับรหัสผ่าน)
- การสร้างมุมมองที่สามารถเลื่อนหน้าจอได้ด้วย `ScrollView` และความแตกต่างเมื่อเปรียบเทียบกับ `View` ทั่วไป
- การจัดการเหตุการณ์การสัมผัสและการกดปุ่มด้วยคอมโพเนนต์ `Pressable` (การใช้สถานะ Active/Pressed และการเปรียบเทียบข้อดีเมื่อเทียบกับ Touchables รุ่นเก่า)
- การป้องกันการแสดงผลทับซ้อนพื้นที่ขอบจอ (เช่น รอยบาก กล้อง หรือแถบสถานะ) ด้วยคอมโพเนนต์ `SafeAreaView`
- **แนวคิดของ Props:** การสร้างคอมโพเนนต์ย่อยที่กำหนดขึ้นเอง (Custom Components) และการรับส่งข้อมูลระหว่างคอมโพเนนต์ด้วยคุณสมบัติ Props
- **การประกาศ Props ใน TypeScript:** การใช้ `interface` เพื่อกำหนดประเภทข้อมูลของ Props เพื่อป้องกันข้อผิดพลาดในการส่งข้อมูล

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถอธิบายวิธีแสดงผลภาพจากสองแหล่งข้อมูล (Local และ Network URI) และการปรับสัดส่วนรูปภาพได้ถูกต้อง
2. ผู้เรียนสามารถใช้ `TextInput` ในการรับข้อมูลประเภทต่าง ๆ และผูกเหตุการณ์ตรวจสอบข้อความผ่านคีย์บอร์ดได้สำเร็จ
3. ผู้เรียนสามารถระบุความแตกต่างและเปรียบเทียบกรณีใช้งานที่เหมาะสมระหว่าง `View` และ `ScrollView` ได้อย่างถูกต้อง
4. ผู้เรียนสามารถสร้างปุ่มสัมผัสที่ตอบสนองเมื่อถูกกดโดยประยุกต์ใช้คอมโพเนนต์ `Pressable` ร่วมกับการจัดระเบียบขอบเขตจอด้วย `SafeAreaView` ได้อย่างถูกต้อง
5. ผู้เรียนสามารถสร้าง Custom Component และเขียนโค้ดส่งผ่านค่าข้อมูล (Props) ไปทำงานภายในคอมโพเนนต์ย่อยได้อย่างมีหลักการ
6. ผู้เรียนสามารถเขียนโครงสร้าง Interface ใน TypeScript เพื่อกำกับและตรวจสอบความถูกต้องของประเภทข้อมูล Props ที่ส่งเข้ามาได้ถูกต้องสมบูรณ์

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายประกอบการสาธิตเชิงปฏิบัติ (Live Coding Demonstration):** ผู้สอนสาธิตวิธีการแสดงผลรูปภาพ การพิมพ์รับค่าข้อความ การจัดการคีย์บอร์ด ตลอดจนวิธีการแยกโค้ดสร้าง Custom Component และการส่งค่าข้อมูลผ่าน Props
2. **กิจกรรมต่อยอดโค้ดทีละส่วน (Guided Coding Practice):** ให้ผู้เรียนทดลองสร้างคอมโพเนนต์ปุ่มสัมผัสเป็นของตัวเอง (Custom Button) โดยเปิดให้ส่งข้อความหน้าปุ่มและสีปุ่มต่าง ๆ ผ่านคุณสมบัติ Props เข้าไปปรับเปลี่ยนสไตล์
3. **การลงมือปฏิบัติทดสอบปฏิสัมพันธ์ (Interactive Interaction Lab):** มอบหมายให้ผู้เรียนเขียนแบบฟอร์มรับค่าจำลอง โดยเรียกใช้ Custom Components ที่สร้างขึ้น และจัดส่งผ่านโครงสร้าง Props ของข้อมูลผู้เรียนไปทำงาน
4. **กิจกรรมเรียนรู้เรื่อง Type-Safety (Props Interface Practice):** ฝึกเขียนและจำลองข้อผิดพลาด (Debug) เมื่อส่งประเภทข้อมูลตัวแปร Props ผิดกฎที่ระบุไว้ใน TypeScript Interface

## สื่อการเรียนการสอน
1. เอกสารสไลด์นำเสนอหัวข้อ "Core Components and Custom Components with Props in React Native"
2. วิดีโอแนะนำแนวคิดและการแยกส่วนคอมโพเนนต์ (Component Modularization) ด้วย Props
3. ชุดไอคอนรูปภาพประกอบการทดลองรันคอมโพเนนต์ Image
4. ตัวอย่างซอร์สโค้ดการแยกไฟล์ Custom Button และส่งค่า Props บน Git Repository

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 ประเมินผลจากการทำคะแนนแบบทดสอบปรนัยท้ายบทเรียน 10 ข้อ
1.2 ประเมินทักษะการแยกสร้าง Custom Component และการส่งผ่านตัวแปร Props ระหว่างคอมโพเนนต์ในห้องเรียน
1.3 ตรวจสอบความถูกต้องของการใช้งานคอมโพเนนต์ `Image`, `TextInput` และการตั้งค่า Props Interface ด้วย TypeScript ในแล็บปฏิบัติการท้ายบท

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถสร้าง Custom Component ที่รับส่งค่า Props ได้ถูกต้อง และรันหน้าจอแอปแบบฟอร์มที่มีรูปภาพและกล่องพิมพ์ข้อความได้สำเร็จผ่านเกณฑ์ (ผ่าน/ไม่ผ่าน)
2.3 การสังเกตพฤติกรรมความตั้งใจเรียน ความพยายามในการดีบักข้อผิดพลาดของการส่งค่าตัวแปร และการมีส่วนร่วมในชั้นเรียน
