# แบบทดสอบท้ายบทเรียน บทที่ 3: คอมโพเนนต์พื้นฐานและการจัดการปฏิสัมพันธ์กับผู้ใช้ (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความเข้าใจเกี่ยวกับคอมโพเนนต์พื้นฐาน (Core Components เช่น `Image`, `TextInput`, `ScrollView`, `Pressable`, `SafeAreaView`) และแนวคิดการจัดการ Props ร่วมกับการใช้ TypeScript ใน React Native Expo SDK 54

---

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: เมื่อต้องการแสดงผลรูปภาพจากเครือข่ายอินเทอร์เน็ต (Network Image) ด้วยคอมโพเนนต์ `Image` ข้อใดคือรูปแบบการเขียนและข้อกำหนดที่สำคัญที่สุดเพื่อให้รูปภาพแสดงผลขึ้นมา?
ก. `<Image source="https://example.com/pic.png" />` โดยไม่ต้องระบุขนาด  
ข. `<Image source={{ uri: 'https://example.com/pic.png' }} />` โดยไม่ต้องระบุขนาด  
ค. `<Image source={{ uri: 'https://example.com/pic.png' }} style={{ width: 100, height: 100 }} />`  
ง. `<Image localPath="https://example.com/pic.png" style={{ width: 100, height: 100 }} />`  

### ข้อ 2: ค่า resizeMode รูปแบบใดของคอมโพเนนต์ `Image` ที่ทำหน้าที่หดหรือขยายรูปภาพเพื่อให้อยู่ภายใต้กรอบของคอมโพเนนต์ได้ครบถ้วนทั้งภาพ โดยยังคงรักษาอัตราส่วนภาพเดิมไว้ครบถ้วน?
ก. `cover`  
ข. `contain`  
ค. `stretch`  
ง. `center`  

### ข้อ 3: พร็อพส์ (Props) ใดของ `TextInput` ที่ใช้ในการกำหนดให้อักขระที่พิมพ์แสดงผลเป็นจุดกลมดำเพื่อป้องกันความปลอดภัยในการกรอกรหัสผ่าน (Password)?
ก. `passwordMask={true}`  
ข. `secureTextEntry={true}`  
ค. `keyboardType="password"`  
ง. `autoCapitalize="none"`  

### ข้อ 4: เมื่อต้องการรับอินพุตเฉพาะค่าตัวเลข (เช่น ยอดเงิน หรือรหัส OTP) ผู้พัฒนาควรตั้งค่าพร็อพส์ `keyboardType` บน `TextInput` อย่างไร?
ก. `keyboardType="email-address"`  
ข. `keyboardType="phone-pad"`  
ค. `keyboardType="numeric"`  
ง. `keyboardType="default"`  

### ข้อ 5: ข้อใดอธิบายความแตกต่างในพฤติกรรมการเรนเดอร์เนื้อหาระหว่าง `View` และ `ScrollView` ได้อย่างถูกต้อง?
ก. `View` สามารถเลื่อนขึ้นลงได้อัตโนมัติ แต่ `ScrollView` จะไม่สามารถเลื่อนหน้าจอได้  
ข. `View` เหมาะสำหรับการจัดแถวในเลย์เอาต์ ส่วน `ScrollView` จะจัดแนวด้วย Flexbox ได้ไวกว่า  
ค. `View` จะซ่อนเนื้อหาที่ล้นขอบจอและไม่มีแถบเลื่อน ส่วน `ScrollView` จะแสดงพื้นที่และช่วยให้ใช้นิ้วปัดเลื่อนอ่านเนื้อหาที่เกินขอบจอได้  
ง. `View` ใช้สำหรับแสดงรูปภาพเท่านั้น ส่วน `ScrollView` ใช้แสดงตัวอักษรเท่านั้น  

### ข้อ 6: หากต้องการระบุสไตล์ของระยะห่างภายใน (Padding) หรือจัดกึ่งกลางให้กับคอมโพเนนต์ย่อยที่อยู่ภายใต้ `ScrollView` จะต้องส่งคำสั่งสไตล์ผ่านพร็อพส์ใดจึงจะทำงานได้ถูกต้อง?
ก. `style`  
ข. `contentContainerStyle`  
ค. `innerStyle`  
ง. `scrollStyle`  

### ข้อ 7: ข้อใดกล่าวถึงประโยชน์และความสามารถเด่นของคอมโพเนนต์ `Pressable` ที่โดดเด่นกว่าคอมโพเนนต์ในกลุ่ม Touchable รุ่นดั้งเดิมได้ถูกต้องที่สุด?
ก. `Pressable` สามารถแสดงผลภาพเคลื่อนไหว GIF ได้ในตัวเอง  
ข. `Pressable` รองรับการปรับเปลี่ยนรูปแบบสไตล์ชีท (Style) และคอมโพเนนต์ลูก (Children) แบบไดนามิกตามสถานะการกดสัมผัสจริงของนิ้วมือผู้ใช้งานผ่านค่า `{ pressed }`  
ค. `Pressable` ทำการเชื่อมต่ออินเทอร์เน็ตได้เร็วกว่าคอมโพเนนต์อื่น  
ง. `Pressable` ไม่จำเป็นต้องมีพร็อพส์ `onPress` ในการดักจับพฤติกรรมผู้ใช้งาน  

### ข้อ 8: วัตถุประสงค์หลักของการประยุกต์ใช้คอมโพเนนต์ `SafeAreaView` คือข้อใด?
ก. เพื่อสแกนและล้างไวรัสออกจากตัวเครื่องสมาร์ทโฟนที่เปิดแอป  
ข. เพื่อบันทึกข้อมูลอินพุตของผู้ใช้ลงในฐานข้อมูลเครื่องอย่างปลอดภัย  
ค. เพื่อหลีกเลี่ยงการแสดงผลของเนื้อหาแอปพลิเคชันไปทับซ้อนกับพื้นที่ทางกายภาพ เช่น รอยบาก กล้อง หรือแถบแสดงสถานะระบบของเครื่อง  
ง. เพื่อเร่งความเร็วในการประมวลผลโค้ด JavaScript ให้ไวขึ้น  

### ข้อ 9: ภายใต้กฎพื้นฐานของไลบรารี React และ React Native ข้อใดอธิบายทิศทางการไหลของข้อมูลผ่านคุณสมบัติ Props ได้ถูกต้อง?
ก. ไหลจากคอมโพเนนต์ลูกย้อนกลับไปแก้ไขค่าในคอมโพเนนต์แม่โดยตรง  
ข. ไหลแบบทิศทางเดียว (Unidirectional) จากคอมโพเนนต์แม่ส่งต่อไปยังคอมโพเนนต์ลูก และเป็นข้อมูลแบบอ่านอย่างเดียว (Read-only) สำหรับลูก  
ค. ข้อมูลจะไหลเวียนและแก้ไขได้อิสระผ่านทุกคอมโพเนนต์ในโปรเจกต์โดยไม่ต้องประกาศรับ  
ง. ไหลเฉพาะจากไฟล์ CSS เข้าสู่เนื้อหาข้อความในแอปพลิเคชันเท่านั้น  

### ข้อ 10: ในการเขียนแบบมีระเบียบ (Type-safe) การระบุเครื่องหมายคำถามท้ายชื่อตัวแปรใน TypeScript Interface เช่น `avatarUrl?: string;` มีความหมายอย่างไร?
ก. ตัวแปร `avatarUrl` จะต้องส่งค่าเป็นประเภทข้อมูล Boolean เท่านั้น  
ข. ตัวแปร `avatarUrl` มีสถานะเป็น Optional Props (ผู้ใช้งานคอมโพเนนต์ย่อยจะส่งค่ามาให้หรือไม่ส่งค่ามาให้ก็ได้โดยไม่เกิดข้อผิดพลาดในการรันไทม์)  
ค. ตัวแปร `avatarUrl` ห้ามนำไปใช้งานภายในคอมโพเนนต์เด็ดขาด  
ง. ตัวแปร `avatarUrl` จะมีระบบสุ่มค่าข้อความให้อัตโนมัติ

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** พร็อพส์ `source` ไม่รับสตริง URL ตรง ๆ สำหรับรูปภาพจากอินเทอร์เน็ต แต่ต้องประกาศผ่านโครงสร้างออบเจกต์ที่มีพร็อพส์ `uri`
  * **ข. ผิด:** แม้จะระบุเป็นรูปเล่มออบเจกต์ `uri` แล้ว แต่เนื่องจากรูปภาพผ่านเครือข่ายไม่มีตัวตนในเครื่องก่อนบิลด์ แอปพลิเคชันจึงไม่ทราบสัดส่วนขนาด หากไม่เขียนระบุกว้าง-สูง (`style={{ width, height }}`) ขนาดภาพจะถูกเซ็ตเป็น 0x0 ทำให้ไม่ปรากฏรูป
  * **ค. ถูก:** ต้องเขียนระบุรูปเล่มออบเจกต์ `{ uri: 'URL' }` ร่วมกับการประกาศสไตล์ควบคุมขนาดกว้างและสูงอย่างเด่นชัดเพื่อให้ระบบแสดงรูป
  * **ง. ผิด:** ไม่มีพร็อพส์ชื่อ `localPath` ในการดึงรูปภาพของคอมโพเนนต์ `Image`

### เฉลย ข้อ 2: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `cover` จะดึงขยายรูปภาพให้เต็มกรอบ ซึ่งอาจทำให้อัตราส่วนของภาพยังคงเดิมแต่ส่วนที่เลยกรอบออกไปจะถูกตัดออก (Cropped)
  * **ข. ถูก:** `contain` จะปรับเปลี่ยนขนาดรูปภาพให้อยู่ในขอบเขตกรอบทั้งหมดโดยที่ไม่มีชิ้นส่วนใดถูกบีบเบี้ยวหรือตัดทิ้ง โดยรักษาอัตราส่วนภาพเดิมไว้ร้อยเปอร์เซ็นต์
  * **ค. ผิด:** `stretch` จะยืดสัดส่วนแกนให้พอดีกล่องซึ่งส่งผลให้หน้าตาของรูปภาพเกิดการยืดบิดเบี้ยวได้
  * **ง. ผิด:** `center` จะวางรูปภาพขนาดปกติไว้กึ่งกลางกล่องโดยไม่มีการขยายขนาดรูปภาพ

### เฉลย ข้อ 3: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ไม่มีพร็อพส์ชื่อ `passwordMask` ในคอมโพเนนต์ `TextInput`
  * **ข. ถูก:** การตั้งค่า `secureTextEntry={true}` เป็นมาตรฐานของ React Native ในการสั่งให้ช่องรับข้อมูลซ่อนอักขระจริง และเรนเดอร์เป็นสัญลักษณ์จุดวงกลมเพื่อความปลอดภัย
  * **ค. ผิด:** `keyboardType` ใช้เปลี่ยนปุ่มแป้นพิมพ์คีย์บอร์ด ไม่ใช่หน้าตาข้อความข้างในช่องรับค่า
  * **ง. ผิด:** `autoCapitalize` เป็นเพียงคำสั่งปิดตัวพิมพ์ใหญ่ภาษาอังกฤษ ซึ่งไม่มีฟังก์ชันบังขอบเขตข้อความ

### เฉลย ข้อ 4: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `email-address` คีย์บอร์ดจะมีปุ่มสัญลักษณ์ตัวอักษรและปุ่ม `@` โผล่มาหลัก ๆ
  * **ข. ผิด:** `phone-pad` จะแสดงตัวเลขรวมถึงเครื่องหมายที่ใช้ต่อสายโทรศัพท์ เช่น `*`, `#`, `+` ซึ่งเกินกว่าตัวเลขข้อมูลปกติ
  * **ค. ถูก:** การส่งค่า `keyboardType="numeric"` จะสั่งให้อุปกรณ์ระบบปฏิบัติการเปลี่ยนมาแสดงแป้นพิมพ์ปุ่มหลักเฉพาะตัวเลข 0-9 เพื่อความรวดเร็วและป้องกันไม่ให้ผู้ใช้พิมพ์ตัวอักษรตัวหนังสือที่ผิดรูปแบบเข้ามาได้
  * **ง. ผิด:** `default` จะเรียกคีย์บอร์ดปุ่มกดตัวอักษรข้อความปกติของสมาร์ทโฟนขึ้นมาทำงาน

### เฉลย ข้อ 5: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** สลับกัน คอมโพเนนต์ `View` ไม่สามารถเลื่อนหน้าจอได้
  * **ข. ผิด:** ทั้งสองตัวทำสไตล์จัดสัดส่วนผ่าน Flexbox ได้ไม่ต่างกันในด้านความยากง่าย
  * **ค. ถูก:** `View` มีพฤติกรรมแบบคงที่ หากเนื้อหาภายในมีขนาดใหญ่หรือยาวล้นออกไปนอกพื้นที่ของแม่ เนื้อหาที่ล้นจะถูกตัดการแสดงผลทับไปและเลื่อนดูไม่ได้ ส่วน `ScrollView` จะมีความสามารถดักจับการปัดสัมผัสเลื่อนแกนเพื่อขยับวัตถุที่ล้นเข้ามาในหน้าจอได้ทันที
  * **ง. ผิด:** ทั้งสองคอมโพเนนต์ใช้จัดวางหรือใส่ประเภทชิ้นงานรูปภาพข้อความข้างในได้ตามกฎปกติ

### เฉลย ข้อ 6: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** การใช้ `style` บน `ScrollView` จะทำงานควบคุมเฉพาะขอบเขตสไตล์ภายนอกของตัวกรอบเลื่อน แต่ไม่ส่งสไตล์จัดแนวลงไปหาคอมโพเนนต์ลูกด้านใน
  * **ข. ถูก:** พร็อพส์ `contentContainerStyle` ถูกออกแบบมาเพื่อนำสไตล์การจัดช่องไฟ (เช่น `padding`) หรือการจัดตำแหน่ง (เช่น `justifyContent`, `alignItems`) ส่งเข้าไปควบคุมกล่องห่อหุ้มลูกภายในพื้นที่เลื่อนได้สมบูรณ์
  * **ค. และ ง. ผิด:** ไม่มีพร็อพส์ชื่อดังกล่าวใน React Native SDK

### เฉลย ข้อ 7: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `Pressable` ไม่เกี่ยวข้องกับการจำลองรันไฟล์ภาพเคลื่อนไหว
  * **ข. ถูก:** `Pressable` เป็นคอมโพเนนต์รุ่นใหม่ที่ออกแบบมาให้ตรวจเช็คสัญญานแรงกดนิ้วได้ยืดหยุ่น โดยสามารถประเมินพารามิเตอร์ `{ pressed }` เพื่อนำไปเปลี่ยนสีปุ่ม ขนาด หรือสไตล์ของวัตถุข้างในได้แบบเรียลไทม์ ทำให้แอปพลิเคชันดูมีชีวิตชีวาและโต้ตอบได้สมบูรณ์
  * **ค. ผิด:** ไม่มีความเกี่ยวดองกับการทำงานเน็ตเวิร์กของระบบ
  * **ง. ผิด:** ยังคงจำเป็นต้องใช้พร็อพส์ `onPress` ในการทำงานดักจับสัมผัสคลิกปุ่มเช่นเดียวกับปุ่มทั่วไป

### เฉลย ข้อ 8: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ไม่มีระบบรักษาความปลอดภัยประเภทกำจัดโปรแกรมที่เป็นภัยต่อตัวเครื่อง
  * **ข. ผิด:** การบันทึกเก็บข้อมูลดั้งเดิมจะพึ่งพากลุ่ม Local Storage เช่น AsyncStorage ไม่ใช่หน้าที่ของ View
  * **ค. ถูก:** `SafeAreaView` ทำหน้าที่ปรับปรุงระเบียบกล่องเลย์เอาต์โดยการตัดพิกเซลเว้นระยะในจุดที่มีความเสี่ยงต่อการโดนฮาร์ดแวร์ขอบหน้าจอบัง (เช่น ขอบบนที่ติดกล้อง รอยบาก หรือขอบล่างของปุ่มนำทาง Home Indicator) ให้เรียบร้อยโดยไม่ต้องปรับเอง
  * **ง. ผิด:** ไม่ได้มีคุณลักษณะในการคอมไพล์โค้ดหรือปรับปรุงประสิทธิภาพ JavaScript Engine

### เฉลย ข้อ 9: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** คอมโพเนนต์ลูกไม่สามารถเข้าทำรายการหรือแก้ไขตัวแปรต้นฉบับในคอมโพเนนต์แม่ผ่าน Props ได้โดยตรง ยกเว้นแต่ว่าคอมโพเนนต์แม่จะส่งฟังก์ชันสำหรับ Callback ลงไปให้ใช้งาน
  * **ข. ถูก:** ตามหลักสถาปัตยกรรม React การส่งข้อมูลผ่าน Props จะมีลักษณะจากบนลงล่าง (จาก Parent สู่ Child) เท่านั้น และตัวคอมโพเนนต์ลูกที่ได้รับ Props ไปจะมีสถานะใช้อ่านอย่างเดียว (Immutable/Read-only) เพื่อไม่ให้สเตตของโปรแกรมสับสนกระจัดกระจาย
  * **ค. ผิด:** ต้องระบุตัวแปรที่รับมาในพารามิเตอร์ฟังก์ชันเสมอเพื่อให้ TypeScript และ JavaScript อ้างอิงตัวแปรได้ถูกต้อง
  * **ง. ผิด:** ข้อมูล Props ไม่เกี่ยวกับการเปลี่ยนทางสไตล์ชีทแบบดังกล่าว

### เฉลย ข้อ 10: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ชนิดของตัวแปรในโจทย์ระบุชัดเจนว่าเป็น `string`
  * **ข. ถูก:** ใน TypeScript การเขียนต่อท้ายคีย์ด้วยเครื่องหมายคำถาม (`?`) บ่งบอกว่าพร็อพส์นั้น ๆ เป็น "Optional" (ไม่จำเป็นต้องระบุ) ทำให้เวลาเรียกใช้คอมโพเนนต์ย่อยนี้ เราไม่จำเป็นต้องป้อนค่าพร็อพส์ตัวนี้เสมอไป ซึ่งช่วยรันระบบได้เป็นระเบียบและลดคำสั่งแจ้งเตือนความผิดพลาด (Linter warning)
  * **ค. ผิด:** ยังคงสามารถอ้างถึงและใช้งานตัวแปรได้ตามปกติ เพียงแต่ค่าอาจเป็น `undefined` หากไม่มีการส่งมาจากแม่
  * **ง. ผิด:** ระบบ TypeScript ทำหน้าที่เพียงตรวจสอบประเภทตัวแปร ไม่สามารถประดิษฐ์สุ่มคำให้เกิดขึ้นเองรันไทม์ได้
