# บทที่ 4: การจัดการสถานะและการแสดงผลรายการข้อมูล (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- แนวคิดและกลไกของสถานะ (State) ใน React Native ด้วยการใช้งาน `useState` Hook
- ข้อแตกต่างและการทำงานร่วมกันระหว่าง State (สถานะภายในเปลี่ยนแปลงได้) และ Props (คุณสมบัติภายนอกใช้อ่านอย่างเดียว)
- การส่งผ่านและแบ่งปันสถานะระหว่างคอมโพเนนต์ด้วยหลักการยกสถานะขึ้น (Lifting State Up)
- ข้อจำกัดด้านประสิทธิภาพของ `ScrollView` เมื่อแสดงผลรายการข้อมูลขนาดใหญ่
- การประยุกต์ใช้คอมโพเนนต์ `FlatList` สำหรับเรนเดอร์ข้อมูลรายการปริมาณมาก (Virtualization, Lazy Rendering)
- โครงสร้างและพร็อพส์ที่จำเป็นของ `FlatList` (`data`, `renderItem`, `keyExtractor`)
- การปรับแต่งองค์ประกอบเพิ่มเติมของรายการด้วย `ListEmptyComponent`, `ListHeaderComponent` และ `ListFooterComponent`
- การสร้างระบบดึงข้อมูลเพื่อรีเฟรชหน้าจอ (Pull-to-Refresh) ด้วยพร็อพส์ `refreshing` และ `onRefresh`

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถอธิบายบทบาท หน้าที่ และความแตกต่างระหว่าง State และ Props ได้อย่างถูกต้อง
2. ผู้เรียนสามารถใช้งาน `useState` Hook ในการสร้างและอัปเดตสเตตของคอมโพเนนต์เพื่อเปลี่ยนแปลงการแสดงผลบนหน้าจอได้สำเร็จ
3. ผู้เรียนสามารถประยุกต์ใช้หลักการ Lifting State Up เพื่อควบคุมและจัดการข้อมูลร่วมกันระหว่างคอมโพเนนต์แม่และลูกได้อย่างถูกต้อง
4. ผู้เรียนสามารถอธิบายสาเหตุและเลือกใช้คอมโพเนนต์ `FlatList` แทน `ScrollView` เพื่อปรับปรุงประสิทธิภาพการทำงานหน่วยความจำได้เหมาะสม
5. ผู้เรียนสามารถเขียนโค้ดเรนเดอร์รายการแบบตาราง/รายการยาวด้วย `FlatList` พร้อมตั้งค่าคีย์เฉพาะ (`keyExtractor`) ได้ถูกต้องสมบูรณ์
6. ผู้เรียนสามารถเปิดระบบการดึงหน้าจอเพื่อรีเฟรชรายการข้อมูล (Pull-to-Refresh) และใส่หน้าจอจำลองกรณีไม่มีข้อมูลได้สำเร็จ

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายและการจำลองพฤติกรรมสเตต (State & Props Visualization Lecture):** บรรยายทฤษฎีและสาธิตการเปลี่ยนแปลงค่าตัวแปรในหน่วยความจำที่มีผลต่อการวาดหน้าจอใหม่ (Re-render) เพื่อชี้ให้เห็นบทบาทสเตต
2. **การทำ Live Coding Demonstration หัวข้อ Performance Challenge:** ผู้สอนจำลองการโหลดข้อมูล 1,000 รายการบน `ScrollView` เปรียบเทียบกับ `FlatList` เพื่อให้ผู้เรียนสังเกตปริมาณการใช้ RAM และความลื่นไหลของแอปผ่านเครื่องมือตรวจสอบ
3. **กิจกรรมปฏิบัติการยกสถานะ (Lifting State Up Activity):** ให้ผู้เรียนจับคู่แก้ไขโค้ดแอปรายการซื้อของย่อย โดยต้องดึงค่าไอเท็มที่เพิ่มจากฟอร์มส่งไปขึ้นแสดงผลที่การ์ดสรุปยอดเงินผ่านการย้าย State ขึ้นไปไว้ที่คอมโพเนนต์ระดับบนสุด
4. **กิจกรรมทดลองปรับแต่งส่วนประกอบลิสต์ (Advanced List Component Tuning):** ฝึกเพิ่มหัวข้อการ์ดตกแต่งส่วนหัวรายการ ลิสต์ว่างเปล่ากรณีไม่มีข้อมูล และฟังก์ชันจำลองหน่วงเวลาอัปโหลดเมื่อลากนิ้วรีเฟรชหน้าจอ

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 ประเมินจากการทำคะแนนแบบทดสอบปรนัยท้ายบทเรียน 10 ข้อ
1.2 ประเมินทักษะการแยกแยะการใช้ State/Props และการส่งฟังก์ชัน Callback อัปเดตสเตตกลับไปยัง Component แม่
1.3 ตรวจประเมินผลลัพธ์ประสิทธิภาพโค้ดและการเลือกใช้ `FlatList` ในการเรนเดอร์ข้อมูลรายการแล็บปฏิบัติการ

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถเขียนโปรแกรมสร้างรายการข้อมูลด้วย `FlatList` ที่รองรับระบบดึงข้อมูลรีเฟรชและจัดการข้อมูลสถานะได้ถูกต้องครบถ้วนตามเกณฑ์ปฏิบัติการ (ผ่าน/ไม่ผ่าน)
2.3 การสังเกตพฤติกรรมการมีส่วนร่วมในการตอบคำถามและช่วยเหลือเพื่อนในกลุ่มระหว่างปฏิบัติงานจริง
