## 2. โครงสร้างโฟลเดอร์และไฟล์บทเรียน
ทุกบทเรียนจะแยกออกเป็นโฟลเดอร์เฉพาะของบทนั้น ๆ โดยใช้รูปแบบการตั้งชื่อ `chapter-XX-[topic-slug]` และในแต่ละโฟลเดอร์จะประกอบไปด้วยเอกสาร Markdown จำนวน 4 ไฟล์ เรียงลำดับดังนี้:

```text
course/
├── README.md                            # คู่มือเล่มนี้ (คู่มือโครงสร้างและข้อกำหนด)
├── chapter-XX-[topic-slug]/
│   ├── 01-lesson-plan.md                # แผนการสอนประจำบท
│   ├── 02-content.md                    # เนื้อหาทางทฤษฎีและคำอธิบายโค้ด
│   ├── 03-quiz.md                       # แบบทดสอบท้ายบทเรียน (Quiz)
│   └── 04-lab.md                        # ปฏิบัติการท้ายบทเรียน (Lab)
```

## 3. มาตรฐานการจัดรูปแบบแต่ละไฟล์ (Style Guide)

### 3.1 แผนการสอนประจำบท (`01-lesson-plan.md`)
ทุกแผนการสอนต้องมีหัวข้อหลักและการจัดรูปแบบรายการ (List) ดังนี้:
* **หัวข้อเนื้อหาประจำบท:** ใช้การจัดรูปแบบรายการเป็นสัญลักษณ์ Bullet Points (`-`)
* **วัตถุประสงค์เชิงพฤติกรรม:** ใช้การจัดรูปแบบรายการเป็นลำดับตัวเลข (`1.`, `2.`)
* **วิธีสอนและกิจกรรมการเรียนการสอนประจำบท:** ใช้การจัดรูปแบบรายการเป็นลำดับตัวเลข (`1.`, `2.`)
* **สื่อการเรียนการสอน:** ใช้การจัดรูปแบบรายการเป็นลำดับตัวเลข (`1.`, `2.`)
* **การวัดผลและการประเมินผล:** แบ่งออกเป็น 2 หัวข้อหลักคือ:
  * **1. การวัดผล** ใช้รูปแบบตัวเลขทศนิยม (`1.1`, `1.2`, ...)
  * **2. การประเมินผล** ใช้รูปแบบตัวเลขทศนิยม (`2.1`, `2.2`, ...)

### 3.2 เนื้อหาการเรียนการสอน (`02-content.md`)
* **Heading 2 (`##`):** จะต้องรันหมายเลขหัวข้อตามเลขบทเรียน (เช่น ในบทที่ X จะเป็น `## X.1 หัวข้อ`, `## X.2 หัวข้อ`)
* **เนื้อหาและตัวอย่างโค้ด:** จะต้องอ้างอิงและอัปเดตอิงตาม **Expo SDK 54** (เช่น การใช้งาน Expo Router v6, TypeScript แบบ Strict, คอมโพเนนต์ `expo-image`, ฟังก์ชันจาก React 19)

### 3.3 แบบทดสอบท้ายบทเรียน (`03-quiz.md`)
* มีจำนวนคำถามปรนัยอย่างน้อย **10 ข้อ**
* คำถามและตัวเลือก ก, ข, ค, ง ต้องพิมพ์เป็นภาษาไทย
* ส่วนท้ายของบทความจะต้องมี **เฉลยคำตอบ** พร้อมคำอธิบายอ้างอิงเหตุผลประกอบอย่างละเอียดของทุกตัวเลือก

### 3.4 ปฏิบัติการและแบบฝึกหัดทักษะ (`04-lab.md`)
* เป็นคู่มือแบบเป็นขั้นตอน (Step-by-Step Tutorial) ในการสร้างแอปพลิเคชันหรือเพิ่มฟังก์ชันแบบเห็นผลลัพธ์จริง
* ต้องแนบโค้ดโปรเจคฉบับสมบูรณ์ (Complete Solution) เพื่อให้ผู้เรียนใช้สำหรับตรวจสอบความถูกต้องหรือใช้อ้างอิงขณะเรียนรู้
