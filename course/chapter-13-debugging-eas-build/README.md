# บทที่ 13: การดีบั๊ก การสร้างแอปพลิเคชัน และการเผยแพร่ด้วย EAS Build (Overview)

ในบทเรียนสุดท้ายนี้ ผู้เรียนจะได้ศึกษาขั้นตอนและทักษะที่สำคัญในการปิดงานพัฒนาแอปพลิเคชันเพื่อเตรียมนำส่งมอบให้กลุ่มเป้าหมายใช้งานจริง โดยครอบคลุมตั้งแต่การสแกนค้นหาสาเหตุการเกิดข้อพังบกพร่องในโค้ดดิ้งจาวาสคริปต์ด้วยตัวดีบั๊กเกอร์ (Chrome DevTools), แนวคิดการใช้เครื่องมือประมวลผลระยะไกลบนระบบคลาวด์ **Expo Application Services (EAS Build)** เพื่อข้ามข้อจำกัดด้านฮาร์ดแวร์คอมพิวเตอร์และการติดตั้งสภาพแวดล้อมเนทีฟ, ขั้นตอนการอัปเกรดเวอร์ชันของ **Expo SDK** และการแก้ปัญหาแพ็คเกจขัดแย้งข้ามรุ่นด้วย `npx expo install --fix`, การระบุพารามิเตอร์โดเมนและเวอร์ชันใน `app.json` และ `eas.json` ตลอดจนคำสั่งประกอบสร้างเพื่อผลิตไฟล์ติดตั้ง **Android APK** และแนวทางการดำเนินงานส่งมอบแอปพลิเคชันขึ้นสู่ตลาด Google Play Store และ Apple App Store

## 📂 สารบัญเอกสารประจำบทเรียน (Chapter Documents)
คุณสามารถเลือกอ่านและเข้าถึงไฟล์เอกสารต่าง ๆ ในโฟลเดอร์บทเรียนนี้ได้โดยตรง:

1. 📋 **[แผนการสอนประจำบท (01-lesson-plan.md)](./01-lesson-plan.md)**  
   หัวข้อการสอน วัตถุประสงค์เชิงพฤติกรรม กิจกรรมการจำลองล่าบั๊กด้วยเบราว์เซอร์ดีบั๊กเกอร์ สื่อประกอบการสอน และหัวข้อการประเมินวัดผล
2. 📖 **[เนื้อหาหลักการเรียนรู้ (02-content.md)](./02-content.md)**  
   เนื้อหาหลักภาคทฤษฎีอธิบายการเปิด Expo Developer Menu และดักจับ Breakpoints ใน Chrome, สถาปัตยกรรมบริการคลาวด์ของ EAS, การตั้งค่าความพร้อมระบบรุ่นเวอร์ชันและการอัปเกรดย้ายค่าย SDK พร้อมคำสั่งแก้ไข dependency ที่เข้ากัน และคู่มือการยื่นส่งขึ้น App Stores
3. 📝 **[แบบทดสอบท้ายบทเรียน (03-quiz.md)](./03-quiz.md)**  
   แบบทดสอบปรนัยทบทวนความเข้าใจ 10 ข้อ ภาษาไทยเจาะลึกคำสั่งอัปเกรดความเข้ากันได้ของแพ็คเกจ เครื่องมือดีบั๊ก สารระบบการบิลด์ของ EAS CLI และรูปแบบข้อกำหนดการอัปโหลดไฟล์ของแต่ละสโตร์ในปัจจุบัน พร้อมเฉลยและคำอธิบายโดยละเอียดครบทุกตัวเลือก
4. 💻 **[คู่มือปฏิบัติการขั้นพื้นฐาน (04-lab.md)](./04-lab.md)**  
   โจทย์ปฏิบัติการสร้างแอปพลิเคชัน **"แอปวินิจฉัยและสืบสวนบั๊กจำลอง (Diagnostics & Bug Hunting App)"** เพื่อฝึกฝนแก้ไขคอนฟิกจริง เรียนรู้วิธีขยับอัปเกรด SDK เคลียร์แคชตัว Metro, รันตัวดักข้อผิดพลาดร่วมกับ Chrome DevTools และสั่ง Build โปรเจ็กต์ขึ้น EAS Build Cloud สำเร็จได้ไฟล์ติดตั้ง APK สำหรับสมาร์ทโฟนจริง พร้อมซอร์สโค้ดและไฟล์กำหนดค่าเฉลยฉบับสมบูรณ์ (Complete Solution)

## 🛠️ สรุปโปรเจคปฏิบัติการท้ายบทเรียน (Lab Project Summary)
ในปฏิบัติการของบทสุดท้าย ผู้เรียนจะได้ฝึกฝนการเตรียมบิลด์และดีบั๊กแอปพลิเคชันแบบครบวงจร:
* **Unique Identification Registration:** ปรับแต่ง `app.json` ระบุชื่อแพ็คเกจ `com.student.securediary` และตั้งรหัส adaptiveIcon และหน้าจอ Splash ให้ถูกต้อง
* **EAS & SDK Reconciler:** เรียนรู้และฝึกฝนการสั่งใช้งาน `npx expo install --fix` เพื่อแก้ไขปัญหา library ไม่เข้ากันก่อนสั่งบิลด์ พร้อมเคลียร์แคชด้วย `-c`
* **Internal Distribution Profile Setup:** ลงคอนฟิก `eas.json` บังคับประเภทบิลด์ของ Android Preview Profile เป็น `"buildType": "apk"` เพื่อเลี่ยงการสร้าง AAB และจัดดึงตัวบิลด์ APK มาใช้นอกสโตร์ได้ง่าย
* **Interactive Bug Simulation:** หน้าอินเทอร์เฟซ Dashboard วินิจฉัยที่จำลองปุ่ม Runtime Rendering Error ให้โปรแกรมค้างชั่วคราว เพื่อให้ผู้เรียนได้ฝึกฝนแกะรอยหาบรรทัดโค้ดที่เป็นบั๊กผ่านการปัก Breakpoint ใน Chrome DevTools
* **Cloud Compilation Pipeline:** เรียนรู้ลำดับขั้นตอนการยิงคำสั่ง `eas build` ส่งสิทธิ์ Credentials ของผู้พัฒนาขึ้นไปคอมไพล์บนเซิร์ฟเวอร์คลาวด์ Expo จนสำเร็จได้ลิงก์ดาวน์โหลดและภาพ QR Code มาแสกนรันลงเครื่องจริง
