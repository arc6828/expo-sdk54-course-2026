# บทที่ 13: การดีบั๊ก การสร้างแอปพลิเคชัน และการเผยแพร่ด้วย EAS Build (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- เทคนิคการดีบั๊ก (Debugging) และตรวจสอบข้อผิดพลาดของแอปพลิเคชันใน Expo SDK 54 (Chrome Developer Tools, React Developer Tools, Expo CLI console)
- การอ่านค่า Log การวิเคราะห์ Stack Traces และการใช้คำสั่งควบคุมการดีบั๊กในโค้ดดิ้ง
- แนวคิดและสถาปัตยกรรมของบริการ Expo Application Services (EAS) สำหรับการสร้างแอปพลิเคชันบนระบบคลาวด์
- การเตรียมความพร้อมและตั้งค่าคุณสมบัติแอปพลิเคชันในไฟล์ `app.json` (เช่น Package Name, Bundle Identifier, Versioning, App Icons, Splash Screens)
- การกำหนดค่าโปรไฟล์การคอมไพล์ในไฟล์ `eas.json` (Development, Preview, และ Production profiles)
- ขั้นตอนการสั่งรันคอมไพล์แอปพลิเคชันเพื่อสร้างไฟล์ติดตั้ง APK (สำหรับ Android) หรือสิทธิ์ทดสอบจำลองผ่านระบบ EAS Build
- ขั้นตอนการอัปเกรดเวอร์ชันของ Expo SDK (Upgrading Expo SDK) และการปรับปรุงความเข้ากันได้ของไลบรารีต่าง ๆ (Dependency Reconciliation)
- แนวทางและขั้นตอนการนำส่งแอปพลิเคชันขึ้นสู่ตลาดสาธารณะ (Google Play Store & Apple App Store)

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถใช้เครื่องมือดีบั๊กเกอร์ (Chrome DevTools) ในการดักจับ Breakpoints และตรวจสอบค่าตัวแปรในแอปพลิเคชันได้ถูกต้อง
2. ผู้เรียนสามารถตั้งค่าชื่อแพ็คเกจสากล (Package Name / Bundle Identifier) และเลขรุ่นเวอร์ชันในไฟล์ `app.json` ได้ถูกต้องตามเกณฑ์ของ Store
3. ผู้เรียนสามารถจัดทำไฟล์ตั้งค่า `eas.json` เพื่อแบ่งแยกโปรไฟล์การคอมไพล์แอปประเภทต่าง ๆ ได้สำเร็จ
4. ผู้เรียนสามารถรันคำสั่งสั่งคอมไพล์แอปพลิเคชันบนคลาวด์ของ Expo (EAS Build) เพื่อสร้างไฟล์ติดตั้ง APK สำหรับ Android ได้สำเร็จ
5. ผู้เรียนสามารถทำและอธิบายขั้นตอนการอัปเกรดเวอร์ชัน Expo SDK และการปรับรุ่นไลบรารีอื่น ๆ ด้วยคำสั่ง `npx expo install --fix` ได้อย่างถูกต้อง
6. ผู้เรียนสามารถอธิบายขั้นตอนเตรียมเอกสารและบัญชีผู้พัฒนาสำหรับการส่งมอบแอปขึ้น Google Play Store และ Apple App Store ได้ถูกต้อง

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายเรื่อง Debugging, Upgrades & EAS Compilations:** บรรยายหลักการแกะหาสาเหตุบั๊กด้วยเครื่องมือดีบั๊กเกอร์ผ่าน Chrome, วิธีการวางแผนขั้นตอนการอัปเกรด SDK ของ Expo ข้ามรุ่น, และเปรียบเทียบสถาปัตยกรรม EAS Cloud Build กับการคอมไพล์แอปแบบดั้งเดิม
2. **การทำ Live Coding Demonstration หัวข้อ Config, Upgrade & EAS Android Preview Build:** ผู้สอนสาธิตการเรียกคำสั่งตรวจสอบและเคลียร์แคชระบบเมื่อมีการขยับรุ่น SDK, การปรับแต่งเลขรุ่นใน `app.json`, การสร้างไฟล์โปรไฟล์ `eas.json` และเรียกใช้คำสั่งดีทริกเกอร์ `eas build` เพื่อสร้างไฟล์ APK สำเร็จบนเว็บคลาวด์
3. **กิจกรรมตั้งชื่อรหัสสากลป้องกันซ้ำ (Bundle Identifier Challenge):** ให้ผู้เรียนฝึกฝนตรวจเช็คและระบุรหัสชื่อแพ็คเกจ เช่น `com.yourcompany.appname` ใน `app.json` และทดลองคำนวณเวอร์ชันอัปเกรดแอปแบบทศนิยมเพื่อส่งอัปเดตระบบขึ้นคลาวด์
4. **กิจกรรมล่าบั๊กและซ่อมไลบรารีข้ามรุ่น (Bug Hunting & SDK Upgrade Challenge):** ผู้สอนจัดเตรียมโค้ดเก่าที่มีพารามิเตอร์ตกค้างของ SDK รุ่นเก่า ให้ผู้เรียนทดลองใช้คำสั่ง `npx expo install --fix` เพื่ออัปเกรดและแก้ความขัดแย้งของไลบรารี (Dependency conflicts) จากนั้นเปิดแก้บั๊กด้วย Chrome Debugger ให้สำเร็จ

## สื่อการเรียนการสอน
1. เอกสารสไลด์เรื่อง "Debugging, SDK Upgrades, EAS Build Cloud Engine and Publishing to App Stores"
2. บัญชีเข้าใช้งานระบบ Expo Dev (Expo Accounts)
3. เครื่องมือระบบ CLI ของ Expo และชุดคำสั่งคอมไพล์ EAS CLI
4. ลิงก์คู่มือเอกสารอย่างเป็นทางการของ EAS Build, Google Play Console, Apple Developer Portal และคู่มือ Expo Upgrade Guide

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 ประเมินผลคะแนนการทำแบบทดสอบท้ายบทเรียน 10 ข้อ
1.2 ตรวจสอบโครงสร้างความถูกต้องของการจัดป้อนข้อมูลในไฟล์ `app.json` และ `eas.json` รวมถึงคำสั่งที่ใช้ในการแก้ไขความขัดแย้งไลบรารีในใบงานปฏิบัติการ
1.3 สังเกตความสำเร็จในการรันคำสั่ง Build และได้รับไฟล์ติดตั้ง APK ปลายทางที่มีขนาดพร้อมใช้งาน

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถสร้างโครงสร้างตั้งค่า แก้ไขไลบรารีตกหล่นในการขยับรุ่น และส่งแอปขึ้นไปคอมไพล์บนระบบคลาวด์ผ่านคำสั่ง EAS Build จนได้รับไฟล์ผลลัพธ์ติดตั้งได้สำเร็จผ่านเกณฑ์ (ผ่าน/ไม่ผ่าน)
2.3 สังเกตความระมัดระวังในการเก็บรักษา Credentials คีย์การลงทะเบียน และความถูกต้องในการตรวจสอบระบบ Log
