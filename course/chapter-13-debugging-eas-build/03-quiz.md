# แบบทดสอบท้ายบทเรียน บทที่ 13: การดีบั๊ก การสร้างแอปพลิเคชัน และการเผยแพร่ด้วย EAS Build (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความรู้และความเข้าใจเกี่ยวกับกระบวนการตรวจสอบและดีบั๊กโค้ดแอปพลิเคชัน, สถาปัตยกรรมคลาวด์คอมไพล์ของบริการ Expo Application Services (EAS), การตั้งค่าคุณสมบัติแอปพลิเคชันในไฟล์ `app.json` และการจัดการตั้งค่าโปรไฟล์ใน `eas.json` ตลอดจนกลไกการอัปเดตอัปเกรดเวอร์ชันของ Expo SDK และการนำส่งแอปขึ้นเผยแพร่บนร้านค้าสากล

---

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: หากกำลังเชื่อมต่อแอปพลิเคชันเพื่อทดสอบอยู่บนสมาร์ทโฟนเครื่องจริง และผู้สอนสั่งให้ทำการตรวจสอบหน้าจอ UI ผ่านคำสั่งใน "Expo Developer Menu" จะต้องทริกเกอร์เรียกแผงคำสั่งนี้ขึ้นมาอย่างไรบนหน้าจอโทรศัพท์?
ก. ทำการแตะหน้าจอรัว ๆ 10 ครั้งติดต่อกัน  
ข. ทำการเขย่าตัวเครื่องโทรศัพท์มือถือ (Shake Device)  
ค. กดปุ่มลดระดับเสียงของเครื่อง 3 ครั้ง  
ง. สั่งตัดสัญญาณอินเทอร์เน็ตบนตัวเครื่องชั่วคราว  

### ข้อ 2: การเข้าถึงหน้าจอดีบั๊กเกอร์สำหรับตรวจเช็คการทำงานของรหัสจาวาสคริปต์ ดักตรวจสอบตัวแปรในแท็บ Console และการปักหมุดจุดหยุดทำงาน (Breakpoints) นักพัฒนาสามารถเปิดหน้าต่างนี้ผ่านเครื่องมือใด?
ก. หน้าจอ Android Studio Logcat  
ข. Google Chrome Developer Tools (โดยกดเรียกดีบั๊กเกอร์ผ่าน CLI)  
ค. ระบบสไลด์เมนูของตัวจำลอง Simulator  
ง. แอปพลิเคชันตั้งค่าพื้นฐานของตัวเครื่องมือถือ  

### ข้อ 3: คำว่า "EAS" ในระบบบริการคลาวด์ของ Expo ย่อมาจากคำเต็มในข้อใด?
ก. Expo Application Services  
ข. Easy App Solutions  
ค. Expo Android Systems  
ง. Enterprise App Server  

### ข้อ 4: ข้อใดคือประโยชน์สูงสุดและเห็นภาพชัดเจนที่สุดในการเลือกใช้งานบริการระบบคลาวด์ EAS Build เข้ามาจัดสร้างประกอบไฟล์ติดตั้งแอปพลิเคชัน?
ก. ช่วยเร่งความเร็วในการดาวน์โหลดไฟล์รูปภาพในแอปพลิเคชันของฝั่งลูกค้า  
ข. เปิดโอกาสให้นักพัฒนาสามารถสร้างคอมไพล์ไฟล์แอปเนทีฟ (เช่น แอป iOS) ได้สำเร็จจากระยะไกลโดยไม่จำเป็นต้องติดตั้งสภาพแวดล้อม Xcode หรือมีคอมพิวเตอร์ Mac เป็นของตนเอง  
ค. ทำหน้าที่เขียนคำสั่งสืบค้น SQL-like ส่งไปยัง Supabase อัตโนมัติ  
ง. สั่งปิดสัญญาณสัญญานเครือข่ายมือถือเพื่อประหยัดหน่วยความจำ RAM  

### ข้อ 5: แฟ้มข้อมูลใดในโฟลเดอร์โครงการ Expo ที่ทำหน้าที่บรรจุรายละเอียดระบุตัวตน ชื่อแอป ไอคอนแอป หน้า Splash Screen และรหัสแพ็คเกจปลายทางของตัวแอปพลิเคชัน?
ก. `eas.json`  
ข. `package.json`  
ค. `app.json`  
ง. `babel.config.js`  

### ข้อ 6: การกรอกฟิลด์ระบุตัวตนสากลของแอปใน `app.json` เช่น `android.package` หรือ `ios.bundleIdentifier` ข้อใดคือรูปแบบการเขียนชื่อที่ถูกต้องและเป็นมาตรฐานยอมรับในทุกสโตร์?
ก. เขียนข้อความสตริงเพลน ๆ ตัวพิมพ์ใหญ่ เช่น `MY_DIARY_APP`  
ข. เขียนเป็นรหัสพิกัดพิกสเปซ เช่น `/app/mydiary`  
ค. เขียนโดยใช้รูปแบบโดเมนกลับหัว (Reverse Domain Name Notation) เช่น `com.yourcompany.appname`  
ง. เขียนเป็นตัวเลขแสดงลำดับไอดีโครงการ เช่น `123.456.789`  

### ข้อ 7: ในกรณีที่ต้องการเปลี่ยนประเภทของการคอมไพล์สำหรับฝั่ง Android ให้เปลี่ยนจากไฟล์ App Bundle มาเป็นไฟล์ดาวน์โหลดติดตั้งได้โดยตรง (APK File) เพื่อทดสอบภายในองค์กร จะต้องระบุค่าออปชันใดในโปรไฟล์ Preview ของไฟล์ `eas.json`?
ก. `"buildType": "apk"`  
ข. `"android": { "format": "apk" }`  
ค. `"android": { "buildType": "apk" }`  
ง. `"distribution": "apk"`  

### ข้อ 8: ชุดไฟล์กำหนดค่าการทำงานใน `eas.json` ทำหน้าที่จัดแจงแบ่งแยกกลุ่มระดับประเภทการ Build เพื่ออะไร?
ก. เพื่อจัดกลุ่มหน้าจอ Route ใน Expo Router  
ข. เพื่อแยกคุณลักษณะโปรไฟล์การสร้างแอปพลิเคชัน เช่น สำหรับพัฒนา (Development), สำหรับทดสอบภายใน (Preview) หรือสำหรับจัดส่งสโตร์จริง (Production)  
ค. เพื่อจำกัดสิทธิ์ความปลอดภัยการขออนุญาตกล้องและพิกัดแผนที่  
ง. เพื่อเก็บรักษาคีย์ anon key และ URL ของฐานข้อมูล  

### ข้อ 9: ภายหลังจากที่เราทำการสั่งอัปเกรดรหัสแกนหลักของ Expo ในไคลเอนต์สำเร็จ (เช่น ขยับจาก SDK 53 เป็น SDK 54) ข้อใดคือชุดคำสั่งคำสั่งวิเศษของ Expo CLI ที่ต้องรันสั่งการเพื่อแก้ปัญหาความเข้ากันได้ของรุ่นไลบรารีอื่น ๆ (Dependency Reconciliation) ป้องกันแอปพัง?
ก. `npm update --force`  
ข. `npx expo install --fix`  
ค. `npx expo doctor`  
ง. `npx expo upgrade`  

### ข้อ 10: เมื่อนักพัฒนาเตรียมส่งมอบและอัปโหลดแอปพลิเคชันขึ้นร้านค้า Google Play Store เพื่อเผยแพร่เชิงพาณิชย์เป็นไฟล์รุ่นผลิต (Production) รูปแบบนามสกุลไฟล์แอปประเภทใดที่ Google Play Console บังคับให้จัดส่งอัปโหลดขึ้นสโตร์ในปัจจุบัน?
ก. `.apk` (Android Package)  
ข. `.exe` (Executable)  
ค. `.aab` (Android App Bundle)  
ง. `.ipa` (iOS App Store Package)

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** วิธีเหล่านี้ไม่ใช่สัญญาณกระตุ้นการเปิดหน้าต่าง Dev Menu ของตัวเครื่องเนทีฟ
  * **ข. ถูก:** การเขย่าเครื่องจริง (Shake Device) เป็นกลไกระดับฮาร์ดแวร์เพื่อสั่งเด้งตัวแผงควบคุม Expo Developer Menu ขึ้นมาให้กดเลือกเปิดฟังก์ชันตรวจสอบ

### เฉลย ข้อ 2: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** Logcat เก็บเพียงรายงาน Log ของระบบ ไม่สามารถปัก Breakpoint หรือใช้งานเครื่องมือ JavaScript Console ได้ และแอปพลิเคชันบราวเซอร์ไม่ควบคุมการทำงานของแอปเนทีฟตรง ๆ
  * **ข. ถูก:** Google Chrome DevTools (สามารถสั่งเปิดรันได้ผ่านคำสั่งดีบั๊กเกอร์ของ Expo CLI คีย์ลัด `j`) ทำหน้าที่เชื่อมช่องทางเข้าเว็บ JS Engine ในแอป และดึงมาสืบค้นและวาง Breakpoints ดับเบิ้ลตรวจสอบได้อย่างราบรื่น

### เฉลย ข้อ 3: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ข. ค. และ ง. ผิด:** คำสะกดอื่นระบุความหมายผิดพลาด
  * **ก. ถูก:** **EAS** ย่อมาจากคำว่า **`Expo Application Services`** ซึ่งเป็นกลุ่มบริการระบบคลาวด์แบบเช่าคลาวด์ของฝั่ง Expo (ประกอบด้วย EAS Build, EAS Submit และ EAS Update)

### เฉลย ข้อ 4: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** ไม่เกี่ยวข้องกับขนาดการดึงรูปภาพ ไม่เขียนคำสั่ง CRUD ให้ และไม่ปิดเครือข่ายเพื่อประหยัดแรม
  * **ข. ถูก:** ระบบ EAS Build จะยกภาระงานทั้งหมดไปประมวลผลบนคลาวด์เซิร์ฟเวอร์ของ Expo ทำให้นักพัฒนาที่ใช้คอมพิวเตอร์ Windows หรือ Linux ทั่วไปสามารถกดสั่งเพื่อสร้างแอปพลิเคชันระบบ iOS ออกมาเป็นไฟล์ `.ipa` เพื่อติดตั้งทดสอบได้โดยไม่ต้องซื้อเครื่อง Mac ราคาแพง

### เฉลย ข้อ 5: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `eas.json` คุมการตั้งค่าโปรไฟล์และคอมไพล์ของ EAS CLI
  * **ข. ผิด:** `package.json` เก็บข้อมูลชื่อแพ็คเกจ Node และสคริปต์รันคำสั่ง  
  * **ค. ถูก:** **`app.json`** คือไฟล์ที่เก็บข้อมูลและ Metadata ของโปรเจกต์ Expo ทั้งหมดที่จะติดตัวไปกับโครงสร้างไบนารีหลังคอมไพล์เสร็จ
  * **ง. ผิด:** babel คุมทรานสไพล์ภาษาจาวาสคริปต์

### เฉลย ข้อ 6: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** สตริงพิมพ์ใหญ่หรือพาธและไอดีตัวเลขไม่ใช่เกณฑ์ยอมรับในมาตรฐานการระบุแอปของ Google และ Apple
  * **ค. ถูก:** มาตรฐานสากลสำหรับชื่อแอปพลิเคชันเพื่อป้องกันการชนกันของชื่อ (Unique Identifier) คือเขียนในรูป Reverse Domain Name เช่น `com.ชื่อบริษัท.ชื่อแอป` เสมอ

### เฉลย ข้อ 7: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** โครงสร้าง JSON ของ `eas.json` ต้องมีคีย์ย่อยควบคุมตามแพลตฟอร์มชัดเจน คำสั่งอื่นกำหนดผิดจุดประสงค์
  * **ค. ถูก:** รูปแบบโครงสร้างใน `eas.json` ต้องกำหนดคีย์ภายในโปรไฟล์เป็น `"android": { "buildType": "apk" }` เพื่อบังคับให้เซิร์ฟเวอร์คลาวด์ของ Expo เปลี่ยนประเภทจาก App Bundle กลายมาเป็นไฟล์ติดตั้งด่วนประเภท `.apk`

### เฉลย ข้อ 8: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** ไม่เกี่ยวข้องกับ Route, ไม่ใช้ตั้งค่า Permissions บล็อกกล้อง และไม่ใช้เก็บคีย์ลับ Supabase
  * **ข. ถูก:** ไฟล์ `eas.json` ถูกดีไซน์มาเพื่อจัดสร้างและแยกแยะสภาพแวดล้อมในการคอมไพล์แอปพลิเคชันปลายทาง ว่าจะใช้เพื่อทดสอบเขียนในเครื่อง, ส่งออกไฟล์เทสให้เพื่อนร่วมงาน หรือสร้างเพื่อเตรียมโพสต์ขึ้น App Stores

### เฉลย ข้อ 9: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** ชุดคำสั่งอื่นไม่ช่วยปรับปรุงหรือแก้ไขรุ่นความปลอดภัยของ library อื่นให้ตรงกับ SDK โดยออโต้ และบางตัวใช้สำหรับตรวจสอบความขัดข้องระบบเฉย ๆ
  * **ข. ถูก:** **`npx expo install --fix`** คือคำสั่งพิเศษเฉพาะกิจของ Expo ในการตรวจเช็ค library ทุกตัวในไฟล์ `package.json` แล้วสั่งแก้ไขอัปเดตรุ่นให้สัมพันธ์และเสถียรปลอดภัยเข้าเกณฑ์รองรับของ SDK รุ่นใหม่ที่พึ่งอัปเกรดมาทันที

### เฉลย ข้อ 10: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ไฟล์ `.apk` ใช้สำหรับแจกจ่ายทดสอบภายนอกสโตร์เท่านั้น ปัจจุบันไม่รับอัปโหลดในระดับโปรดักชัน
  * **ข. ผิด:** `.exe` เป็นไฟล์โปรแกรม Windows  
  * **ค. ถูก:** ตั้งแต่ปี 2021 เป็นต้นมา Google Play Console บังคับว่าแอปพลิเคชันใหม่ทั้งหมดที่ส่งขึ้นเป็นโปรดักชันบนสโตร์จะต้องนำส่งข้อมูลในรูปแบบไฟล์ **`Android App Bundle (.aab)`** เท่านั้นเพื่อให้ขนาดแอปพลิเคชันมีขนาดเหมาะสมที่สุดเมื่อถูกโหลดลงโทรศัพท์มือถือปลายทาง
  * **ง. ผิด:** `.ipa` เป็นรูปแบบติดตั้งของฝั่ง iOS
