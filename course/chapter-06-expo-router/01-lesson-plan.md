# บทที่ 6: ระบบนำทางและสถาปัตยกรรมการเปลี่ยนหน้าจอด้วย Expo Router (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- แนวคิดระบบนำทางแบบอ้างอิงโครงสร้างไฟล์ (File-based Routing) ใน Expo Router v6 (สอดคล้องกับ SDK 54)
- การจัดลำดับโฟลเดอร์หลักภายใต้ตำแหน่ง `app/` และการสร้างไฟล์เลย์เอาต์กำหนดกรอบการนำทาง (`_layout.tsx`)
- การเขียนควบคุมแบบแผนหน้าจอประเภทการสแต็คหน้าจอซ้อนกัน (Stack Navigation)
- การสร้างและปรับแต่งแท็บนำทางส่วนล่างของจอ (Tab Navigation) พร้อมกำหนดไอคอนประจำแท็บ
- การเปลี่ยนหน้าจอแบบประกาศผ่านคอมโพเนนต์ `<Link>` และแบบประมวลคำสั่ง JavaScript ด้วยอ็อบเจกต์ `router`
- การจัดการเส้นทางแบบไดนามิก (Dynamic Routes) ด้วยการตั้งชื่อไฟล์รูปเล่มวงเล็บเหลี่ยม (เช่น `app/user/[id].tsx`)
- การดึงตัวแปรและข้อมูลพารามิเตอร์ข้ามหน้าจอด้วย `useLocalSearchParams`

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถอธิบายหลักการทำงานและประโยชน์ของ File-based Routing เปรียบเทียบกับระบบจัดเส้นทางแบบดั้งเดิมได้ถูกต้อง
2. ผู้เรียนสามารถสร้างไฟล์โครงสร้าง `_layout.tsx` เพื่อแบ่งโครงสร้างหน้าจอแบบ Stack และ Tabs ได้สำเร็จ
3. ผู้เรียนสามารถเขียนเปลี่ยนเส้นทางหน้าจอโดยประยุกต์ใช้ `<Link>` และฟังก์ชันคำสั่งในกลุ่ม `router.push()` / `router.replace()` / `router.back()` ได้ถูกต้องตามสถานการณ์
4. ผู้เรียนสามารถดึงตัวแปรและคีย์ไดนามิกพารามิเตอร์ผ่าน `useLocalSearchParams` มาแสดงผลข้อมูลเจาะจงรายชิ้นสำเร็จ
5. ผู้เรียนสามารถเขียนสไตล์ส่วนหัวบาร์หน้าจอ (Screen Headers Option) เพื่อแก้ไขชื่อหัวข้อหรือปุ่มนำทางซ่อน/โชว์ได้สำเร็จ

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายอธิบายสถาปัตยกรรม (Routing Architecture Lecture):** บรรยายการจัดวางโครงสร้างของโฟลเดอร์ `app/` ในระบบ Expo Router สาธิตการส่งสัญญาณลิงก์และบทบาทของ Layout ของหน้าจอ
2. **การทำ Live Coding Demonstration หัวข้อ Multi-page Business Card App:** ผู้สอนสาธิตสดการออกแบบแอป 2 หน้าหลัก (หน้าแคตตาล็อกสินค้า และหน้ารายละเอียดสินค้าเฉพาะชิ้น) โดยเน้นเขียนส่ง ID สินค้าพ่วงไปกับเส้นทางลิงก์และดึงแสดงรูปสไตล์ไดนามิก
3. **กิจกรรมปฏิบัติกลุ่มย่อยฝึกแก้เส้นทางเลย์เอาต์ (Tabs-to-Stack Refactoring):** ให้กลุ่มผู้เรียนทดลองเพิ่มแท็บเมนูใหม่ "Settings" ในเลย์เอาต์ Tabs และเขียนปุ่มในหน้าแท็บนั้นเพื่อนำทางผู้ใช้เจาะลึกลงไปยังหน้า Stack ย่อย "Edit Profile" 
4. **กิจกรรมเขียนระบบล็อกอินนำทางจำลอง (Auth Redirect Flow Practice):** ฝึกเขียนและดักจับเหตุการณ์หากผู้ใช้ยังไม่ได้กรอกรหัส ให้เปลี่ยนหน้าจอด้วย `router.replace()` เข้าสู่หน้า Login แทนการย้อนกลับได้

## สื่อการเรียนการสอน
1. เอกสารสไลด์เรื่อง "File-based Routing and Advanced Navigation with Expo Router"
2. ตัวอย่างโครงการตั้งต้นที่มีหน้าจอเตรียมไว้แล้วสำหรับใช้ฝึกร้อยเรียงระบบนำทาง (Navigation Sandbox) บน Git
3. ลิงก์เอกสารคู่มืออย่างเป็นทางการของ Expo Router v6 (Expo SDK 54)
4. อุปกรณ์สมาร์ทโฟนตัวทดสอบเพื่อสังเกตความต่างของ Transition หน้าจอระหว่าง iOS และ Android

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 ประเมินระดับทฤษฎีจากการทำคะแนนแบบทดสอบปรนัยท้ายบทเรียน 10 ข้อ
1.2 ตรวจสอบโครงสร้างการจัดวางแผนผังไฟล์และตำแหน่งเลย์เอาต์ `_layout.tsx` ภายใต้โฟลเดอร์ `app/`
1.3 ตรวจประเมินทักษะการดึงพารามิเตอร์ข้ามหน้าด้วย `useLocalSearchParams` และการทำงานของแถบเมนูนำทางในแล็บปฏิบัติการ

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถสร้างหน้าจอแอปพลิเคชันที่มีหน้า Tabs ควบคู่หน้า Stack และส่งผ่านข้อมูลข้ามหน้าได้โดยไม่มีการแจ้งเตือนเส้นทางขาด (Broken Link) สำเร็จผ่านเกณฑ์ (ผ่าน/ไม่ผ่าน)
2.3 พฤติกรรมการทำงานร่วมกับผู้อื่น ความมีส่วนร่วมการช่วยนำทางและประคองกลุ่มปฏิบัติการแก้โจทย์
