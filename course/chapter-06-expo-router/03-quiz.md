# แบบทดสอบท้ายบทเรียน บทที่ 6: ระบบนำทางและสถาปัตยกรรมการเปลี่ยนหน้าจอด้วย Expo Router (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความเข้าใจเกี่ยวกับสถาปัตยกรรม File-based Routing หน้าที่ของไฟล์ `_layout.tsx` เลย์เอาต์แบบ Stack และ Tabs วิธีเปลี่ยนหน้าจอด้วย `<Link>` และ `router` ตลอดจนการทำระบบจัดเส้นทางไดนามิกและการอ่านค่าด้วย `useLocalSearchParams` ใน React Native Expo SDK 54

---

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: ข้อใดคือคำอธิบายของแนวคิดระบบการจัดเส้นทางแบบ "File-based Routing" ใน Expo Router ได้ถูกต้องที่สุด?
ก. เส้นทางการเชื่อมต่อจะตั้งชื่อตามฟังก์ชันภายในของ Database  
ข. โครงสร้างไฟล์และโฟลเดอร์ภายในโฟลเดอร์ `app/` จะถูกนำมาใช้ระบุขอบเขตเส้นทางการนำหน้าจอของแอปพลิเคชันโดยอัตโนมัติ  
ค. ต้องทำรายการประกาศเชื่อมพาธในไฟล์ CSS เสมอ  
ง. แอปพลิเคชันจะแปลงหน้าทั้งหมดให้ออกมารันบนเว็บบราวเซอร์ผ่านชื่อไฟล์เท่านั้น  

### ข้อ 2: โฟลเดอร์ใดที่เป็นตำแหน่งรากหลัก (Root Directory) สำหรับการตั้งวางและตรวจจับไฟล์เส้นทางหน้าจอของระบบ Expo Router?
ก. `src/`  
ข. `components/`  
ค. `app/`  
ง. `assets/`  

### ข้อ 3: ไฟล์พิเศษที่ใช้ชื่อว่า `_layout.tsx` มีหน้าที่และความสำคัญหลักอย่างไรในโฟลเดอร์เส้นทางนำทาง?
ก. คอยจัดเตรียมสไตล์สีและฟอนต์ภายนอกทั้งหมดโดยไม่เกี่ยวกับหน้าจอ  
ข. ใช้ระบุกรุปเลย์เอาต์การนำทางร่วมกัน (เช่น Stack หรือ Tabs) ให้แก่กลุ่มไฟล์ในโฟลเดอร์ระดับเดียวกันและโฟลเดอร์ย่อย  
ค. เป็นคลังเก็บไฟล์รูปภาพไอคอนของโปรเจกต์  
ง. ล้างค่าย้อนหลังข้อมูลสเตตเมื่อผู้ใช้งานปิดเปิดเครื่องใหม่  

### ข้อ 4: ในระบบนำทางแบบ Stack Navigation พฤติกรรมแบบดั้งเดิมในการจัดการสแต็คหน้าจอเป็นอย่างไร?
ก. หน้าจอจะสลับเรียงแนวนอนแบบข้างชนข้างโดยไม่จัดหน้าทับซ้อนกัน  
ข. แสดงผลหน้าจอใหม่ซ้อนทับขึ้นมาอยู่ด้านบนสุด (Push) และดึงหน้าจอนั้นออกไปเพื่อกลับสู่หน้าด้านล่าง (Pop)  
ค. นำทางผู้ใช้ไปยังเบราว์เซอร์ภายนอกเพื่อดาวน์โหลดรูปภาพ  
ง. ปิดหน้าจออื่นทั้งหมดทันทีและเหลือเพียงหน้าแรกสุดเสมอ  

### ข้อ 5: หากต้องการแก้ไขชื่อหัวข้อแถบส่วนหัว (Header Title) สำหรับหน้าจอเฉพาะเจาะจงที่อยู่ภายใต้เลย์เอาต์ `<Stack>` ควรเขียนอย่างไร?
ก. กำหนดใน Property ชื่อ `title` ของสไตล์ชีท  
ข. เรียกใช้งานคอมโพเนนต์ `<Stack.Screen options={{ title: 'ชื่อหัวข้อที่ต้องการ' }} />` ภายในไฟล์หน้าจอนั้น  
ค. ตั้งชื่อไฟล์ด้วยอักขระตัวพิมพ์ใหญ่ทั้งหมด  
ง. ส่งค่าตัวหนังสือไปป้อนที่ตัวแปรของ `useLocalSearchParams`  

### ข้อ 6: การระบุพร็อพส์ `asChild` ในคอมโพเนนต์ `<Link>` (เช่น `<Link href="/profile" asChild>`) ส่งผลอย่างไรต่อคอมโพเนนต์ลูกด้านใน?
ก. สั่งให้ล้างประวัติการนำทางทั้งหมดทิ้งและจำลองเป็นหน้าแรกสุด  
ข. ทำหน้าที่คอยจำลองการทำงานส่งต่อเหตุการณ์กดสัมผัส (onPress) และคุณสมบัติการนำทางไปควบคุมชิ้นส่วนของคอมโพเนนต์ลูกแทนการสร้างแท็ก `<a>` ครอบ  
ค. บังคับเปลี่ยนรูปภาพลูกให้ยืดหยุ่นตามขนาดของขอบจอ  
ง. สั่งเปิดลิงก์ไปยังเว็บเบราว์เซอร์ภายนอก  

### ข้อ 7: การนำทางโดยเรียกใช้งานคำสั่ง `router.replace('/home')` แตกต่างจากการเรียกใช้งาน `router.push('/home')` อย่างไร?
ก. `replace` จะผลักหน้าซ้อนสแต็คต่อประวัติเดิม แต่ `push` จะล้างข้อมูลออก  
ข. `replace` จะทำลายหน้าจอปัจจุบันและสวมทับเส้นทางใหม่ลงไปแทนที่ประวัติเดิม (ย้อนกลับมาไม่ได้) ส่วน `push` จะวางหน้าจอทับซ้อนเพิ่มขึ้นไปในสแต็คสะสม  
ค. `replace` ทำงานได้เฉพาะระบบ iOS ส่วน `push` ทำงานเฉพาะระบบ Android  
ง. `replace` ไม่รองรับการส่งผ่านค่าข้อมูลไดนามิกพารามิเตอร์  

### ข้อ 8: หากต้องการออกแบบหน้าต่างแสดงรายละเอียดไอเท็มแบบรับตัวแปรไดนามิก ID เช่น พาธ `/details/12` หรือ `/details/abc` ผู้พัฒนาต้องตั้งชื่อไฟล์อย่างไร?
ก. `details-id.tsx`  
ข. `[details].tsx`  
ค. `[id].tsx` เก็บไว้ในโฟลเดอร์ชื่อ `details`  
ง. `details.id.tsx`  

### ข้อ 9: ในหน้าจอปลายทางรับค่าเส้นทางไดนามิก เราจะดึงข้อมูลตัวแปรพารามิเตอร์ที่พ่วงส่งผ่านมาดักรับใช้งานด้วยเครื่องมือ (Hook) ตัวใด?
ก. `useState`  
ข. `useLocalSearchParams`  
ค. `useRouteOptions`  
ง. `useNavigationState`  

### ข้อ 10: เลย์เอาต์การนำทางประเภทใดที่เหมาะสำหรับการสร้างแถบเมนูปุ่มนำทางหลักยึดติดอยู่ท้ายสุดของหน้าจอ เพื่อสลับหน้าการทำงานใหญ่ ๆ ในแอป?
ก. `<Stack>`  
ข. `<Tabs>`  
ค. `<Drawer>`  
ง. `<ScrollView>`

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** การนำทางในแอปมือถือไม่เกี่ยวกับการจัดการดาต้าเบสหลังบ้านโดยตรง
  * **ข. ถูก:** File-based Routing ของ Expo Router ทำงานโดยจับคู่พิกัดไฟล์ใน `app/` มาแปลเป็น Route นำทางอัตโนมัติ ช่วยลดความซับซ้อนในการเขียนแผนผังลงทะเบียน
  * **ค. และ ง. ผิด:** CSS จัดตกแต่งสีสันเท่านั้น และแอปมือถือรันระบบนำทางระดับเนทีฟไม่ใช่รันเว็บพาธตรงตัว

### เฉลย ข้อ 2: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** เป็นโฟลเดอร์เก็บซอร์สโค้ดอื่น คอมโพเนนต์แชร์ หรือภาพประกอบที่ไม่ใช่จุดตั้งเส้นทางนำทางหลัก
  * **ค. ถูก:** Expo Router บังคับออกแบบให้โฟลเดอร์ `app/` ที่ระดับรากโครงการเป็นตำแหน่งจับคู่เส้นทางนำทางทั้งหมด

### เฉลย ข้อ 3: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ดีไซน์ภายนอกฟอนต์จะใช้การโหลด Asset ปกติ
  * **ข. ถูก:** ไฟล์ `_layout.tsx` ใช้คุมเส้นทางร่วมกัน หากเราประกาศ Stack หรือ Tabs ไว้ในนี้ หน้าจอย่อยทั้งหมดในชั้นโฟลเดอร์เดียวกันจะทำงานอยู่ใต้สไตล์เลย์เอาต์นำทางดังกล่าว
  * **ค. และ ง. ผิด:** ไม่ใช่ที่รวมภาพ และสเตตจะล้างตามปกติหากปิดตัวแอป

### เฉลย ข้อ 4: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** การสลับเรียงแนวนอนแบบ Slide เป็นเพียง Transition ไม่ใช่หลักพฤติกรรมการจัดการสแต็ค
  * **ข. ถูก:** โครงสร้างนำทางแบบ Stack ทำงานแบบ LIFO (Last In First Out) บล็อกใหม่ซ้อนบนสุด (Push) และเคลียร์ออกด้านหลังเพื่อกลับจุดเดิม (Pop)
  * **ค. และ ง. ผิด:** ไม่ใช่คำสั่งนำออกเบราว์เซอร์ และยังมีกลไกคงหน้าเก่าค้างไว้ใต้สแต็ค

### เฉลย ข้อ 5: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** สไตล์ชีทไม่สามารถเปลี่ยนข้อความของแถบหัวเรื่องระบบเนทีฟได้
  * **ข. ถูก:** ตัวเสริม `<Stack.Screen>` ของ Expo Router เปิดช่องให้เราสั่ง Override ค่าตัวเลือก (Options) ประจำหน้าจอได้จากไฟล์ของหน้านั้น ๆ โดยตรง
  * **ค. และ ง. ผิด:** การตั้งชื่อไฟล์ด้วยตัวพิมพ์ใหญ่ไม่ส่งผลต่อชื่อหัวข้อ และพารามิเตอร์มีไว้รับส่งตัวแปรข้อมูล

### เฉลย ข้อ 6: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** ไม่เกี่ยวกับการเคลียร์สแต็ค ปรับขนาดรูป หรือเชื่อมเว็บภายนอก
  * **ข. ถูก:** พร็อพส์ `asChild` เป็นตัวช่วยส่งต่อ Event Handler `onPress` ของปุ่มลิงก์ผ่านลงไปยังปุ่ม `Pressable` หรือชิ้นงานลูกย่อยด้านใน ป้องกันการซ้อนทับกันของแท็กระบบนำทาง

### เฉลย ข้อ 7: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** สลับพฤติกรรมกัน
  * **ข. ถูก:** `router.push()` จะวางหน้าจอทับซ้อนเพิ่มในประวัติทำให้กดย้อนกลับได้ ส่วน `router.replace()` จะเปลี่ยนหน้าแบบทับแทนที่หน้าเก่าทันที เหมาะสำหรับหน้าระบบลงทะเบียนเสร็จแล้วไม่อยากให้ย้อนกลับมาเจอหน้าฟอร์มกรอกซ้ำ
  * **ค. และ ง. ผิด:** ทั้งคู่ทำงานได้เหมือนกันทั้งสองระบบ OS และสนับสนุนการส่งพารามิเตอร์ปกติ

### เฉลย ข้อ 8: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** ชื่อรูปแบบอื่นไม่ถูกแปลงเป็นตัวแปรไดนามิกในระบบ Expo Router v6
  * **ค. ถูก:** การห่อชื่อไฟล์ด้วยสัญลักษณ์วงเล็บเหลี่ยม `[id].tsx` เป็นการระบุว่า คีย์ชื่อ `id` จะเป็นตัวดักจับสตริงข้อมูลใน URL หรือพาธเพื่อนำไปประมวลผลต่อ

### เฉลย ข้อ 9: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `useState` จัดการสถานะภายในของตัวหน้านั้น ไม่เกี่ยวข้องกับการแปลงตัวแปร URL
  * **ข. ถูก:** `useLocalSearchParams` ถูกสร้างขึ้นโดยเฉพาะใน Expo Router เพื่ออ่านค่าพารามิเตอร์ที่แนบมากับเส้นทางเดินหน้าจอปลายทาง (เช่น ดึงค่า ID สินค้า)
  * **ค. และ ง. ผิด:** ไม่ใช่ชื่อคำสั่งมาตรฐาน

### เฉลย ข้อ 10: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** Stack เหมาะสำหรับหน้าจอย่อยซ้อนลึก Drawer เหมาะสำหรับแถบปัดด้านข้าง ส่วน ScrollView ใช้สำหรับเลื่อนข้อความไม่ใช่เลย์เอาต์การเดินหน้าจอหลัก
  * **ข. ถูก:** เลย์เอาต์แบบ `<Tabs>` ออกแบบมาจัดวางแท็บเดินหน้าระดับล่าง (Bottom Tab Bar) เพื่อความลื่นไหลในการสลับใช้งานฟีเจอร์หลัก ๆ ของแอป
