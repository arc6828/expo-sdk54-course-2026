# บทที่ 11: การใช้งานกล้องและอัลบั้มภาพเนทีฟ (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- กลไกการขอสิทธิ์เข้าใช้งานกล้องถ่ายรูป (Camera Permissions) และการเข้าถึงคลังสื่อภาพถ่าย (Media Library / Photos Permissions)
- การจัดแสดงพรีวิวภาพสดของกล้องด้วยคอมโพเนนต์ `CameraView` และการใช้งาน Hook `useCameraPermissions` ของไลบรารี `expo-camera`
- การสลับมุมกล้องหน้าและกล้องหลัง (Camera Facing Toggle) ด้วยการควบคุมพร็อพส์ `facing` (front / back)
- การสั่งถ่ายภาพนิ่งแบบ Asynchronous และรับข้อมูลที่อยู่ไฟล์ชั่วคราวผ่านคำสั่ง `takePictureAsync`
- การเลือกรูปภาพในอัลบั้มมือถือมาแสดงในแอปพลิเคชันด้วยคำสั่ง `launchImageLibraryAsync` ของไลบรารี `expo-image-picker`
- การจัดการระบบไฟล์ (File System) 3 รูปแบบในแอปพลิเคชันมือถือ:
  1. แบบแอปมองเห็นเฉพาะตัว (Private Sandbox Storage) ด้วย `expo-file-system` (`cacheDirectory`, `documentDirectory`)
  2. แบบแอปอื่นสามารถเข้าถึงข้อมูลร่วมกันได้ (Shared Media Storage) ด้วย `expo-media-library` (บันทึกรูปภาพลงคลังภาพสาธารณะของระบบเครื่อง)
  3. แบบจัดเก็บบนเครือข่ายคลาวด์ (Cloud Storage Network) ด้วยการอัปโหลดและดาวน์โหลดไฟล์ผ่านบริการ Supabase Storage Buckets
- การตั้งค่าปรับขนาด ย่อรูปภาพ และครอปจัดสัดส่วน (Image Editing & Aspect Ratio Options)

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถเขียนโครงสร้างขออนุญาตสิทธิ์เข้าถึงกล้อง คลังภาพ และการบันทึกสื่อลงเครื่องตามมาตรฐานความปลอดภัยได้ถูกต้อง
2. ผู้เรียนสามารถประยุกต์ใช้คอมโพเนนต์ `CameraView` ในการเรนเดอร์หน้าจอเลนส์กล้องพรีวิวสดได้สำเร็จ
3. ผู้เรียนสามารถเขียนปุ่มคำสั่งควบคุมการถ่ายรูปและรับค่าพาธรูปภาพชั่วคราว (Image URI) มาใช้งานได้ถูกต้อง
4. ผู้เรียนสามารถดึงภาพถ่ายจากอัลบั้มเครื่องผู้ใช้ขึ้นมาแสดงผลเป็นอวาตาร์กลมด้วย `expo-image-picker` ได้สำเร็จ
5. ผู้เรียนสามารถอธิบายความแตกต่างและเปรียบเทียบการใช้งาน Private Sandbox (`expo-file-system`) กับ Shared Gallery (`expo-media-library`) ได้อย่างถูกต้อง
6. ผู้เรียนสามารถเรียกใช้ `expo-file-system` คัดลอกรูปภาพมาเก็บไว้ในพื้นที่โฟลเดอร์ถาวรของแอปเพื่อป้องกันไฟล์หายได้สำเร็จ
7. ผู้เรียนสามารถบันทึกภาพถ่ายเนทีฟลงคลังรูปภาพส่วนกลางของระบบเครื่องด้วย `expo-media-library` เพื่อให้แอปพลิเคชันอื่นเข้าถึงรูปได้สำเร็จ
8. ผู้เรียนสามารถส่งอัปโหลดไฟล์ภาพโปรไฟล์ขึ้นสู่ระบบคลาวด์ผ่าน Supabase Storage และดึง URL สาธารณะมาใช้แสดงผลในแอปพลิเคชันได้สำเร็จ

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายการเชื่อมโยงระบบสื่อและแฟ้มข้อมูล 3 ระดับ (Three-Level File Storage Lecture):** บรรยายการสร้าง View ของกล้องถ่ายรูป, โครงสร้างของ Image Picker และเจาะลึกความแตกต่างระหว่างไฟล์ในแคชชั่วคราว, สารระบบ Sandbox ของแอป, คลังภาพสาธารณะของระบบปฏิบัติการ และฐานข้อมูลจัดเก็บไฟล์รูปภาพบนคลาวด์ (Supabase Storage)
2. **การทำ Live Coding Demonstration หัวข้อ Multi-Destination Profile Image Saver:** ผู้สอนสาธิตการออกแบบระบบอวาตาร์ โดยถ่ายรูปหรือเลือกภาพจากอัลบั้ม แล้วประมวลผลเซฟพร้อมกัน 3 ช่องทาง: คัดลอกลงโฟลเดอร์ Document ส่วนตัวของแอป, บันทึกส่งออกคลังรูปภาพสาธารณะของเครื่อง และอัปโหลดไฟล์ตรงเข้าสู่ Supabase Storage Bucket
3. **กิจกรรมสลับหน้าเลนส์และสิทธิ์ความปลอดภัย (Facing & Permissions Challenge):** ให้ผู้เรียนทดลองเพิ่มระบบขอยื่นสิทธิ์ของกล้องและคลังภาพอย่างสมบูรณ์แบบ และทดลองสลับกล้องหน้า/หลังในการพรีวิว
4. **กิจกรรมฝึกฝนอัปโหลดและดาวน์โหลดผ่าน Cloud Network (Supabase Storage Challenge):** ให้กลุ่มผู้เรียนสร้างการแปลงภาพถ่ายจากพาธ URI ดิบเป็น ArrayBuffer หรือ Base64 เพื่อยิงส่งอัปโหลดเข้า Bucket และดึง URL ลิงก์ตรงมาป้อนเข้าคอมโพเนนต์แสดงผลภาพ

## สื่อการเรียนการสอน
1. เอกสารสไลด์เรื่อง "Native Camera View, Image Picker and Three-Level Storage Systems in React Native"
2. เครื่องสมาร์ทโฟนจริงสำหรับเสียบทดสอบการทำงานของกล้อง แกลเลอรี และการบันทึกรูปภาพจริงลงดิสก์
3. ข้อมูลบัญชีผู้ใช้บริการ Supabase สำหรับการสร้าง Bucket และกำหนดนโยบาย RLS ของ Storage
4. ลิงก์เอกสารอ้างอิงของ API `expo-camera`, `expo-image-picker`, `expo-file-system`, `expo-media-library` และ Supabase JS Client

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 ประเมินผลคะแนนการทำแบบทดสอบท้ายบทเรียน 10 ข้อ
1.2 ตรวจสอบโครงสร้างความถูกต้องของการอนุญาตสิทธิ์กล้อง อัลบั้มภาพ และการจัดการเขียนไฟล์ลงในระบบเอกสาร
1.3 สังเกตขั้นตอนการส่งอัปโหลดไฟล์ขึ้นระบบคลาวด์และเรียกคืนภาพผ่านโครงข่ายเครือข่าย

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถสร้างโมดูลเปลี่ยนรูปถ่ายโปรไฟล์ที่บันทึกข้อมูลภาพลงแซนด์บ็อกซ์ของแอป, อัลบั้มแกลเลอรีส่วนกลาง และ Supabase Storage ได้สำเร็จผ่านเกณฑ์ (ผ่าน/ไม่ผ่าน)
2.3 สังเกตทักษะความประณีตในการออกแบบหน้าจอกล้อง และความถูกต้องปลอดภัยในการเชื่อมต่อกับบริการคลาวด์
