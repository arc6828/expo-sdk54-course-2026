# แบบทดสอบท้ายบทเรียน บทที่ 11: การใช้งานกล้อง อัลบั้มภาพ และระบบจัดเก็บไฟล์เนทีฟร่วมกับคลาวด์ (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความรู้และความเข้าใจเกี่ยวกับแนวคิดระบบความปลอดภัยของภาพถ่ายเนทีฟ, พฤติกรรมการทำงานของคอมโพเนนต์ `CameraView` และ `expo-image-picker` ใน Expo SDK 54, ตลอดจนกลไกสถาปัตยกรรมการจัดเก็บรูปภาพ 3 รูปแบบหลัก ได้แก่ ระบบจัดเก็บไฟล์เฉพาะแอป (Sandbox), การส่งออกไฟล์สาธารณะข้ามแอป (Shared Gallery) และระบบจัดเก็บไฟล์ข้ามเครือข่ายบนคลาวด์ (Supabase Storage)

---

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: ข้อใดอธิบายพฤติกรรมการเข้าถึงและขออนุมัติใช้งานสื่อรูปภาพแบบเนทีฟได้ถูกต้องที่สุด?
ก. เมื่อแอปพลิเคชันได้รับสิทธิ์เปิดใช้กล้องถ่ายรูปแล้ว จะสามารถดึงประวัติรูปภาพเก่าในคลังอัลบั้มมาประมวลผลได้ทันทีโดยไม่ต้องร้องขอสิทธิ์แกลเลอรี  
ข. เนื่องจากเกี่ยวข้องกับสิทธิ์ความเป็นส่วนตัวต่างประเภทกัน ระบบจึงบังคับแยกขอสิทธิ์ระหว่างกล้องถ่ายรูป (Camera Permission) และคลังรูปภาพในอุปกรณ์ (Media Library / Photos Permission) แยกจากกันเป็นเอกเทศ  
ค. สิทธิ์ในการเข้าถึงคลังสื่อภาพนิ่งจะถูกสั่งอนุมัติให้อัตโนมัติในฝั่งเซิร์ฟเวอร์โดยที่ผู้ใช้งานมือถือไม่มีสิทธิ์ปฏิเสธ  
ง. สิทธิ์การใช้กล้องของระบบเนทีฟทำงานโดยไม่ต้องอาศัยการกำหนดค่าใด ๆ ในไฟล์ `app.json`  

### ข้อ 2: คอมโพเนนต์ใดที่ได้มาตรฐานของ Expo SDK 54 ที่แนะนำให้นักพัฒนาใช้สร้างหน้าต่างเลนส์พรีวิวภาพสด (Camera Live Preview) จากเลนส์มือถือ?
ก. `<Camera>`  
ข. `<CameraPreview>`  
ค. `<CameraView>`  
ง. `<NativeLens>`  

### ข้อ 3: การตั้งค่าสลับการใช้หน้าเลนส์พรีวิวสดของกล้องถ่ายรูปสลับไปมาระหว่าง "กล้องหน้า" และ "กล้องหลัง" บนคอมโพเนนต์กล้องรุ่นใหม่ ต้องควบคุมค่าที่พร็อพส์ (Prop) และส่งค่าใด?
ก. พร็อพส์ `type` รับค่า `front` หรือ `back`  
ข. พร็อพส์ `facing` รับค่า `front` หรือ `back`  
ค. พร็อพส์ `lens` รับค่า `selfie` หรือ `rear`  
ง. พร็อพส์ `facing` รับค่า `user` หรือ `environment`  

### ข้อ 4: พฤติกรรมการกักเก็บรูปภาพเริ่มต้นของผลลัพธ์ที่ได้จากการยิงคำสั่งถ่ายภาพ `takePictureAsync()` หรือเลือกภาพจาก `launchImageLibraryAsync()` ข้อใดกล่าวได้ถูกต้อง?
ก. ข้อมูลไบนารีของภาพถ่ายทั้งหมดจะถูกเขียนพ่วงบันทึกลงฐานข้อมูลหลักของเครื่องโดยทันที  
ข. ผลลัพธ์จะถูกจัดเก็บเป็นที่อยู่พาธภาพถ่ายชั่วคราวชี้ไปที่โฟลเดอร์แคช (`cacheDirectory`) ซึ่งระบบปฏิบัติการ (OS) มีสิทธิ์สุ่มล้างข้อมูลทิ้งได้ตลอดเวลาเมื่อดิสก์เครื่องเต็ม  
ค. ตัวไฟล์ภาพจะถูกกักเก็บไว้อยู่ในสารระบบ `documentDirectory` แบบคงทนถาวรทันทีหลังเสร็จสิ้นกระบวนการ  
ง. ไฟล์ภาพจะถูกส่งอัปโหลดผ่านระบบเครือข่ายอินเทอร์เน็ตขึ้นคลาวด์โดยทันทีโดยไม่ต้องรอเขียนโปรแกรมเชื่อมต่อ  

### ข้อ 5: แฟ้มไดเรกทอรีระบบใดภายใน Sandbox ของแอปพลิเคชันที่มีคุณสมบัติเก็บรักษาทรัพยากรได้อย่างถาวรปลอดภัย ไม่มีวันโดนระบบปฏิบัติการลบทิ้ง ยกเว้นผู้ใช้จะสั่งถอนการติดตั้งแอป?
ก. `FileSystem.cacheDirectory`  
ข. `FileSystem.temporaryDirectory`  
ค. `FileSystem.documentDirectory`  
ง. `FileSystem.sharedDirectory`  

### ข้อ 6: หากต้องการพัฒนาแอปพลิเคชันถ่ายรูปแล้วสั่งบันทึกส่งออกไฟล์ภาพไปเก็บไว้ในอัลบั้มหลักของมือถือ เพื่อให้แอปพลิเคชันอื่นนอกตัวแอป (เช่น Facebook, IG) สามารถดึงรูปนี้ไปใช้ได้ด้วย จะต้องบูรณาการผ่านคำสั่งของไลบรารีใด?
ก. `expo-file-system`  
ข. `expo-media-library`  
ค. `expo-image-picker`  
ง. `expo-sharing`  

### ข้อ 7: คำสั่งใดของไลบรารี `expo-media-library` ที่ใช้ระบุตัวเลือกสั่งบันทึกไฟล์ภาพเนทีฟตรงเข้าสู่แกลเลอรีแชร์ข้ามแอปพลิเคชัน?
ก. `MediaLibrary.saveToLibraryAsync()` หรือ `MediaLibrary.createAssetAsync()`  
ข. `MediaLibrary.uploadImageAsync()`  
ค. `MediaLibrary.copyToSharedSpace()`  
ง. `MediaLibrary.exportPhoto()`  

### ข้อ 8: การอัปโหลดไฟล์รูปภาพจากฝั่ง React Native ขึ้นสู่ระบบฝากไฟล์ Supabase Storage Buckets วิธีการส่งชุดข้อมูลวิธีใดที่ทำงานได้เสถียรที่สุดเพื่อหลีกเลี่ยงข้อจำกัดการอ่านไฟล์แบบ Blob เนทีฟ?
ก. การดึงที่อยู่พาธ URI ไปยิงคำสั่งอัปโหลดผ่าน REST API ตรง ๆ โดยไม่ผ่าน SDK  
ข. การอ่านไฟล์ข้อมูลออกมาเป็นรหัสสตริง Base64 แล้วทำการแปลงถอดรหัสออกมาเป็นแบบ `ArrayBuffer` ไบนารี  
ค. การดึงพิกเซลสีทั้งหมดมาเรนเดอร์ในคอมโพเนนต์ Canvas แล้วเก็บรหัส JSON  
ง. การอัปโหลดเฉพาะชื่อไฟล์ขึ้นคลาวด์ แล้วให้ฝั่งคลาวด์ดาวน์โหลดไฟล์จากมือถือกลับไปเอง  

### ข้อ 9: ในกระบวนการอ่านไฟล์และแปลงรหัส Base64 จากพื้นที่จัดเก็บเครื่องฝั่ง Client ของโทรศัพท์มือถือเป็น Asynchronous จะต้องสั่งการเรียกใช้โมดูลใดเป็นแกนหลัก?
ก. `FileSystem.readAsStringAsync()` ร่วมกับออปชัน `{ encoding: FileSystem.EncodingType.Base64 }`  
ข. `ImagePicker.readAsBase64Async()`  
ค. `AsyncStorage.getItem()` ร่วมกับคีย์ไฟล์  
ง. `MediaLibrary.readBytesAsync()`  

### ข้อ 10: หลังจากที่ทำการส่งอัปโหลดไฟล์เข้าไปใน Supabase Storage Bucket เป็นที่เรียบร้อย หากต้องการลิงก์ตรงภายนอก (Direct URL) สำหรับนำมาเรนเดอร์แสดงผลในแอปทันที จะต้องเรียกใช้เมธอดสืบค้นใดของ Supabase Client?
ก. `supabase.storage.from('bucket').getPublicUrl(filePath)`  
ข. `supabase.storage.from('bucket').download(filePath)`  
ค. `supabase.storage.from('bucket').select('url')`  
ง. `supabase.storage.from('bucket').retrieveLink(filePath)`

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** การอนุญาตสิทธิ์กล้อง ไม่ได้ครอบคลุมการสืบค้นภาพเก่าในคลังอัลบั้ม ผู้ใช้ต้องเลือกยินยอมสิทธิ์แยกส่วนกัน
  * **ข. ถูก:** ระบบความปลอดภัยของ iOS/Android แยกสิทธิ์การใช้เลนส์สัมผัสกล้อง (Camera) และการอ่านคลังไฟล์ส่วนตัวในอัลบั้มเครื่อง (Photos / Media Library) ออกจากกันอย่างเด็ดขาดเพื่อความเป็นส่วนตัวสูงสุดของผู้ใช้
  * **ค. และ ง. ผิด:** ระบบสิทธิ์ถูกควบคุมโดย OS ฝั่งมือถือ ไม่ใช่ฝั่งเซิร์ฟเวอร์ และจำเป็นต้องมีการลงทะเบียนไว้ในไฟล์ `app.json` เสมอ

### เฉลย ข้อ 2: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** ไม่มีส่วนคอมโพเนนต์ชื่อเหล่านี้จัดเตรียมไว้ใน API ของ Expo SDK 54
  * **ค. ถูก:** Expo ได้ยุบและปรับปรุงคอมโพเนนต์การพรีวิวเลนส์กล้องตัวเก่า (`Camera`) และเปิดตัวคอมโพเนนต์ **`CameraView`** ขึ้นมาขับเคลื่อนแทนในแพ็คเกจ `expo-camera`

### เฉลย ข้อ 3: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** พร็อพส์ `type` ถูกเปลี่ยนชื่อออกไปแล้วใน API รุ่นใหม่
  * **ข. ถูก:** พร็อพส์ที่ระบุหน้าเลนส์คือ **`facing`** โดยยอมรับค่าสตริงสองชนิดคือ `'front'` (เซลฟี่) หรือ `'back'` (กล้องหลัง) เท่านั้น
  * **ค. และ ง. ผิด:** ค่าโหมดหรือเลนส์เหล่านี้ไม่เข้าเกณฑ์พารามิเตอร์ของ Expo Camera API

### เฉลย ข้อ 4: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** จะไม่มีการเขียนหรือสร้างข้อมูลลงในฐานข้อมูลหลักให้อัตโนมัติ
  * **ข. ถูก:** ทั้งภาพที่ถ่ายสดหรือดึงจาก Picker จะได้รับการสร้างไว้ในห้องแคชชั่วคราว (`cacheDirectory`) ซึ่งมีความเสี่ยงสูงที่จะโดนกวาดล้างลบไฟล์ทิ้งโดยระบบ OS เมื่อดิสก์มีความจุน้อยลงหรือแอปพลิเคชันปิดการทำงาน
  * **ค. และ ง. ผิด:** ไม่ได้เข้าสารระบบ Document อัตโนมัติ และไม่สามารถวิ่งส่งขึ้นระบบคลาวด์ได้เองโดยไม่มีอินเทอร์เฟซคำสั่งอัปโหลด

### เฉลย ข้อ 5: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. และ ข. ผิด:** โฟลเดอร์ห้อง Cache หรือ Temporary มีสิทธิ์โดนลบขยะได้ตลอดเวลาโดย OS
  * **ค. ถูก:** แฟ้มเอกสาร **`FileSystem.documentDirectory`** คือพื้นที่กักเก็บไฟล์ส่วนตัวแบบปลอดภัยภายใน Sandbox ของตัวแอปพลิเคชัน ซึ่งปลอดภัยจากการลบกวาดล้างของ OS เสมอ
  * **ง. ผิด:** ไม่มีไดเรกทอรีชื่อนี้

### เฉลย ข้อ 6: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `expo-file-system` ใช้จัดเก็บไฟล์ภายในแซนด์บ็อกซ์ส่วนตัวของตัวแอปเองเป็นหลัก
  * **ข. ถูก:** **`expo-media-library`** คือเครื่องมือและไลบรารีที่เปิดทางให้แอปพลิเคชันเข้าถึง จัดการ และเซฟสื่อบันทึกลงสู่โฟลเดอร์ภาพสาธารณะ (Shared Storage) เพื่อแชร์แชร์ข้ามแอปพลิเคชันเครื่องได้สำเร็จ
  * **ค. และ ง. ผิด:** `expo-image-picker` ใช้เปิดเลือกรูป ไม่ใช่การเซฟบันทึกคลังภาพ ส่วน `expo-sharing` ใช้ยิงหน้าต่างระบบแชร์ไฟล์ระบบ (Share Sheet) ไม่ได้จัดเก็บถาวรในแกลเลอรี

### เฉลย ข้อ 7: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** ฟังก์ชันที่รับหน้าที่จัดสร้าง asset รูปภาพและนำเข้าคลังรูปสาธารณะคือ `MediaLibrary.createAssetAsync()` หรือสั่งบันทึกไฟล์ภาพเนทีฟตรงด้วย `MediaLibrary.saveToLibraryAsync()`
  * **ข. ค. และ ง. ผิด:** ชื่อเมธอดไม่มีระบุไว้ในสารระบบของคลังคำสั่ง `expo-media-library`

### เฉลย ข้อ 8: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** การทำ Blob ใน React Native เจอปัญหาความเข้ากันได้ และการส่งพิกเซลสีหรือส่งเพียงชื่อไฟล์ไม่สามารถย้ายไฟล์จริงขึ้นคลาวด์ได้
  * **ข. ถูก:** การสกัดดึงข้อมูลออกมาเป็นสตริงความปลอดภัยรหัส Base64 ด้วย `expo-file-system` แล้วใช้ฟังก์ชันถอดค่าออกมาเป็นไบนารีแบบ **`ArrayBuffer`** ถือเป็นรูปแบบมาตรฐานที่เสถียรที่สุดในการยิงรีเควสผ่าน `@supabase/supabase-js`

### เฉลย ข้อ 9: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** โมดูล Asynchronous ของระบบไฟล์ของ Expo คือ `FileSystem.readAsStringAsync` ซึ่งเราต้องกำหนดคีย์ออปชันรหัสผ่านเป็น `{ encoding: FileSystem.EncodingType.Base64 }`
  * **ข. ค. และ ง. ผิด:** คำสั่งอื่นไม่มีอยู่จริง หรือใช้เก็บบันทึกความสัมพันธ์ตัวเลขธรรมดาอย่าง AsyncStorage ไม่สามารถแปลงไฟล์ไบนารีได้

### เฉลย ข้อ 10: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** เมธอดสืบค้นลิงก์ตรงคือ `.getPublicUrl(filePath)` ซึ่งจะส่งอ็อบเจกต์ที่มีแอตทริบิวต์ระบุพาธ `.data.publicUrl` กลับออกมาให้ใช้ส่งเข้า Image Component เพื่อเรนเดอร์ภาพจากอินเทอร์เน็ตได้ทันที
  * **ข. ผิด:** `.download()` คุมการรับข้อมูลไบนารีดิบลงหน่วยความจำ ไม่ใช่ขอลิงก์ URL
  * **ค. และ ง. ผิด:** โครงสร้างคำสั่งเหล่านี้ระบุพารามิเตอร์และ API ชื่อไม่ถูกต้อง
