# บทที่ 9: พิกัดตำแหน่ง แผนที่ และเซนเซอร์อุปกรณ์ (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- กลไกการขอสิทธิ์เข้าถึงตำแหน่งตัวเครื่อง (Location Permissions Model) ในระบบ iOS และ Android
- การเข้าถึงพิกัดละติจูดและลองจิจูด (Latitude, Longitude) ล่าสุดด้วยไลบรารี `expo-location`
- การทำภูมิสารสนเทศย้อนกลับ (Reverse Geocoding) เพื่อแปลงรหัสพิกัดให้เป็นรายละเอียดที่อยู่จริง (ถนน, จังหวัด, รหัสไปรษณีย์)
- การติดตั้งและจัดแสดงภาพแผนที่แบบโต้ตอบได้ด้วยไลบรารี `react-native-maps`
- การควบคุมศูนย์กลางกล้องแผนที่ (Map Region) และการปักหมุดจุดพิกัดด้วยคอมโพเนนต์ `Marker`
- การดักตรวจจับเซนเซอร์จับความเร่งการเคลื่อนไหว (Accelerometer) ด้วยไลบรารี `expo-sensors`
- แนวทางการเขียนโปรแกรมควบคุมขอบเขตกรณีผู้ใช้ปฏิเสธการให้สิทธิ์ตำแหน่ง (Location Permission Denied Handler)

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถระบุความแตกต่างระหว่าง Foreground และ Background Location Permissions ได้ถูกต้อง
2. ผู้เรียนสามารถใช้คำสั่งดึงค่าพิกัดปัจจุบันและแปลงพิกัดกลับมาเป็นที่อยู่ภาษาไทยผ่าน `expo-location` ได้สำเร็จ
3. ผู้เรียนสามารถนำเสนอแผนที่พิกัดดาวเทียมผ่านคอมโพเนนต์ `MapView` ในหน้าจอแอปพลิเคชันได้สำเร็จ
4. ผู้เรียนสามารถเขียนโค้ดปักหมุดแสดงตำแหน่ง (Marker Pin) และระบุขอบข่ายละติจูดลองจิจูดได้ถูกต้อง
5. ผู้เรียนสามารถผูกรับสัญญาณข้อมูลจากเซนเซอร์ตรวจความเร่ง `Accelerometer` ร่วมกับระบบควบคุมแผนที่ได้ถูกต้อง

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายสถาปัตยกรรมแผนที่และภูมิศาสตร์ (Location Services & Maps Architecture Lecture):** บรรยายการทำงานร่วมกับดาวเทียม (GPS), โครงสร้างจัดระเบียบสิทธิ์ตำแหน่ง และการนำคอมโพเนนต์ `react-native-maps` มาเรนเดอร์ในอุปกรณ์พกพา
2. **การทำ Live Coding Demonstration หัวข้อ Current Location on Interactive Map:** ผู้สอนสาธิตการสร้างหน้าจอแอปแสดงพิกัดแบบเรียลไทม์ โดยดึงค่าตำแหน่งเครื่องปัจจุบันมาเซ็ตค่าศูนย์กลางของ `MapView` และปักหมุดเป็นสัญลักษณ์รูปบ้าน
3. **กิจกรรมแกะที่อยู่ออกจากพิกัด (Geocoding Coordinates Challenge):** ให้ผู้เรียนทดลองพิมพ์พิกัดละติจูดลองจิจูดจำลอง และเขียนคำสั่งแปลงค่ากลับเป็นรายละเอียดตำบล อำเภอ จังหวัด เพื่อแสดงขึ้นบนจอแผนที่ย่อย
4. **กิจกรรมเขย่าแผนที่ย้ายพิกัด (Shake to Center Device Sensor Activity):** ให้กลุ่มผู้เรียนช่วยกันใช้ความสามารถของ `expo-sensors` (Accelerometer) ในการจับจังหวะสั่นสะเทือนเครื่อง เพื่อบังคับให้กล้องแผนที่ขยับศูนย์กลางวิ่งกลับมาที่ตำแหน่งพิกัดเครื่องปัจจุบันของผู้ใช้โดยอัตโนมัติ

## สื่อการเรียนการสอน
1. เอกสารสไลด์เรื่อง "Geolocation, Map Rendering, and Device Sensors in React Native"
2. อุปกรณ์มือถือสำหรับทดสอบเปิด GPS และเซนเซอร์ความเคลื่อนไหว (หรือตั้งค่า Mock Coordinates ใน Simulator)
3. ตัวอย่างการเซ็ตค่าสิทธิ์ใน `app.json` สำหรับเปิดบริการ Location
4. ชุดรหัสพิกัดจำลองสถานที่ท่องเที่ยวสำคัญเพื่อใช้ฝึกฝนการปักหมุด

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 ประเมินผลคะแนนการทำแบบทดสอบท้ายบทเรียน 10 ข้อ
1.2 ตรวจสอบโครงสร้างความถูกต้องของการเรียกขอสิทธิ์ Location และการกำหนดสเตตัสในโค้ดแผนที่
1.3 สังเกตขั้นตอนการคืนค่า (Unsubscribe) เซนเซอร์ Accelerometer ตอนปิดหน้าจอเพื่อความปลอดภัยไม่ให้แรมรั่ว

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถสร้างหน้าจอแผนที่ที่ปักหมุดพิกัดปัจจุบันและแจ้งที่อยู่ถนนจริงของผู้ใช้ได้อย่างราบรื่นผ่านเกณฑ์ (ผ่าน/ไม่ผ่าน)
2.3 สังเกตพฤติกรรมความใส่ใจในการเขียนป้องกันเหตุฉุกเฉินเมื่อเครื่องไม่มีบริการ GPS หรือเมื่อผู้ใช้อนุมัติสิทธิ์ไม่สำเร็จ
