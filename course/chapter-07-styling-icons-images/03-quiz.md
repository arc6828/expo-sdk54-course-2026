# แบบทดสอบท้ายบทเรียน บทที่ 7: การตกแต่งอินเทอร์เฟซ ไอคอน และรูปภาพประสิทธิภาพสูง (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความรู้และความเข้าใจเกี่ยวกับระบบการควบคุมสไตล์ขั้นสูง (Absolute Positioning), การโหลดและใช้งานแบบอักษรพิเศษ (Custom Fonts), การประยุกต์ใช้เวกเตอร์ไอคอน (@expo/vector-icons), การเพิ่มประสิทธิภาพจัดแสดงรูปภาพ (expo-image) และการรองรับโหมดมืด (Dark Mode / useColorScheme) ใน React Native Expo SDK 54

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: ในการกำหนดสไตล์ของ React Native ข้อใดอธิบายพฤติกรรมของ `position: 'absolute'` ได้ถูกต้องที่สุด?
ก. ชิ้นงานจะขยับเยื้องจากจุดเดิมตามแถว Flexbox และยังคงจองพื้นที่ความกว้างและความสูงไว้  
ข. ชิ้นงานจะหลุดออกจากโฟลว์การจัดแถว Flexbox ของหน้าจอ โดยสามารถระบุตำแหน่งทับซ้อนได้อย่างอิสระเทียบกับจุดเริ่มต้นของคอนเทนเนอร์แม่ตัวแรกสุดที่มีโครงสร้างล้อมรอบมัน  
ค. ชิ้นงานจะถูกลบออกจากสเตตและการแสดงผลบนหน้าจอทันที  
ง. ชิ้นงานจะขยายขนาดกว้าง 100% เสมอโดยอ้างอิงจากขนาดพิกเซลความสูงของหน้าจอระบบปฏิบัติการ  

### ข้อ 2: พร็อพส์บอกทิศทางใดต่อไปนี้ **ไม่นิยม** นำมาใช้ร่วมกับคอมโพเนนต์ที่กำหนดสไตล์เป็น `position: 'absolute'`?
ก. `top`  
ข. `right`  
ค. `flexDirection`  
ง. `bottom`  

### ข้อ 3: การประยุกต์ใช้ไลบรารี `expo-font` ร่วมกับ `expo-splash-screen` เพื่อจุดประสงค์ใดเป็นสำคัญ?
ก. เพื่อบีบอัดขนาดไฟล์รูปภาพ PNG ในแอปพลิเคชันให้เล็กลง  
ข. เพื่อให้ Splash Screen แสดงผลค้างไว้ในขณะที่แอปกำลังดาวน์โหลดฟอนต์เข้ามาเก็บในหน่วยความจำชั่วคราว ป้องกันปัญหากรอบอักษรพังหรือกระตุก (Flash of Unstyled Text)  
ค. เพื่อเปลี่ยนสีหน้าจอหลักของโทรศัพท์ให้กลายเป็นสีธีมเข้มโดยทันที  
ง. เพื่อสร้างไฟล์ติดตั้ง APK สำหรับส่งขึ้น Google Play Store  

### ข้อ 4: จากโค้ดเรียกใช้งาน `const [fontsLoaded, error] = useFonts({...})` หากฟอนต์ดาวน์โหลดพังหรือระบุพาธไฟล์ผิดพลาด ตัวแปรใดที่จะช่วยดักจับและส่งต่อสัญญาณสีแดงแจ้งปัญหาให้เราซ่อน Splash Screen เพื่อป้องกันหน้าจอค้างตลอดกาล?
ก. `fontsLoaded`  
ข. `error`  
ค. `useFonts`  
ง. `SplashScreen`  

### ข้อ 5: แหล่งสืบค้นคำสั่งนำเข้า (Import) และรายชื่อไอคอนเวกเตอร์อย่างเป็นทางการของ Expo ที่สามารถตรวจสอบรูปทรง ขนาด และสีก่อนเขียนโค้ดได้คือเว็บไซต์ใด?
ก. google.com  
ข. npmjs.com

### ข้อ 6: เหตุใดเราจึงควรเลือกใช้งาน `@expo/vector-icons` ในการออกแบบเมนูหรือปุ่มสัญลักษณ์ แทนการใช้รูปภาพไฟล์ PNG ขนาดเล็ก?
ก. ไอคอนแบบเวกเตอร์สามารถย่อขยายขนาดและเปลี่ยนเฉดสีได้อย่างคมชัดโดยภาพไม่แตก และช่วยประหยัดเนื้อที่ขนาดไฟล์แอปพลิเคชันได้ดีกว่า  
ข. ไอคอนเวกเตอร์ช่วยเชื่อมต่อบลูทูธของระบบมือถือเข้าหาอุปกรณ์เสริมได้ไวกว่า  
ค. รูปภาพแบบ PNG สามารถปรับเปลี่ยนทิศทางการหมุนได้ดีกว่าเวกเตอร์  
ง. เพื่อป้องกันไม่ให้ผู้ใช้งานสามารถก๊อปปี้โลโก้ของแอปได้  

### ข้อ 7: ข้อใดกล่าวถึงประโยชน์และความจำเป็นของการเปลี่ยนมาใช้คอมโพเนนต์ `Image` จากไลบรารี `expo-image` แทนคอมโพเนนต์ `Image` ดั้งเดิมของ React Native ได้อย่างถูกต้อง?
ก. เพื่อให้แอปพลิเคชันรันความละเอียดรูปภาพแบบ 8K ได้ลื่นไหลบนโทรศัพท์รุ่นเก่า  
ข. เพื่อความสามารถในการแคชรูปภาพระดับเนทีฟ (Native Caching) ทั้งความจำสำรองและฮาร์ดดิสก์, ลดการใช้แบนด์วิดท์ และทำให้ปัดเลื่อนรายการที่มีภาพยาว ๆ ได้ลื่นไหลขึ้น  
ค. เพื่อให้สามารถแต่งรูปภาพ ฟิลเตอร์สี และตัดภาพได้จากฝั่งโค้ดโปรเจกต์  
ง. เพื่อปิดกั้นการใช้รูปจาก URL ภายนอก และอนุญาตเฉพาะภาพในเครื่องเท่านั้น  

### ข้อ 8: การทำงานของ "Blurhash" ในความสามารถของ `expo-image` มีหน้าที่หลักอย่างไรในการเพิ่มความพึงพอใจให้ผู้ใช้ (User Experience)?
ก. ช่วยสลับรูปภาพหลักให้หมุนเป็นวงกลมเร่งความเร็วการประมวลผล  
ข. แปลงภาพจริงให้เป็นรหัสข้อความสั้น ๆ เพื่อนำมารันเป็นภาพตัวอย่างไล่เฉดสีเบลอ ๆ ทันทีในระหว่างที่รอดึงไฟล์ภาพใหญ่จากเน็ตเวิร์ก ช่วยแก้ปัญหาหน้าจอว่างเปล่า  
ค. บังคับใส่เอฟเฟกต์เบลอบนหน้าจอเมื่อผู้ใช้พยายามแคปหน้าจอแอป  
ง. ทำหน้าที่ย่อขนาดไฟล์ภาพจริงให้เหลือความกว้างเพียง 10 พิกเซลเก็บลงเซิร์ฟเวอร์  

### ข้อ 9: พร็อพส์ (Props) ตัวใดของ `expo-image` ที่ใช้ในการกำหนดเอฟเฟกต์เฟดภาพจริงขึ้นมาอย่างราบรื่นหลังจากดาวน์โหลดเสร็จสิ้น เพื่อไม่ให้แอปเกิดการกระตุกเปลี่ยนผ่านภาพกะทันหัน?

### ข้อ 10: หากต้องการรับทราบการเปลี่ยนสีธีมระบบเครื่องของระบบปฏิบัติการมือถือปลายทาง (Light/Dark Mode) แบบเรียลไทม์ ผู้พัฒนาต้องเรียกใช้งาน React Native Hook ตัวใด?

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** เป็นพฤติกรรมของ `position: 'relative'` ที่ยังคงครอบจองพื้นที่เดิมไว้ใน Layout flow
  * **ข. ถูก:** `position: 'absolute'` จะดึงตัวมันเองออกจากโฟลว์ปกติ ทำให้สไตล์ตัวถัดไปขยับขึ้นมาแทนที่ และใช้พิกัด `top/bottom/left/right` ระบุการทับซ้อนเหนือนิ้วและชิ้นงานอื่น ๆ โดยยึดตามพิกัดของ Container แม่ที่มีการระบุขอบเขต
  * **ค. และ ง. ผิด:** ไม่ได้เกี่ยวกับสเตตและไม่ได้บังคับขยาย 100% ตลอดเวลาแต่อย่างใด

### เฉลย ข้อ 2: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** `top`, `bottom` และ `right` (รวมถึง `left`) เป็นหัวใจสำคัญของ Absolute Positioning เพื่อระบุพิกัดช่องไฟจากขอบแม่
  * **ค. ถูก:** `flexDirection` เป็นพร็อพส์หลักของระบบจัดเรียง Flexbox Layout (เช่นจัดแนวแถวหรือแนวคอลัมน์) ซึ่งปกติคอมโพเนนต์ที่เป็น `absolute` จะหลุดออกจากระบบจัดเรียงนี้ จึงไม่นำมาประกาศคุมทิศทางตนเองโดยตรง (แต่มักใช้คุม Container แม่ หรือ Container ลูกภายใน absolute view อีกที)

### เฉลย ข้อ 3: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ไม่สามารถลดขนาดไฟล์ PNG ในโครงสร้างโฟลเดอร์ได้โดยตรง
  * **ข. ถูก:** `expo-splash-screen` ช่วยบล็อกไม่ให้แอปปิดหน้าจอโลโก้ของระบบตั้งต้นก่อนที่สเตตัสการดาวน์โหลด Custom Fonts ของ `expo-font` จะเสร็จสิ้นลง ซึ่งหากโหลดเสร็จสมบูรณ์จึงปล่อยให้เปิดหน้าแอปจริง จะได้ฟอนต์ที่แสดงผลสวยงามทันทีไม่มีหน้ากระตุกอักษรเปลี่ยนรูป
  * **ค. และ ง. ผิด:** ธีมเครื่องและฟังก์ชันการบิลด์ APK เป็นบทบาทหน้าที่ของตัวแปรและเอนจิ้นภายนอกตัวอื่น ๆ

### เฉลย ข้อ 4: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `fontsLoaded` จะเปลี่ยนเป็น `true` เมื่อฟอนต์ถูกโหลดผ่านเกณฑ์สำเร็จเท่านั้น
  * **ข. ถูก:** ตัวแปร `error` จากอาร์เรย์ผลลัพธ์ของ `useFonts` จะเก็บสถานะข้อผิดพลาดเมื่อเกิดปัญหาโหลดไฟล์ฟอนต์ไม่ขึ้น ซึ่งเราต้องเขียนดักเช็ค `if (fontsLoaded || error) { SplashScreen.hideAsync(); }` เพื่อให้แอปข้ามไปเรนเดอร์หน้าจอปกติ ไม่ติดค้างอยู่ที่หน้า Splash Screen
  * **ค. และ ง. ผิด:** `useFonts` เป็นเพียงฟังก์ชัน Hook และ `SplashScreen` เป็นอ็อบเจกต์คำสั่งปิดหน้ากาก ไม่ใช่ตัวแปรดักความผิดพลาด

### เฉลย ข้อ 5: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** เป็นเว็บไซต์ค้นหาทั่วไป และไม่ใช่หน้าต่างเอกสารแสดงเวกเตอร์ไอคอนอย่างเจาะจง
  * **ค. ถูก:** [icons.expo.fyi](https://icons.expo.fyi/) คือสารบัญไอคอนอย่างเป็นทางการของ Expo ที่ช่วยให้เราค้นหาชื่อและคีย์เวิร์ดของชุดไอคอนต่าง ๆ จากคลังเนทีฟได้แม่นยำ

### เฉลย ข้อ 6: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** เวกเตอร์ไอคอนถูกเก็บข้อมูลเป็นพิกัดทางคณิตศาสตร์ (Vector SVG/Font) ทำให้ย่อขยายขนาดได้ไม่จำกัดโดยไม่แตกพร่ามัว สามารถปรับแต่งสีสันได้เสรีผ่านโค้ดสไตล์ และมีขนาดข้อมูลที่เล็กกว่าไฟล์ภาพบิตแมป PNG อย่างมหาศาล
  * **ข. ค. และ ง. ผิด:** ไม่เกี่ยวข้องกับสัญญาณการจับคู่บลูทูธ และการลากหมุนเวกเตอร์ทำได้ดีกว่า PNG รวมถึงไม่ใช่เครื่องมือป้องกันการก๊อปปี้สัญลักษณ์

### เฉลย ข้อ 7: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** ไม่สามารถอัพสเกลคุณภาพหน้าจอเครื่อง หรือใช้ตัดแต่งแก้ไขภาพระดับโฟโต้ชอปได้ และรองรับรีโมท URL ปกติ
  * **ข. ถูก:** `expo-image` พัฒนาขึ้นเพื่อแก้ปัญหาด้านประสิทธิภาพโดยใช้ Engine native ระดับสูง มีประสิทธิภาพสูงในการสร้าง Cache เก็บไฟล์ภาพลงฮาร์ดดิสก์เครื่องอัตโนมัติ ช่วยลดปัญหาความล่าช้าและการกินแรมตอนผู้ใช้เลื่อนดูหน้าต่างข้อมูลยาว ๆ

### เฉลย ข้อ 8: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** ไม่เกี่ยวข้องกับกลไกความปลอดภัยการก๊อปปี้หน้าจอ หรือการลดความกว้างข้อมูลจริงบนเครื่องแม่ข่าย
  * **ข. ถูก:** `Blurhash` คือรูปแบบการคำนวณที่แปลงภาพความละเอียดสูงให้เหลือสตริงอักษรขนาดสั้น เพื่อสร้างแถบสีพรีวิวเบลอ ๆ แสดงขึ้นมาแทนที่กล่องภาพเปล่าระหว่างโหลด ทำให้ผู้ใช้รู้สึกว่าแอปตอบสนองได้อย่างรวดเร็ว

### เฉลย ข้อ 9: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** พร็อพส์ `transition` ของ `expo-image` รับค่าเวลาเป็นมิลลิวินาที (เช่น `600` = 0.6 วินาที) สำหรับกำหนดเวลาในการเล่นอนิเมชัน Fade-in เมื่อโหลดรูปจริงมาแสดงผลต่อจาก placeholder
  * **ข. ผิด:** `contentFit` ใช้ปรับระดับสเกลภาพให้พอดีกล่อง (เช่น cover, contain)
  * **ค. ผิด:** `placeholder` ใช้รับข้อมูลจำลองเพื่อรอโหลดภาพจริง
  * **ง. ผิด:** `cachePolicy` ใช้จัดการนโยบายการจัดเก็บความจำสำรอง

### เฉลย ข้อ 10: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** เป็นตัวจัดการฟอนต์, จัดระบบส้นทางเราเตอร์ และการนำทางตามลำดับ
  * **ข. ถูก:** `useColorScheme` Hook ดึงค่าสภาพแวดล้อมสว่าง/มืด (`'light'` หรือ `'dark'`) ของระบบมือถือผู้ใช้งานกลับมาผูกใน Component เพื่อสลับสไตล์ชีทตามระบบการใช้งานในโหมดมืดได้ทันที
