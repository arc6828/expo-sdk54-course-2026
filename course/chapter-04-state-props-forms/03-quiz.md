# แบบทดสอบท้ายบทเรียน บทที่ 4: การจัดการสถานะ พร็อพส์ และแบบฟอร์มรับข้อมูล (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความเข้าใจเกี่ยวกับแนวคิดเรื่องสถานะ (State) การทำงานร่วมกับ Props โครงสร้าง Controlled Components ของแบบฟอร์ม การส่งค่าข้อมูลกลับขึ้นด้านบนด้วยหลัก Lifting State Up และการดักจับข้อผิดพลาดอินพุต (Form Validation) ใน React Native Expo SDK 54

---

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: ข้อใดอธิบายวัตถุประสงค์และพฤติกรรมการทำงานหลักของ "State" ใน React Native ได้ถูกต้องและสมบูรณ์ที่สุด?
ก. เป็นสไตล์ชีทภายนอกที่ใช้อ่านข้อมูลจากเครื่องระบบปฏิบัติการปลายทางเท่านั้น  
ข. เป็นชุดตัวแปรภายในคอมโพเนนต์ที่สามารถเปลี่ยนค่าข้อมูลได้ โดยระบบจะบังคับทำการวาดโครงสร้างหน้าจอใหม่ (Re-render) อัตโนมัติเมื่อค่านั้นเปลี่ยนไป  
ค. เป็นพร็อพส์ภายนอกที่ส่งมาจากคอมโพเนนต์แม่เพื่อสแกนล้างหน่วยความจำ RAM  
ง. เป็นฟังก์ชันสำหรับส่งข้อความสั้นหรือแจ้งเตือนแบบพุช (Push Notification) ไปยังมือถือเครื่องอื่น  

### ข้อ 2: เมื่อต้องการประกาศใช้งานตัวแปรสถานะภายใน Functional Component ผู้พัฒนาต้องเรียกใช้งาน React Hook ตัวใด?
ก. `useEffect`  
ข. `useRef`  
ค. `useState`  
ง. `useMemo`  

### ข้อ 3: การประกาศรูปแบบฟังก์ชันสถานะตามหลักการทั่วไป เช่น `const [isOnline, setIsOnline] = useState(false);` ข้อใดกล่าวถึงบทบาทหน้าที่ของตัวแปร `setIsOnline` ได้ถูกต้อง?
ก. เป็นตัวแปรเก็บสถานะปัจจุบัน มีค่าประเภทข้อความ (String) เสมอ  
ข. เป็นฟังก์ชันที่ทำหน้าที่เดียวในการลบตัวแปร `isOnline` ออกจากสเตตระบบ  
ค. เป็นฟังก์ชันประเภท Setter ที่ใช้ในการอัปเดตและเปลี่ยนแปลงค่าของสถานะ `isOnline` เพื่อส่งสัญญาณจัดทำ Re-render หน้าจอ  
ง. เป็นคำสั่งที่ส่งไปสร้างจุดกลมสีเขียวนีออนแสดงการเชื่อมต่อเน็ตเวิร์กของเครื่องอัตโนมัติ  

### ข้อ 4: ข้อใดคือความแตกต่างทางด้านสิทธิ์การแก้ไขค่า (Mutability) ระหว่าง State และ Props ของ React/React Native?
ก. State มีลักษณะใช้อ่านอย่างเดียว (Read-only) ส่วน Props สามารถเขียนโค้ดสลับเปลี่ยนค่าด้านในได้อย่างอิสระ  
ข. State สามารถเปลี่ยนแปลงค่าได้ตามจังหวะพิมพ์สัมผัส ส่วน Props มีลักษณะใช้อ่านอย่างเดียวจากมุมมองของคอมโพเนนต์ลูกที่รับไป  
ค. ทั้ง State และ Props ต่างไม่ยอมรับการเปลี่ยนแปลงค่าเลยเมื่อแอปพลิเคชันถูกบิลด์เสร็จ  
ง. State ใช้แก้ไขเฉพาะประเภทข้อมูลตัวเลข ส่วน Props ใช้แก้ไขประเภทข้อความตัวอักษรเท่านั้น  

### ข้อ 5: แนวคิดแบบ Controlled Component ของช่องรับอินพุต `TextInput` มีลักษณะการจัดระเบียบข้อมูลอย่างไร?
ก. ปล่อยให้แป้นพิมพ์ของระบบควบคุมและจัดเก็บตัวหนังสือไว้ในฮาร์ดแวร์โดยตรง  
ข. นำ State ของ React มาเป็นศูนย์กลางข้อมูลผูกติดกับคอมโพเนนต์ เพื่อคอยควบคุมและสะท้อนค่าตัวหนังสือปัจจุบันเข้าหากันตลอดเวลา  
ค. ปิดแป้นคีย์บอร์ดทั้งหมดและรับข้อมูลด้วยวิธีลากผ่านแถบเสียงเท่านั้น  
ง. ดึงสไตล์จากไฟล์ภายนอกมาควบคุมและตรวจสอบสีของหน้าแบบฟอร์ม  

### ข้อ 6: พร็อพส์ (Props) หลักคู่ใดของคอมโพเนนต์ `TextInput` ที่ต้องเขียนผูกทำงานร่วมกันเพื่อให้มีพฤติกรรมเป็น Controlled Component ที่สมบูรณ์?
ก. `placeholder` และ `keyboardType`  
ข. `value` และ `onChangeText`  
ค. `secureTextEntry` และ `autoCapitalize`  
ง. `editable` และ `maxLength`  

### ข้อ 7: ในกรณีที่คอมโพเนนต์ลูกสองชิ้นที่ไม่มีความสัมพันธ์เป็นแม่-ลูกกันโดยตรง (เช่น ปุ่มสลับธีม และการ์ดแสดงสี) ต้องการใช้สถานะข้อมูลเดียวกัน วิธีการจัดสถาปัตยกรรมสเตตที่ดีที่สุดคืออะไร?
ก. เขียนคัดลอกไฟล์ State แบบเดียวกันไว้ในคอมโพเนนต์ลูกทั้งสองที่แยกอิสระจากกัน  
ข. ประยุกต์ใช้เทคนิค "Lifting State Up" โดยย้าย State ขึ้นไปไว้ที่คอมโพเนนต์ที่เป็นบรรพบุรุษร่วมกันที่ใกล้ที่สุด (Common Parent)  
ค. ยกเลิกการใช้ State และใช้รูปแบบ CSS Inline ในการเปลี่ยนสี  
ง. ส่งข้อมูลผ่านทางเครือข่ายอินเทอร์เน็ตเพื่อเก็บข้อมูลที่ฐานข้อมูลกลางทุกรอบพิมพ์  

### ข้อ 8: คอมโพเนนต์ลูก (Child Component) สามารถส่งข้อมูลกลับขึ้นไปแจ้งเปลี่ยนแปลงหรืออัปเดตสเตตในคอมโพเนนต์แม่ (Parent Component) ได้อย่างไร?
ก. เรียกใช้ฟังก์ชัน `useState` ของแม่ตรง ๆ ผ่านการอ้างอิงชื่อคลาส  
ข. แม่ต้องส่งฟังก์ชันประเภท Callback (ฟังก์ชันอัปเดตค่า) ลงไปให้ลูกในรูปแบบของ Props และให้ลูกเรียกใช้ฟังก์ชันนั้นพร้อมป้อนข้อมูลล่าสุดเป็นอาร์กิวเมนต์  
ค. เขียนเปลี่ยนสเตตลงในไฟล์สไตล์ชีทภายนอกเพื่อให้คอมไพเลอร์เปลี่ยนค่า  
ง. ไม่มีวิธีการตามข้อกำหนดของ React เนื่องจากข้อมูลไหลได้เฉพาะทิศทางลงล่างเท่านั้น  

### ข้อ 9: ข้อใดคือเป้าหมายหลักของการเขียนตรวจสอบความถูกต้องแบบฟอร์มในฝั่งไคลเอนต์ (Client-Side Form Validation)?
ก. เพื่อสแกนและเก็บฐานข้อมูลรหัสผ่านส่งต่อไปยังเซิร์ฟเวอร์แบบลับ ๆ  
ข. เพื่อให้ผู้ใช้สัมผัสปุ่มกดและเห็นการตอบสนองได้ช้าลงเล็กน้อย  
ค. เพื่อให้ข้อมูลที่ป้อนไม่ขัดต่อเงื่อนไขเบื้องต้น ส่งผลให้แสดงข้อเสนอแนะความผิดพลาด (Error Feedback) แก่ผู้ใช้ได้ทันที และช่วยลดการส่งรีเควสขยะไปยังระบบหลังบ้าน  
ง. เพื่อเร่งความเร็วการส่งไฟล์รูปภาพขึ้นไคล์เซิร์ฟเวอร์  

### ข้อ 10: หากฟอร์มรับอินพุตยังกรอกข้อมูลไม่ถูกต้องหรือไม่ครบเงื่อนไข เราจะใช้เทคนิคใดในการปิดการทำงานของปุ่มตอบรับ `Pressable` เพื่อความปลอดภัย?
ก. ตั้งค่าพร็อพส์ `disabled={!isFormValid}` ร่วมกับการปรับสไตล์สีปุ่มให้จางลง (เช่น ใช้ Opacity ต่ำ) เมื่อข้อมูลไม่ถูกต้อง  
ข. ตั้งค่าพร็อพส์ `editable={false}` เพื่อลบสุ่มปุ่มออกจากหน้าต่างจอหลัก  
ค. เขียนฟังก์ชัน `Alert.alert()` ซ้อนในสไตล์เพื่อให้ผู้ใช้ลบตัวหนังสือทิ้ง  
ง. บังคับเปลี่ยนคีย์บอร์ดของมือถือผู้ใช้ให้กลายเป็นหน้าปุ่มตัวเลขทั้งหมด

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** State ไม่ใช่ตัวควบคุมหรืออ่านสไตล์ภายนอกของระบบปฏิบัติการ
  * **ข. ถูก:** State คือสภาพหน่วยความจำชั่วคราวที่เป็นส่วนตัวภายในคอมโพเนนต์ เมื่อเรียกตัวจัดการสเตตมาอัปเดตข้อมูล React จะจัดทำการ Re-render หน้าอินเทอร์เฟซเพื่อให้ผู้เรียนหรือผู้ใช้งานพบภาพสัดส่วนข้อมูลปัจจุบันของคอมโพเนนต์นั้นได้ทันที
  * **ค. และ ง. ผิด:** Props เป็นคนละเรื่องกับสเตตภายใน และ State ไม่เกี่ยวข้องกับการจัดการพุชโนติฟาย

### เฉลย ข้อ 2: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `useEffect` ใช้จัดการผลกระทบข้างเคียง (Side Effects) เช่น การดึงค่า API หรือผูกเซนเซอร์เครื่อง
  * **ข. ผิด:** `useRef` ใช้เก็บตัวแปรที่ใช้อ้างอิงโดยตรงหรืออ็อบเจกต์ที่เมื่อเปลี่ยนค่าแล้วจะไม่กระตุ้นการเรนเดอร์ใหม่
  * **ค. ถูก:** `useState` เป็น Hook ดั้งเดิมที่ React เตรียมไว้ใช้สร้างและเข้าถึงตัวจัดการสเตตใน Functional Components
  * **ง. ผิด:** `useMemo` ใช้แคชค่าการคำนวณที่ซับซ้อนเพื่อเพิ่มประสิทธิภาพ

### เฉลย ข้อ 3: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ตัวเก็บค่าคือ `isOnline` ไม่ใช่ตัวแปรฟังก์ชัน `setIsOnline`
  * **ข. ผิด:** ฟังก์ชันใช้เปลี่ยนค่าสเตต ไม่ใช่คีย์ลบโครงสร้างตัวแปร
  * **ค. ถูก:** ตัวแปรตัวที่สองที่ได้จากการทำ Array Destructuring ของ `useState` จะทำหน้าที่เป็นตัว Setter คอยรับเปลี่ยนสเตต และแจ้งสั่งการให้ตัวเอนจิ้นของ React เข้าสู่วงรอบการแสดงผลหน้าจอใหม่
  * **ง. ผิด:** ไม่ได้เป็นคำสั่งออโต้สร้างจุดไอคอน แต่ผู้พัฒนาสามารถนำค่าของสเตต `isOnline` ไปคุมเงื่อนไขสีวงกลมได้เอง

### เฉลย ข้อ 4: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** สลับบทบาทกันอย่างสิ้นเชิง
  * **ข. ถูก:** State กำหนดในตนเองและเปลี่ยนข้อมูลได้เองเรื่อย ๆ ส่วน Props เป็นคุณสมบัติที่คนนอก (Component แม่) ยื่นส่งเข้ามาให้ โดยลูกที่รับไปห้ามแก้ไขสลับค่านั้นเองเพื่อไม่ให้กระแสข้อมูลขัดกัน
  * **ค. และ ง. ผิด:** มีการเปลี่ยนแปลงค่าได้ปกติยามรันไทม์ และทำงานได้ทุกประเภทข้อมูลอย่างไม่มีข้อจำกัดด้านชนิด

### เฉลย ข้อ 5: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** แป้นพิมพ์ไม่ควรเป็นผู้ตัดสินใจเก็บข้อความหลักเดี่ยว ๆ เพราะแอปจะไม่ทราบค่าปัจจุบันแบบทันที
  * **ข. ถูก:** Controlled Component หมายถึงการออกแบบให้อินพุตผูกข้อมูลข้อความปัจจุบันไว้บนสเตตของ React โดยตรง ทุกจังหวะการเปลี่ยนอักขระจะเข้าไปอัปเดตสเตต และตัวสเตตนั้นเองจะสะท้อนความจริงกลับมาป้อนบนหน้าต่างจออินพุต
  * **ค. และ ง. ผิด:** ไม่ใช่การลบหรือควบคุมแป้น และไม่ใช่เรื่องการดีไซน์เฉดสีอย่างเดียว

### เฉลย ข้อ 6: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** พร็อพส์อื่น ๆ ช่วยอำนวยสไตล์และความปลอดภัยเบื้องต้น แต่ไม่เพียงพอต่อการควบคุมสถานะข้อมูล
  * **ข. ถูก:** เพื่อให้คอมโพเนนต์รับค่าอยู่ภายใต้การควบคุม (Controlled) พร็อพส์ `value` จะเป็นตัวชี้นำแสดงค่าสตริงของสเตต และพร็อพส์ `onChangeText` จะเป็นตัวส่งสัญญาณเมื่อผู้ใช้ป้อนอักขระล่าสุดเพื่อเปลี่ยนแปลงค่าสเตตตามลำดับ

### เฉลย ข้อ 7: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** การทำสำเนาสเตตแยกอิสระจะส่งผลให้หน้าจอทำงานไม่สอดคล้องกัน (เช่น ปุ่มเปลี่ยนธีมแล้วแต่การ์ดยังสีเดิม)
  * **ข. ถูก:** ตามหลัก Lifting State Up การยก State ไปฝากไว้ที่บรรพบุรุษร่วม (Parent Component) จะช่วยให้เราแชร์สเตตชิ้นนั้นส่งผ่านลงมาในรูปแบบ Props ให้ลูกทั้งสองคนได้พร้อม ๆ กันอย่างเป็นระบบและสม่ำเสมอ
  * **ค. และ ง. ผิด:** การใช้ CSS ตรงไม่แก้ไขการเปลี่ยนข้อมูล และการโหลดเน็ตเวิร์กทุกรอบพิมพ์จะทำให้เครื่องช้าและสิ้นเปลือง

### เฉลย ข้อ 8: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** React ไม่อนุญาตให้ดึงใช้งานอ้างอิงข้ามฟังก์ชันสเตตของตัวแม่ตรง ๆ ด้วยความสามารถทั่วไป
  * **ข. ถูก:** เราจำเป็นต้องกำหนดพร็อพส์ให้รับฟังก์ชัน Callback จากคอมโพเนนต์แม่ เมื่อเกิดเหตุสัมผัสปุ่มหรืออินพุต คอมโพเนนต์ลูกจะกระตุ้นฟังก์ชันนั้นย้อนกลับไป และสั่งเปลี่ยนค่าสเตตต้นฉบับส่งผลให้แม่และลูกเรนเดอร์อัปเดตกันถ้วนหน้า
  * **ค. และ ง. ผิด:** ไฟล์สไตล์ชีทไม่มีอิมแพคควบคุมข้อมูลสเตต และสามารถส่งข้อมูลขึ้นด้านบนได้ผ่านรูปแบบการใช้ Callback Function

### เฉลย ข้อ 9: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. และ ข. ผิด:** ไม่เกี่ยวกับสปายแวร์หลังบ้าน หรือการดึงจังหวะสัมผัสให้ตอบสนองช้าลง
  * **ค. ถูก:** การทำ Form Validation ฝั่งมือถือปลายทางช่วยเตือนคำแนะนำแก่ผู้ใช้งานแอปอย่างฉับพลันเมื่อป้อนข้อความผิดพลาด และป้องกันการส่งขยะ (ข้อมูลไม่ครบหรือผิดโครงสร้าง) ไปเซิร์ฟเวอร์
  * **ง. ผิด:** ระบบส่งรูปขึ้นคลาวด์จะดีขึ้นตามขนาดไฟล์และแบนด์วิดท์ความเร็วเน็ต ไม่เกี่ยวกับกฎข้อบังคับของฟอร์มอินพุต

### เฉลย ข้อ 10: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** คอมโพเนนต์ `Pressable` และคอมโพเนนต์ปุ่มอื่น ๆ มีพร็อพส์ `disabled` ซึ่งจะระงับเหตุการณ์ทริกเกอร์ `onPress` เมื่อค่าเป็น `true` ผู้พัฒนาสามารถนำการประเมินสเตตฟอร์มมาผูกร่วมและเขียนสไตล์ปรับเปลี่ยนสปอตไลท์ความสว่างปุ่มได้ชัดเจน
  * **ข. ผิด:** `editable` เป็นคุณสมบัติสำหรับกล่อง `TextInput` ไม่ใช่ปุ่มกด
  * **ค. และ ง. ผิด:** `Alert.alert` ไม่ระงับการสัมผัสปุ่ม และประเภทคีย์บอร์ดตัวเลขใช้ควบคุมอินพุตเท่านั้น
