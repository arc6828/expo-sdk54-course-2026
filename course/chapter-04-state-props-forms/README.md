# บทที่ 4: การจัดการสถานะ พร็อพส์ และแบบฟอร์มรับข้อมูล (Overview)

ในบทเรียนนี้ ผู้เรียนจะได้เข้าสู่หัวใจของการพัฒนาแอปพลิเคชันที่มีการตอบสนองและโต้ตอบกับผู้ใช้งาน ด้วยการใช้งานคุณสมบัติของ **State (สถานะ)** ผ่าน `useState` Hook การวิเคราะห์การทำงานร่วมกันกับ Props ตลอดจนการทำความเข้าใจสถาปัตยกรรมกระแสข้อมูลย้อนกลับ (Lifting State Up) และการนำมาสร้างแบบฟอร์ม Controlled Components ที่มีการประมวลผลอินพุต ตรวจสอบความถูกต้อง (Form Validation) ได้อย่างปลอดภัยและมีประสิทธิภาพ

---

## 📂 สารบัญเอกสารประจำบทเรียน (Chapter Documents)
คุณสามารถเลือกอ่านและเข้าถึงไฟล์เอกสารต่าง ๆ ในโฟลเดอร์บทเรียนนี้ได้โดยตรง:

1. 📋 **[แผนการสอนประจำบท (01-lesson-plan.md)](./01-lesson-plan.md)**  
   รายละเอียดหัวข้อ วัตถุประสงค์เชิงพฤติกรรม เกณฑ์ประเมินผลทักษะการตรวจสอบความถูกต้องแบบฟอร์มและการโต้ตอบปุ่ม
2. 📖 **[เนื้อหาหลักการเรียนรู้ (02-content.md)](./02-content.md)**  
   เนื้อหาหลักทางทฤษฎีอธิบายสเตต (`useState`), ตารางเปรียบเทียบความแตกต่างเด่นชัดระหว่าง State vs Props, หลักการจัดวางช่องรับข้อความแบบ Controlled Components, การสื่อสารส่งข้อมูลขึ้นด้านบน (Lifting State Up) และการจัดการแจ้งเตือนข้อผิดพลาดอินพุต (Error Feedback)
3. 📝 **[แบบทดสอบท้ายบทเรียน (03-quiz.md)](./03-quiz.md)**  
   แบบทดสอบปรนัยทวนความรู้ 10 ข้อ ครอบคลุมการเลือกใช้ Hook, กลไกของกระแสข้อมูล, และวิธีการระงับปุ่มกดสัมผัสเมื่อเงื่อนไขฟอร์มไม่เป็นจริง พร้อมเฉลยและคำอธิบายโดยละเอียด
4. 💻 **[คู่มือปฏิบัติการขั้นพื้นฐาน (04-lab.md)](./04-lab.md)**  
   โจทย์ปฏิบัติการสร้าง **หน้าจอแบบประเมินความพึงพอใจของลูกค้า (Customer Feedback & Review Form)** เพื่อให้ผู้เรียนฝึกเขียน Custom Component จัดระดับดาวรับสัญญาณผ่าน callback, ตรวจความยาวข้อมูล, และแสดงตัวอย่างการ์ดประเมินผลสดแบบเรียลไทม์ (Live Preview)

## 🛠️ สรุปโปรเจคปฏิบัติการท้ายบทเรียน (Lab Project Summary)
ในปฏิบัติการของบทนี้ ผู้เรียนจะสร้างส่วนประกอบอินเทอร์เฟซแบบฟอร์มประเมินความชอบ:
* **Live Review Card:** ชิ้นงานการ์ดสรุปผลด้านบนของจอที่จะเปลี่ยนข้อความ ชื่อคน ดาวประเมิน และคำบรรยายตามการป้อนสเตตจริง
* **Custom Star Selector Component:** คอมโพเนนต์เลือกระดับคะแนนที่ใช้หลัก Lifting State Up ส่งค่าปุ่มกดดาวกลับไปยัง Component แม่เพื่อบันทึกสเตต
* **Controlled Form Validation:** ตรวจจับความถูกต้องของชื่อ (ขั้นต่ำ 2 ตัว) และช่องข้อคิดเห็น (ขั้นต่ำ 10 ตัว) พร้อมแจ้งกรอบแดงเมื่อข้อมูลบกพร่อง
* **Dynamic Submit Button:** ตั้งค่าพร็อพส์ `disabled` ของปุ่ม `Pressable` ให้อัปเดตสีและปิดรับการกดสัมผัสจนกว่าผู้ใช้จะกรอกข้อมูลได้ตามเกณฑ์ทั้งหมด
