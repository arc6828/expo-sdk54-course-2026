# บทที่ 4: การจัดการสถานะ พร็อพส์ และแบบฟอร์มรับข้อมูล (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- แนวคิดความสำคัญของสถานะ (State) ใน React Native และการเรียกใช้งาน `useState` Hook
- ข้อแตกต่างในการใช้งานและการเปลี่ยนค่าระหว่าง State (สถานะภายในควบคุมเอง) และ Props (คุณสมบัติจากภายนอก)
- การสร้างและประยุกต์ใช้ Controlled Components สำหรับกล่องรับข้อความ `TextInput` หลายช่องพร้อมกัน
- การส่งต่อสถานะและการสื่อสารย้อนกลับ (Callback Functions) ผ่านหลักการยกสถานะขึ้น (Lifting State Up)
- การจัดการฟอร์มรับข้อมูล การตรวจสอบความถูกต้องเบื้องต้น (Basic Form Validation) และการแสดงข้อความแจ้งเตือนข้อผิดพลาด (Error Feedback)
- การจัดการเหตุการณ์การกดส่งฟอร์ม (Form Submission) ร่วมกับปฏิสัมพันธ์การเปลี่ยนสีปุ่มสัมผัส

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถระบุความแตกต่างและเชื่อมโยงความสัมพันธ์ของ State และ Props ได้อย่างชัดเจน
2. ผู้เรียนสามารถสร้างตัวแปรสถานะและเขียนฟังก์ชันอัปเดตสเตตผ่าน `useState` เพื่อตอบสนองต่ออินเตอร์เฟซได้ถูกต้อง
3. ผู้เรียนสามารถผูกตัวแปร State เข้ากับช่องป้อนค่าแบบ Controlled Components ได้สมบูรณ์
4. ผู้เรียนสามารถประยุกต์ใช้รูปแบบ Lifting State Up เพื่อจัดโครงสร้างให้คอมโพเนนต์ลูกส่งค่ากลับไปบันทึกยังสเตตของคอมโพเนนต์แม่ได้สำเร็จ
5. ผู้เรียนสามารถเขียนโค้ดตรวจสอบเงื่อนไขความถูกต้องของข้อมูลฟอร์ม (เช่น ความยาวรหัสผ่าน หรือโครงสร้างอีเมล) พร้อมแสดงสถานะแจ้งข้อผิดพลาดใต้ช่องอินพุตได้ถูกต้อง

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายและจำลองแบบทดลอง (Visual State Flow Lecture):** ผู้สอนนำเสนอสไลด์ภาพและยกตัวอย่างแผนภาพกระแสข้อมูลไหลจากบนลงล่าง และการก้าวข้ามขีดจำกัดด้วยการทำ Callback ส่งค่าอัปเดตสถานะกลับขึ้นด้านบน
2. **การทำ Live Coding Demonstration หัวข้อ Interactive Form Setup:** ผู้สอนนำเสนอการสร้างฟอร์มสมัครสมาชิก (Sign Up Form) ที่มีช่องกรอกชื่อ อีเมล รหัสผ่าน และมีการตรวจสอบข้อมูลแบบเรียลไทม์เพื่อปลดล็อกปุ่มกดส่ง
3. **กิจกรรมปฏิบัติกลุ่มย่อยท้าทายสเตต (Form Validation Challenge):** แบ่งกลุ่มผู้เรียนทดลองแก้บั๊กในโค้ดฟอร์มลงทะเบียน โดยต้องเขียนฟังก์ชันตรวจสอบว่าอีเมลมีเครื่องหมาย `@` หรือไม่ และแสดงกรอบช่องเป็นสีแดงหากกรอกข้อมูลผิดพลาด
4. **กิจกรรมเขียนระบบรวมการ์ด (Lifting State to Preview Card):** ให้ผู้เรียนฝึกประดิษฐ์ Custom Input Component และส่งสัญญาณเหตุการณ์เปลี่ยนข้อความส่งต่อไปยังการ์ดพรีวิวบนสุดให้เปลี่ยนตามการพิมพ์จริง

## สื่อการเรียนการสอน
1. เอกสารสไลด์นำเสนอ "State, Props, and Controlled Forms in React Native"
2. ตัวอย่างโค้ดแอปพลิเคชันลงทะเบียนสมาชิกพร้อมฟังก์ชันตรวจสอบความถูกต้องของข้อมูล (Form Validation Demo) บน Git Repository
3. โปรแกรมจำลองหน้าจอและอุปกรณ์จริงสำหรับการทดสอบระบบตอบสนองแป้นพิมพ์และสเตตอินพุต

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 วัดความเข้าใจจากแบบทดสอบปรนัยท้ายบทเรียน 10 ข้อ
1.2 ประเมินความถูกต้องของโครงสร้าง Controlled Components และการจัดสเตตอินพุต
1.3 ตรวจประเมินทักษะการตรวจสอบความถูกต้องของฟอร์ม (Form Validation) และการยกสถานะ (Lifting State Up) ในแล็บปฏิบัติการ

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถเขียนโปรแกรมฟอร์มรับข้อมูลที่มีฟังก์ชันตรวจสอบความถูกต้องและโชว์ปุ่มส่งค่าตอบสนองสถานะได้สมบูรณ์ผ่านเกณฑ์ปฏิบัติการ (ผ่าน/ไม่ผ่าน)
2.3 การประเมินพฤติกรรมการแลกเปลี่ยนความคิดเห็นและการช่วยเหลือกันตอบโจทย์ข้อผิดพลาดในคาบเรียน
