# บทที่ 12: ระบบยืนยันตัวตนและการเข้าถึงข้อมูลด้วย Supabase Authentication (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- แนวคิดและสถาปัตยกรรมระบบการยืนยันตัวตน (Authentication) และการอนุญาตสิทธิ์การเข้าถึงข้อมูล (Authorization) บนโมบายแอปพลิเคชัน
- สถาปัตยกรรมระบบ Supabase Auth และการทำความเข้าใจกลไกผู้ให้บริการตัวตน (GoTrue API)
- การสร้างหน้าจอสมัครสมาชิก (Sign Up) และการล็อกอินเข้าใช้งานระบบด้วยอีเมลและรหัสผ่าน (Sign In with Email & Password)
- การควบคุมสถานะและ Lifecycle ของผู้ใช้ปัจจุบันแบบง่ายด้วย Local Component State (`useState` + `useEffect`)
- การดักฟังเหตุการณ์การเปลี่ยนผ่านเซสชันผ่านเมธอด `onAuthStateChange` เพื่ออัปเดต UI แบบรีลไทม์
- การพัฒนาระบบตรวจสอบอินพุตและความปลอดภัยพื้นฐานก่อนยิงรีเควส (Form Validation)
- ความปลอดภัยในการเก็บโทเค็นของระบบด้วย `expo-secure-store` แทนการใช้ AsyncStorage เพื่อป้องกันภัยข้อมูลและคีย์รั่วไหล
- **[เนื้อหาพิเศษตอนท้าย (Bonus Section)]**: แนวทางการรีแฟกเตอร์ (Refactoring) สู่สถาปัตยกรรมสากลด้วย `useContext` (AuthContext) สำหรับการแชร์สเตตัสผู้ใช้ข้ามหน้าจอในแอปพลิเคชันขนาดใหญ่

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถอธิบายแนวคิดเรื่อง JWT (JSON Web Tokens) และสถาปัตยกรรมยืนยันตัวตนของ Supabase Auth ได้ถูกต้อง
2. ผู้เรียนสามารถเขียนคำสั่งและสร้างแบบฟอร์มรับสมัครสมาชิก (`signUp`) และลงชื่อเข้าใช้ (`signInWithPassword`) ได้สำเร็จ
3. ผู้เรียนสามารถดึงค่าเซสชันและใช้งาน `onAuthStateChange` ร่วมกับ `useState` เพื่อทำระบบสลับการแสดงผลหน้าจอแบบ Conditional Rendering ได้สำเร็จ
4. ผู้เรียนสามารถเขียนโปรแกรมนำ `expo-secure-store` ไปใช้จัดเก็บเซสชันล็อกอินของ Supabase Client ในแบบเนทีฟได้อย่างปลอดภัย
5. ผู้เรียนสามารถระบุและรับมือข้อผิดพลาดทั่วไปจาก Supabase API เช่น อีเมลซ้ำ หรือรหัสผ่านไม่ถูกต้อง และแสดงผลให้ผู้ใช้ทราบอย่างปลอดภัย
6. ผู้เรียนสามารถอธิบายหลักการและแนวทางของการประยุกต์ใช้ `useContext` ในการขยายสเกลระบบยืนยันตัวตนของแอปพลิเคชันได้ถูกต้อง

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายเรื่อง JWT & Direct Login Flow:** บรรยายหลักการจัดเก็บสถานะผู้ใช้ด้วยโทเค็นความปลอดภัย, การทำ Conditional Rendering สลับหน้า Login/Dashboard บนหน้าจอเดียว และความสำคัญของการเลือกใช้ Secure Store บนโมบายแอปพลิเคชัน
2. **การทำ Live Coding Demonstration หัวข้อ Single-Page Supabase Login & Secure Storage:** ผู้สอนสาธิตการสร้างแอปที่มีกลไกรันตรวจสอบสถานะเซสชันดั้งเดิม, แสดงผลฟอร์มล็อกอินเมื่อไม่มีผู้ใช้งาน และโชว์หน้าตารางแดชบอร์ดเมื่อล็อกอินเข้าสู่ระบบได้สำเร็จ โดยเชื่อมต่อกับ `expo-secure-store`
3. **กิจกรรมตรวจสอบความหนาแน่นรหัสผ่าน (Auth Validation Challenge):** ให้ผู้เรียนฝึกฝนเขียนรหัสกรองอีเมลและรหัสผ่านก่อนยิง API ของ Supabase เพื่อประหยัดการยิงรีเควสแบบไม่มีคุณภาพ
4. **กิจกรรมสัมมนารีแฟกเตอร์สู่ระบบสากล (Context Refactoring Demonstration):** ผู้สอนสาธิตหัวข้อของแถมพิเศษด้วยการนำโค้ดในหน้าจอหลักมารีแฟกเตอร์ขยายระบบ โดยแยกเป็น `AuthContext.Provider` เพื่ออธิบายว่าหากผู้เรียนต้องการทำแอปที่มีหน้าจอหลากหลายของ Expo Router จะสามารถนำ Context ไปดักปิดป้อนสิทธิ์การเปิดอ่านไฟล์และหน้าจอได้อย่างไร

## สื่อการเรียนการสอน
1. เอกสารสไลด์เรื่อง "Supabase Authentication, Secure Store and Refactoring to useContext"
2. บัญชีผู้ใช้งานระบบคลาวด์ Supabase (สำหรับเปิดดูตาราง Authentication -> Users)
3. เครื่องมือสมาร์ทโฟนหรืออุปกรณ์จำลองสำหรับการทดสอบปิดแอปพลิเคชันแล้วเปิดใหม่เพื่อเช็คการคงอยู่ของล็อกอิน
4. คู่มืออ้างอิง API ของ Supabase Auth, `expo-secure-store` และ React Context API

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 ประเมินผลคะแนนการทำแบบทดสอบท้ายบทเรียน 10 ข้อ
1.2 ตรวจสอบความถูกต้องของการนำ `expo-secure-store` มาควบคุมเซสชันของ Supabase Client ในโค้ดใบงานปฏิบัติการ
1.3 สังเกตประสิทธิภาพความลื่นไหลการเข้าสู่ระบบและจดจำเซสชันเมื่อผู้ใช้เปิดแอปขึ้นมาใหม่

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถสร้างระบบล็อกอิน สมัครสมาชิก และจดจำสเตตัสผู้ใช้งานในหน้าจอหลักผ่าน `expo-secure-store` ได้สำเร็จผ่านเกณฑ์ (ผ่าน/ไม่ผ่าน)
2.3 สังเกตพฤติกรรมการจัดการข้อผิดพลาดระบบหลังบ้าน และความเข้าใจเบื้องต้นในการรีแฟกเตอร์สู่โครงสร้าง Context
