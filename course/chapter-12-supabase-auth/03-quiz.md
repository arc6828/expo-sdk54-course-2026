# แบบทดสอบท้ายบทเรียน บทที่ 12: ระบบยืนยันตัวตนและการเข้าถึงข้อมูลด้วย Supabase Authentication (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความรู้และความเข้าใจเกี่ยวกับหลักสถาปัตยกรรมความปลอดภัยของแอปพลิเคชันมือถือ, การใช้คำสั่ง API ของ Supabase Auth สำหรับการสมัครสมาชิกและเข้าสู่ระบบ, การควบคุมสถานะการใช้งาน (Session Lifecycle), การรักษาความปลอดภัยระดับอุปกรณ์ด้วย `expo-secure-store` และแนวคิดของการจัดสร้างโครงสร้างตัวเก็บสเตตัสส่วนกลางด้วย React Context API

---

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: ข้อใดนิยามความแตกต่างระหว่างแนวคิด "การยืนยันตัวตน (Authentication)" และ "การอนุญาตสิทธิ์ (Authorization)" ได้อย่างถูกต้องที่สุด?
ก. Authentication คือการตั้งข้อจำกัดขนาดกว้างยาวของหน้าจอแอป ส่วน Authorization คือการควบคุมความสว่างของหน้าเลนส์กล้อง  
ข. Authentication คือการตรวจสอบพิสูจน์ตนว่าผู้ใช้คือใครจริงหรือไม่ (เช่น ล็อกอินผ่านอีเมล) ส่วน Authorization คือการดักกรองเงื่อนไขว่าผู้ใช้นั้นมีสิทธิ์เข้าถึงหรือแก้ไขข้อมูลชิ้นใดบ้าง (เช่น กฎ RLS)  
ค. Authentication คือการเขียนสไตล์จัดกลุ่มกล่อง Flexbox ส่วน Authorization คือการส่งคีย์ผ่านทางสายเครือข่ายอินเทอร์เน็ต  
ง. ทั้งสองแนวคิดคือเรื่องเดียวกันและไม่มีส่วนใดแตกต่างกันในทางสถาปัตยกรรมระบบคอมพิวเตอร์  

### ข้อ 2: บริการหลังบ้านตัวใดที่รันทำหน้าที่ควบคุมกลไกความปลอดภัยและระบบสมาชิก (Authentication Engine) อยู่เบื้องหลังของ Supabase?
ก. Firebase Auth  
ข. PostgreSQL Engine  
ค. GoTrue API  
ง. Auth0 Core  

### ข้อ 3: หากต้องการเขียนโค้ดบันทึกสร้างบัญชีผู้ใช้งานคนใหม่ด้วยอีเมลและรหัสผ่านเข้าสู่ดาต้าเบส จะต้องเรียกใช้เมธอด Asynchronous ใดของ Supabase JS Client?
ก. `supabase.auth.createUser()`  
ข. `supabase.auth.signUp()`  
ค. `supabase.auth.register()`  
ง. `supabase.auth.insertUser()`  

### ข้อ 4: เมธอดล็อกอินใดของ Supabase Auth ที่มีบทบาทใช้ส่งอีเมลและรหัสผ่านเพื่อเรียกคืนข้อมูลสิทธิ์และรับเซสชันโทเค็นปัจจุบัน?
ก. `supabase.auth.signIn()`  
ข. `supabase.auth.loginWithPassword()`  
ค. `supabase.auth.signInWithPassword()`  
ง. `supabase.auth.authenticate()`  

### ข้อ 5: เมื่อต้องการเฝ้าติดตามรับฟังการเปลี่ยนผ่านระดับความเคลื่อนไหวเซสชัน (เช่น เมื่อล็อกอินสำเร็จ หรือล็อกเอาต์หลุดระบบ) ควรเลือกรับสัญญาณจากคำสั่งใดของ Supabase?
ก. `supabase.auth.getSession()`  
ข. `supabase.auth.onAuthStateChange()`  
ค. `supabase.auth.watchSession()`  
ง. `supabase.auth.getUser()`  

### ข้อ 6: เพราะเหตุใด AsyncStorage จึงถูกมองว่ามีความปลอดภัยต่ำและไม่ควรนำมาใช้เป็นที่เก็บโทเค็นยืนยันตัวตนหลักในโมบายแอปพลิเคชันระดับโปรดักชัน?
ก. เนื่องจาก AsyncStorage เก็บข้อมูลเป็นไฟล์ข้อความธรรมดา (Plaintext JSON) ที่ไม่มีการเข้ารหัสความปลอดภัยระดับเนทีฟ ทำให้อุปกรณ์ที่ถูก Root/Jailbreak สามารถดึงโทเค็นออกไปสวมสิทธิ์ได้ง่าย  
ข. เนื่องจาก AsyncStorage ถูกจำกัดความจุเพียง 10 ตัวอักษรเท่านั้น  
ค. เพราะคำสั่ง AsyncStorage จะลบข้อมูลทิ้งโดยอัตโนมัติทันทีที่ปิดหน้าจอแอปพลิเคชัน  
ง. เนื่องจาก AsyncStorage รองรับเฉพาะการทำงานในระบบ iOS เท่านั้น  

### ข้อ 7: ไลบรารีใดของ Expo ที่มีหน้าที่ช่วยเข้ารหัสและจัดเก็บรหัสกุญแจสำคัญลงสู่ Keychain/Keystore ของอุปกรณ์มือถือในระดับฮาร์ดแวร์ได้อย่างเสถียรปลอดภัย?
ก. `expo-secure-storage`  
ข. `expo-safe-store`  
ค. `expo-secure-store`  
ง. `expo-keychain`  

### ข้อ 8: การต่อเชื่อมให้ Supabase Client บังคับหันมาบันทึกและจัดการโทเค็นล็อกอินผ่าน Secure Store แทน AsyncStorage จะต้องส่งอ็อบเจกต์ Adapter เข้าผ่านพร็อพเพอร์ตี้ใดในตอนประกาศตัวสร้าง `createClient`?
ก. `auth.storage`  
ข. `auth.secureStore`  
ค. `auth.adapter`  
ง. `auth.persistSession`  

### ข้อ 9: ในกรณีที่เราสร้างแอปพลิเคชันขนาดเล็กแบบ Single-page โดยใช้สเตตล็อกอินในหน้านั้นหน้าเดียว วิธีการสลับหน้าจอระหว่าง "หน้าฟอร์มเข้าสู่ระบบ" และ "หน้าแผงควบคุม Dashboard" ใช้วิธีใดที่ง่ายและกระชับที่สุด?
ก. การเปิดกล่อง Modal ทับไว้ตลอดเวลา  
ข. การใช้งาน Conditional Rendering ผ่านการดักตัวแปรเซสชัน เช่น `session ? <Dashboard /> : <LoginForm />`  
ค. การบังคับยิงคำสั่งเปลี่ยน Router ดีดข้ามโฟลเดอร์หน้าจอโดยออโต้  
ง. การสลับหน้าจอด้วยการเปลี่ยนพิกัดความกว้างในสไตล์ชีท  

### ข้อ 10: หากต้องการขยายแอปพลิเคชันให้ใหญ่ขึ้น (Multi-page) และประสงค์จะแชร์เซสชันผู้ใช้อวาตาร์รวมถึงสิทธิ์การเข้าถึงให้ทุกหน้าจอรับรู้ค่าตรงกันทันทีโดยไม่ต้องส่งพร็อพส์ต่อกันเป็นทอด ๆ (Prop Drilling) ข้อใดคือสถาปัตยกรรม React ที่เหมาะสมที่สุดที่จะนำมาประยุกต์ใช้เป็นของแถมพิเศษ?
ก. การทำลูปสตรีมดักในปุ่มกดของทุกหน้าจอ  
ข. การใช้งาน React Context API ผ่านการทำ `useContext` และ `AuthContext.Provider`  
ค. การนำข้อมูลเซสชันไปจัดบันทึกฝากไว้ในตัวแปร Global Windows Node  
ง. การอัดบันทึกพิกัดตำแหน่งลงใน API พิกัดตำแหน่ง GPS

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** ข้อจำกัดขนาดกล่องเป็นเรื่องของ CSS/Flexbox และระบบทั้งสองมีบทบาทและการจัดเก็บคีย์ที่แตกต่างกันอย่างสิ้นเชิง
  * **ข. ถูก:** Authentication คือด่านระบุตัวตนบุคคล (เช่น การกรอกฟอร์มเพื่อแสดงตัวว่าเป็นเจ้าของบัญชีจริง) ส่วน Authorization คือการเช็คว่าบัญชีที่ระบุตนเข้ามาแล้วนั้นมีสิทธิ์ระดับไหนในการอ่านเขียนแก้ไขแถวข้อมูล (เช่น กฎ RLS ควบคุมตาราง wishlist หลังบ้าน)

### เฉลย ข้อ 2: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** Supabase พัฒนาขึ้นมาโดยใช้ PostgreSQL เป็นเบสฐานข้อมูล แต่ตัวที่ทำหน้าที่รันจัดการ Session/Auth โดยตรงคือ GoTrue Service
  * **ค. ถูก:** GoTrue คือระบบ API เบื้องหลังที่ทำหน้าที่ควบคุม จัดสรร และระบายโทเค็นความปลอดภัยของตารางข้อมูลผู้ใช้งานบน Supabase Auth

### เฉลย ข้อ 3: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** คำสั่งอื่นไม่มีระบุไว้ในไวยากรณ์การประกาศฟังก์ชันของจาวาสคริปต์ Supabase Client
  * **ข. ถูก:** ไวยากรณ์สมัครสมาชิกใช้งานของ Supabase Auth คือคำสั่ง Asynchronous ชื่อ **`supabase.auth.signUp()`**

### เฉลย ข้อ 4: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ในอดีตเคยมี `signIn()` แต่ปัจจุบันใน Supabase JS client v2 ได้ปรับเปลี่ยนมาใช้คำสั่งที่ระบุวิธีการล็อกอินชัดเจนขึ้นเพื่อแยกแยะการใช้เมลกับรหัสผ่าน
  * **ข. และ ง. ผิด:** คำสั่งไม่มีระบุใน API  
  * **ค. ถูก:** คำสั่งที่ถูกต้องในการลงชื่อเข้าใช้ด้วยชุดเมลรหัสผ่านคือ **`supabase.auth.signInWithPassword()`**

### เฉลย ข้อ 5: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `getSession()` คืนค่าเซสชันของจังหวะนั้น ๆ ครั้งเดียว ไม่ได้รับฟังสัญญาณเรียลไทม์
  * **ข. ถูก:** **`supabase.auth.onAuthStateChange()`** ทำหน้าที่เฝ้าดักฟังและพ่นสัญญาณ Callback ออกมาทุกครั้งที่มีการเปลี่ยนผ่านสเตตล็อกอิน ล็อกเอาต์ หรือการเปลี่ยนเปลี่ยนโทเค็น
  * **ค. และ ง. ผิด:** คำสั่งระบุผิดวัตถุประสงค์ หรือไม่มีในสารระบบ Auth API ของ Supabase

### เฉลย ข้อ 6: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** AsyncStorage มีความปลอดภัยต่ำเพราะเก็บข้อมูลเป็นตัวอักษรดิบที่แกะดูได้ง่ายบนอุปกรณ์ที่ผ่านการแฮกหรือเข้าถึง root หากมีข้อมูลสำคัญอย่างโทเค็นล็อกอิน ก็เสี่ยงที่จะถูกขโมยสิทธิ์
  * **ข. ค. และ ง. ผิด:** AsyncStorage เก็บข้อมูลได้ไม่จำกัดสิทธิ์ (ขีดจำกัดปกติคือ 6MB) และข้อมูลไม่ถูกลบอัตโนมัติเมื่อปิดแอป รวมถึงสามารถรันได้ทั้งบน Android และ iOS

### เฉลย ข้อ 7: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** ชื่อสะกดไม่ถูกต้องตามหลักไลบรารีของระบบ Expo
  * **ค. ถูก:** ไลบรารีอย่างเป็นทางการในการเข้ารหัสและจัดเก็บคีย์ลับลง Keychain/Keystore บนระดับเนทีฟของ Expo คือ **`expo-secure-store`**

### เฉลย ข้อ 8: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** ในการประกาศสร้าง client ตัวเลือกสำหรับจัดส่ง Storage ตัวใหม่คือระบุเข้าที่พร็อพเพอร์ตี้ **`auth.storage`** ภายใต้ออบเจกต์ config ของ `createClient`
  * **ข. ค. และ ง. ผิด:** ตัวเลือกชื่ออื่นสะกดผิดไวยากรณ์การรับพารามิเตอร์ของ Supabase

### เฉลย ข้อ 9: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** วิธีอื่นระบุคำสั่งควบคุมซับซ้อน หรือส่งผลเสียต่อการรันสเตตของ React
  * **ข. ถูก:** การทำ **Conditional Rendering** ด้วยการประเมินค่า `session ? <Dashboard /> : <LoginForm />` เป็นแนวทางที่ดีและสั้นที่สุดในการสลับภาพหน้าจอในหน้าเดียว โดยไม่ต้องรันจัดการเรื่อง Navigation หลายระดับ

### เฉลย ข้อ 10: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** วิธีการเหล่านั้นไม่ใช่การแชร์สเตตที่เป็นระเบียบตามมาตรฐาน หรือทำให้เกิดผลเสียต่อ Memory Leak
  * **ข. ถูก:** **React Context API** (`useContext` และ `Provider`) คือสถาปัตยกรรมของ React ที่ดีที่สุดในการจัดแชร์สเตตตรงส่วนกลาง เพื่อแชร์ข้อมูลลงสู่หน้าต่างและคอมโพเนนต์ลูกหลานทั้งหมดโดยตรง
