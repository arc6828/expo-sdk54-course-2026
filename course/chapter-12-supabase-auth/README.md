# บทที่ 12: ระบบยืนยันตัวตนและการเข้าถึงข้อมูลด้วย Supabase Authentication (Overview)

ในบทเรียนนี้ ผู้เรียนจะได้ศึกษาหลักการออกแบบระบบความปลอดภัยและการระบุตัวตนผู้ใช้ในโมบายแอปพลิเคชัน โดยเชื่อมต่อร่วมกับ **Supabase Authentication** (GoTrue Service) ผู้เรียนจะได้เรียนรู้กระบวนการสมัครสมาชิก (Sign Up), ล็อกอินด้วยอีเมลพาสเวิร์ด (Sign In with Email/Password), ตลอดจนการเฝ้าดักฟังสถานะด้วย `onAuthStateChange` เพื่อสลับการเรนเดอร์หน้าจอตามความถูกต้องของเซสชัน คลุมทับความมั่นคงปลอดภัยในการเข้ารหัสและเก็บรักษา Access Tokens ในระดับอุปกรณ์ด้วย **`expo-secure-store`** และปิดท้ายด้วย **แนวเนื้อหารีแฟกเตอร์พิเศษ (Bonus Section)** เพื่อยกระดับสถาปัตยกรรมสู่ศูนย์การจัดการข้อมูลผ่าน **`useContext`** ในระบบของ Expo Router

## 📂 สารบัญเอกสารประจำบทเรียน (Chapter Documents)
คุณสามารถเลือกอ่านและเข้าถึงไฟล์เอกสารต่าง ๆ ในโฟลเดอร์บทเรียนนี้ได้โดยตรง:

1. 📋 **[แผนการสอนประจำบท (01-lesson-plan.md)](./01-lesson-plan.md)**  
   หัวข้อการสอน วัตถุประสงค์เชิงพฤติกรรม กิจกรรมการสร้างแอปพลิเคชันยืนยันตัวตนและวิธีกรอกฟอร์มที่ปลอดภัย สื่อประกอบการสอน และหัวข้อการประเมินวัดผล
2. 📖 **[เนื้อหาหลักการเรียนรู้ (02-content.md)](./02-content.md)**  
   เนื้อหาหลักภาคทฤษฎีอธิบายแนวคิด Authentication vs Authorization, กลไกของโทเค็น (JWT), เมธอดการสมัครและลงลงชื่อเข้าใช้, วิธีใช้งาน `onAuthStateChange` ในการจัดการ Lifecycle, เหตุผลที่ห้ามใช้ AsyncStorage และตัวอย่างการใช้งาน `expo-secure-store` ร่วมกับ Supabase Client และของแถมการใช้ `useContext`
3. 📝 **[แบบทดสอบท้ายบทเรียน (03-quiz.md)](./03-quiz.md)**  
   แบบทดสอบปรนัยทบทวนความเข้าใจ 10 ข้อ ภาษาไทยเจาะลึกระบบยืนยันตน ความปลอดภัยของ Secure Store วิธีแก้ไขปัญหาโทเค็นรั่วไหล และสถาปัตยกรรม React Context พร้อมเฉลยและคำอธิบายโดยละเอียดครบทุกตัวเลือก
4. 💻 **[คู่มือปฏิบัติการขั้นพื้นฐาน (04-lab.md)](./04-lab.md)**  
   โจทย์ปฏิบัติการสร้างแอปพลิเคชัน **"สมุดบันทึกรับล็อกสิทธิ์ความปลอดภัยส่วนตัว (Secure Personal Diary)"** เพื่อฝึกฝนเขียนรหัสแบบฟอร์มล็อกอิน สมัครสมาชิก รันตัวจำหน้าล็อกอินผ่าน `expo-secure-store` และการ Refactor สู่ระบบ `AuthContext` ร่วมกับ Route Guard ของ Expo Router พร้อมโค้ดเฉลยฉบับสมบูรณ์ (Complete Solution)

## 🛠️ สรุปโปรเจคปฏิบัติการท้ายบทเรียน (Lab Project Summary)
ในปฏิบัติการของบทนี้ ผู้เรียนจะได้บูรณาการระบบล็อกอินและการจำกัดสิทธิ์หน้าจอด้วยแนวทางดังนี้:
* **Form Validation Guard:** การใช้ตัวแปรกั้นความถูกต้องของฟอร์มอีเมลและรหัสผ่านขั้นต่ำ 6 อักษรก่อนยิงส่งข้อมูลขึ้นคลาวด์
* **Secure Storage Adapter:** นำ `expo-secure-store` ไปเสียบเป็น adapter ใน config ของ Supabase Client เพื่อทดแทน AsyncStorage และเพิ่มความปลอดภัยระดับเนทีฟ (Keychain/Keystore)
* **Conditional Auth Screen:** การจัดหน้าแอปให้แสดง LoginForm หรือ Dashboard แดชบอร์ดบันทึกตามพารามิเตอร์เซสชันปัจจุบันผ่านสเตตสตรีม
* **Global AuthContext (Bonus Refactoring):** เรียนรู้การเขียน AuthContext.Provider และจัดฟังก์ชัน useAuth() ดักจับความปลอดภัยหน้าจอ Layout Guard ข้ามกลุ่มโฟลเดอร์ของ Expo Router สำหรับการขยายแอปพลิเคชัน
