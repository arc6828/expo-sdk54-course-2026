# บทที่ 5: การแสดงผลรายการข้อมูลและประสิทธิภาพการทำงาน (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- ข้อจำกัดในเรื่องหน่วยความจำและประสิทธิภาพการเรนเดอร์ของคอมโพเนนต์ `ScrollView` เมื่อทำงานกับปริมาณข้อมูลขนาดใหญ่
- กลไกการทำงานของคอมโพเนนต์ `FlatList` เช่น ระบบ Virtualization (การสร้างเฉพาะส่วนที่แสดงบนจอ) และการคืนสภาพมุมมอง (Item View Recycling)
- พร็อพส์พื้นฐานและจำเป็นระดับแกนหลักในการตั้งค่า `FlatList` (`data`, `renderItem`, `keyExtractor`)
- การดึงข้อมูลจากภายนอกผ่าน REST API เบื้องต้นด้วย Fetch API ร่วมกับ `useEffect` เพื่อป้อนเป็นแหล่งข้อมูลให้กับ `FlatList`
- การเขียนตกแต่งรายละเอียดเสริมของตารางรายการข้อมูลด้วย `ListEmptyComponent`, `ListHeaderComponent`, `ListFooterComponent` และ `ItemSeparatorComponent`
- การเชื่อมต่อฟีเจอร์ดึงหน้าจอลงเพื่อรีเฟรชข้อมูลล่าสุด (Pull-to-Refresh) ด้วยพร็อพส์ `refreshing` และ `onRefresh`
- การทำระบบโหลดข้อมูลเพิ่มเติมอัตโนมัติเมื่อเลื่อนลงด้านล่างสุดของรายการ (Infinite Scroll) ด้วย `onEndReached` และ `onEndReachedThreshold`
- แนวคิดความเข้าใจเบื้องต้นและการจัดข้อมูลกลุ่มประเภทหัวข้อย่อยด้วยคอมโพเนนต์ `SectionList`

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถชี้แจงความแตกต่างเชิงประสิทธิภาพและกรณีใช้งานที่เหมาะสมระหว่าง `ScrollView` และ `FlatList` ได้ถูกต้อง
2. ผู้เรียนสามารถระบุและเขียนส่งตัวแปรพร็อพส์หลักของ `FlatList` เพื่อวาดรายการข้อมูลได้สำเร็จโดยไม่มีปัญหาเรื่องกุญแจจำแนกชิ้นงาน (`key`)
3. ผู้เรียนสามารถเขียนคำสั่ง Fetch API ร่วมกับ `useEffect` เพื่อเรียกข้อมูลจากแหล่งข้อมูลออนไลน์ (เช่น JSONPlaceholder) มาแสดงผลบน `FlatList` ได้สำเร็จ
4. ผู้เรียนสามารถประยุกต์ใช้คอมโพเนนต์เสริมตกแต่งส่วนประกอบขอบลิสต์ (Header, Footer, Divider) และการจัดทำกรณีลิสต์ว่างได้อย่างมีหลักการ
5. ผู้เรียนสามารถเขียนจำลองสเตตและฟังก์ชันดึงข้อมูลใหม่เพื่อทำ Pull-to-Refresh แสดงไอคอนหมุนโหลดข้อมูลสำเร็จ
6. ผู้เรียนสามารถเรียกใช้และอธิบายหลักการทำงานของ `onEndReached` ในการโหลดข้อมูลหน้าถัดไป (Pagination) ได้ถูกต้อง
7. ผู้เรียนสามารถสร้างแบบจำลองรายการแบบกลุ่มโครงสร้างผ่าน `SectionList` ได้อย่างเหมาะสม

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายประกอบการเปรียบเทียบเชิงลึก (Theoretical Performance Analysis):** บรรยายการทำงานเบื้องหลังของ React Native VirtualizedList และจัดทำการสาธิตเปรียบเทียบ RAM Usage ระหว่าง ScrollView และ FlatList เมื่อโหลดอาร์เรย์ 2,000 รายการ
2. **การทำ Live Coding Demonstration หัวข้อ REST API FlatList:** ผู้สอนสาธิตการเชื่อมโยงเครือข่ายดึงข้อมูลโพสต์จำลองหรือรูปภาพจำลองจาก JSONPlaceholder REST API ผ่านคำสั่ง `fetch` นำเข้าแสดงผลใน `FlatList`
3. **กิจกรรมปฏิบัติเดี่ยวฝึกความจำ (Advanced Layout Implementation):** มอบหมายโจทย์ให้ผู้เรียนปรับปรุงรายการแอปที่มีอยู่เดิม โดยเพิ่ม Header แสดงโลโก้ และสร้าง Empty Component หน้าตาสวยงามที่จะโชว์เมื่อข้อมูลถูกล้างจนหมดแถว
4. **กิจกรรมประยุกต์ทำกลุ่มประเมินผลการจัดกลุ่ม (SectionList Grouping Activity):** แบ่งกลุ่มศึกษาโค้ดโครงสร้างเมนูอาหารที่จำแนกหมวดหมู่ (เช่น ของคาว ของหวาน เครื่องดื่ม) โดยเรียกนำเสนอผลลัพธ์ผ่านคอมโพเนนต์ `SectionList`

## สื่อการเรียนการสอน
1. สไลด์การเรียนการสอนวิชา "Lists Rendering and Core Optimizations in React Native"
2. ชุดข้อมูลอาร์เรย์จำลองของรายชื่อภาพยนต์หรือรายชื่อผู้ติดต่อจำนวน 500 รายการที่มีโครงสร้าง ID และข้อมูลสเตตัส
3. ตัวอย่างการตรวจสอบ FPS (Frame Per Second) ด้วย Performance Tools ของเครื่องจำลองสมาร์ทโฟน
4. ซอร์สโค้ดต้นแบบระบบสินค้าคงคลัง (Inventory Manager Demo) ที่ใช้ `FlatList` และ `SectionList`

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 วัดความเข้าใจผ่านแบบทดสอบปรนัยท้ายบทเรียน 10 ข้อ
1.2 ตรวจสอบทักษะการเขียนโปรแกรมและการเลือกใช้งาน `FlatList` แทน `ScrollView` ให้เหมาะกับลักษณะขนาดงานข้อมูล
1.3 ตรวจประเมินโค้ดแล็บในการใส่ฟังก์ชันรีเฟรช การจัดแบ่งแยกประเภทคีย์ และความสามารถในการโหลดข้อมูลหน้าถัดไป (Infinite Scroll)

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถเขียนส่งชิ้นงานแล็บแสดงผลรายการข่าวสารดึงข้อความรีเฟรชได้ลื่นไหลและรันผ่านโดยไม่มีคำแจ้งเตือนบั๊กเรื่อง Key ได้สำเร็จตามเกณฑ์ (ผ่าน/ไม่ผ่าน)
2.3 พฤติกรรมการรับฟังข้อเสนอแนะ ความพยายามแก้ไขอิมพลิเมนต์ปุ่มดาวน์โหลดข้อมูลเพิ่มเติมด้วยสเตตจนผ่านเกณฑ์
