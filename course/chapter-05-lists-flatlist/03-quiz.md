# แบบทดสอบท้ายบทเรียน บทที่ 5: การแสดงผลรายการข้อมูลและประสิทธิภาพการทำงาน (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความเข้าใจเกี่ยวกับปัญหาประสิทธิภาพเลื่อนไหล (Scrolling Performance) ข้อแตกต่างระหว่าง `ScrollView` และ `FlatList` โครงสร้างและพร็อพส์หลักในการควบคุมรายการ การเชื่อมต่อเน็ตเวิร์กดึงข้อมูลผ่าน API รวมถึงระบบ Pull-to-Refresh, Infinite Scroll และการจัดรูปเล่มประเภทกลุ่มด้วย `SectionList` ใน React Native Expo SDK 54

---

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: ข้อใดคือสาเหตุหลักที่ทำให้อินเทอร์เฟซเกิดอาการหน่วง เฟรมเรตตก หรือแอปพลิเคชันค้างเด้งปิดตัว (Crash) เมื่อใช้งาน `ScrollView` แสดงผลข้อมูลจำนวนมาก?
ก. `ScrollView` ล็อกไม่ให้อุปกรณ์ดึงสัญญานอินเทอร์เน็ต  
ข. `ScrollView` ทำการเรนเดอร์และสร้างวัตถุลูกทั้งหมดลงสู่หน่วยความจำ (RAM) พร้อมกันในทันที แม้ว่าชิ้นส่วนข้อมูลจะยังไม่แสดงบนหน้าจอจริงก็ตาม  
ค. `ScrollView` ห้ามเขียนจัดตำแหน่งบล็อกสีด้วยระบบ Flexbox  
ง. `ScrollView` ไม่รองรับการแสดงผลกล่องอินพุต `TextInput`  

### ข้อ 2: กลไกแบบ Virtualization ที่ใช้งานภายในคอมโพเนนต์ `FlatList` ช่วยเพิ่มระดับประสิทธิภาพและประหยัดพื้นที่ RAM ได้อย่างไร?
ก. ย้ายเนื้อหางานคำนวณทั้งหมดส่งไปให้คอมพิวเตอร์หลังบ้านประมวลผลแทน  
ข. แสดงผลเฉพาะชิ้นข้อมูลรายการที่กำลังอยู่ในขอบข่ายขนาดหน้าจอจริง และทำลาย/รียูสกล่องจัดแสดงของรายการที่หลุดหน้าจอออกไป  
ค. ซ่อนข้อมูลทั้งหมดไว้ในเครื่องมือ AsyncStorage ตลอดเวลา  
ง. ลดขนาดความละเอียดของภาพที่โหลดให้กลายเป็นภาพขาวดำอัตโนมัติ  

### ข้อ 3: พร็อพส์ (Props) ใดต่อไปนี้มีหน้าที่สำคัญในการเป็นตัวแปลงข้อมูลวัตถุอาร์เรย์ให้ออกมาเป็น JSX คอมโพเนนต์การ์ดแสดงผลบนหน้าจอลิสต์?
ก. `data`  
ข. `renderItem`  
ค. `keyExtractor`  
ง. `ItemSeparatorComponent`  

### ข้อ 4: ในฟังก์ชันการดึงค่าของ `renderItem` ข้อใดระบุรูปแบบพารามิเตอร์แบบกระจายออบเจกต์ (Destructuring) ที่ถูกต้องสำหรับการสกัดข้อมูลไอเท็มปัจจุบันมาแสดง?
ก. `renderItem={(item) => <Text>{item.title}</Text>}`  
ข. `renderItem={({ item }) => <Text>{item.title}</Text>}`  
ค. `renderItem={(title) => <Text>{title}</Text>}`  
ง. `renderItem={({ data }) => <Text>{data.title}</Text>}`  

### ข้อ 5: เหตุใดผู้พัฒนาแอปจึงสมควรใส่ใจกำหนดพร็อพส์ `keyExtractor` ให้ส่งค่าสตริงที่เป็นเอกลักษณ์ (Unique String Key) ให้แก่ FlatList ทุกครั้ง?
ก. เพื่อบังคับให้อุปกรณ์ระบุขนาดของรูปภาพในรายการโดยอัตโนมัติ  
ข. เพื่อให้ React สามารถตรวจเช็คสถานะการเปลี่ยนแปลงตำแหน่ง ย้ายสลับ หรือลบรายการได้อย่างถูกต้องแบบเจาะจงเฉพาะชิ้นงาน โดยไม่ต้องเสียเวลาวาดใหม่ทั้งหมด  
ค. เพื่อจำกัดไม่ให้ปุ่มสัมผัสทำงานได้เมื่อคีย์บอร์ดปรากฏขึ้น  
ง. เพื่อระบุตัวเลขจำนวนหน้าสำหรับดาวน์โหลดข้อมูลเพิ่มเติม  

### ข้อ 6: เมื่อต้องการดึงรายการโพสต์หรือรูปภาพจาก RESTful API ทันทีที่หน้าจอลิสต์เริ่มต้นทำงาน (Mount) ใน React Native ข้อใดคือแนวทางปฏิบัติมาตรฐาน?
ก. เรียกใช้คำสั่ง `fetch()` ในสไตล์ชีทภายนอก  
ข. เขียนฟังก์ชัน `fetch()` ครอบไว้ภายใน `useEffect` Hook ที่มีอาเรย์การพึ่งพาเป็นค่าว่าง (`[]`) เพื่อให้สัญญานอินพุตทำงานเพียงรอบเดียว  
ค. เรียกฟังก์ชันเน็ตเวิร์กภายใน Property ของ `keyExtractor`  
ง. ใช้ `useState` ในการดักจับค่า IP ของโทรศัพท์แบบวนซ้ำ  

### ข้อ 7: พร็อพส์ใดของ `FlatList` ที่ทำหน้าที่แสดงคอมโพเนนต์อื่นทดแทน (เช่น ข้อความแจ้งว่าไม่มีรายการของ) เมื่อพบว่าไม่มีข้อมูลอยู่ในอาเรย์เลย?
ก. `ListFooterComponent`  
ข. `ListHeaderComponent`  
ค. `ListEmptyComponent`  
ง. `ItemSeparatorComponent`  

### ข้อ 8: การเปิดความสามารถให้ผู้ใช้ลากขอบจอลงด้านล่างสุดเพื่อสั่งดึงข้อมูลชุดใหม่ (Pull-to-Refresh) ใน FlatList จะต้องทำรายการตั้งค่าพร็อพส์คู่ใดร่วมกัน?
ก. `refreshing` และ `onRefresh`  
ข. `onEndReached` และ `onEndReachedThreshold`  
ค. `active` และ `onActive`  
ง. `secureTextEntry` และ `onChangeText`  

### ข้อ 9: พร็อพส์ `onEndReached` ในคอมโพเนนต์ `FlatList` มีพฤติกรรมการทำงานหลักอย่างไร?
ก. ทำการตรวจสอบสิทธิข้อมูลเมื่อผู้ใช้นิ้วเลื่อนออกจากปุ่มล้างค่า  
ข. เรียกฟังก์ชันสั่งการดาวน์โหลดข้อมูลหน้าถัดไป (Infinite Scroll) ทันทีเมื่อผู้ใช้งานใช้นิ้วเลื่อนหน้าจอลิสต์ลงไปจนเกือบถึงส่วนท้ายสุดของตารางรายการ  
ค. ล้างสเตตทั้งหมดกลับเป็นค่าเริ่มต้นเมื่อลิสต์แสดงผลเสร็จ  
ง. เคลื่อนย้ายโครงสร้างหัวข้อของลิสต์ให้อยู่ชิดปุ่มล่างสุดของขอบอุปกรณ์  

### ข้อ 10: หากต้องการจัดเลย์เอาต์แสดงผลที่แยกข้อมูลออกเป็นหมวดหมู่ (เช่น คลังเครื่องดื่ม, คลังของหวาน) โดยต้องการให้มีหัวข้อกลุ่มลอยตัวค้างไว้เวลาเลื่อน (Sticky Headers) คอมโพเนนต์ใดตอบโจทย์งานลักษณะนี้ได้ดีที่สุด?
ก. `ScrollView`  
ข. `FlatList`  
ค. `SectionList`  
ง. `SafeAreaView`

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `ScrollView` ไม่ได้บล็อกการดึงเครือข่ายของระบบ
  * **ข. ถูก:** `ScrollView` ไม่มีระบบหน่วยความจำแบบไดนามิก มันจะประมวลผลแปลง JSX ทุกชิ้นข้างใน (Eager Rendering) ไม่ว่าจะล้นจอไปยาวแค่ไหน ทำให้เปลือง RAM ทวีคูณตามสัดส่วนรายการข้อมูล
  * **ค. และ ง. ผิด:** การเขียนจัดตำแหน่งและใส่อินพุตทำได้ตามรูปแบบทั่วไป

### เฉลย ข้อ 2: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** งานเรนเดอร์ UI จะทำบนอุปกรณ์ปลายทางเสมอ ไม่ใช่หลังบ้าน
  * **ข. ถูก:** `FlatList` ยึดหลักเรนเดอร์ขอบเขตเฉพาะส่วนที่มองเห็นจริง และทำการปล่อย RAM คืนระบบเมื่อวัตถุเลื่อนหลุดสายตา พร้อมทั้งดึงเฟรม View ที่ว่างเหล่านั้นกลับมารอรับสวมข้อมูลชิ้นถัดไป (Recycling) ส่งผลให้มีความคงทนสูงและประหยัด RAM
  * **ค. และ ง. ผิด:** AsyncStorage มีไว้เก็บบันทึกถาวร และระบบไม่มีการสุ่มเปลี่ยนภาพเป็นขาวดำ

### เฉลย ข้อ 3: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `data` เป็นเพียงคลังอาร์เรย์ข้อเท็จจริง (ข้อมูลดิบ) ของโพสต์ แต่ยังไม่มีโครงสร้างสไตล์ภาพข้อความ
  * **ข. ถูก:** พร็อพส์ `renderItem` ทำหน้าที่รับฟังก์ชันที่ผู้พัฒนาอธิบายสไตล์ไว้ เพื่อนำข้อมูลดิบในแต่ละดัชนี (Index) มาวาดแปลงให้กลายเป็นโครงสร้าง JSX เพื่อแสดงขึ้นจอจริง
  * **ค. ผิด:** `keyExtractor` ทำหน้าที่กำหนดระบุข้อความกุญแจส่วนตัว ไม่ใช่ผู้วาด UI
  * **ง. ผิด:** `ItemSeparatorComponent` ใช้สำหรับวาดเส้นแบ่งเส้นคั่นเท่านั้น

### เฉลย ข้อ 4: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** การเขียนพารามิเตอร์โดยไม่มีปีกกาอาร์กิวเมนต์ `{}` หรือสับสนชื่อคีย์ จะนำเอาอ็อบเจกต์รวมของ MetaData (ซึ่งประกอบด้วยตัวชิ้นงาน, ลำดับ Index, ข้อมูลภายนอก) มาใช้ตรง ๆ ส่งผลให้เกิดอาการค่าเป็น `undefined` หรือโปรแกรมล่ม
  * **ข. ถูก:** พารามิเตอร์ที่ `renderItem` ส่งมาจะมีรูปเป็นออบเจกต์ `{ item, index, separators }` การใส่ปีกกาสกัดตัวแปรคีย์ชื่อ `item` (`{ item }`) จึงเป็นวิธีที่ถูกต้องดั้งเดิมในการเอาวัตถุเดี่ยวแต่ละดัชนีมาใช้งาน

### เฉลย ข้อ 5: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ขนาดของรูปภาพต้องถูกตั้งผ่าน CSS Style เท่านั้น
  * **ข. ถูก:** ในสถาปัตยกรรม React การใช้คีย์เฉพาะช่วยประหยัดเวลาอย่างมาก เพราะถ้าเกิดการอัปเดตข้อมูลขึ้นมาจุดใดจุดหนึ่ง React จะดึงสตริงกุญแจตรวจสอบเพื่อประมวลผลขยับเฉพาะกล่อง View นั้นโดยไม่ต้องเรนเดอร์ใหม่หมดทั้งแผง ซึ่งเพิ่มความลื่นไหลเป็นอย่างมาก
  * **ค. และ ง. ผิด:** ไม่ได้ควบคุมการทำงานปุ่ม และไม่ได้มีขอบข่ายเกี่ยวกับเลขหน้าข้อมูลดั้งเดิม

### เฉลย ข้อ 6: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** สไตล์ชีทมีหน้าที่จัดสเปซเฉดสีเท่านั้น ห้ามเรียกนำข้อมูลเน็ตเวิร์ก
  * **ข. ถูก:** การประยุกต์ใช้งาน `useEffect` พร้อมกำหนด Dependency Array ว่างเปล่า `[]` จะสั่งให้อุปกรณ์เชื่อมต่อทำงานส่งคำขอ Fetch API เพียงแค่ครั้งเดียวตอนโหลดหน้าจอขึ้นมา (Component Did Mount) ซึ่งช่วยบันทึกข้อมูลสเตตและเตรียมนำไปใช้ป้อนต่อ FlatList ได้อย่างมีประสิทธิภาพ
  * **ค. และ ง. ผิด:** คีย์เฉพาะและการทำสเตตไอพีไม่ได้เอื้อต่อการดึงค่า API โพสต์

### เฉลย ข้อ 7: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. และ ข. ผิด:** ใช้ตกแต่งส่วนบนและส่วนล่างสุดของลิสต์ตาราง
  * **ค. ถูก:** `ListEmptyComponent` คอยตรวจเช็คความยาวอาร์เรย์ใน `data` หากข้อมูลมีค่าว่าง จะเรนเดอร์คอมโพเนนต์อื่นที่เราส่งมอบเข้าไปแทน เช่น ภาพการ์ตูนกล่องเปล่าพร้อมคำเตือน "ไม่พบข้อมูลใด ๆ ในขณะนี้"
  * **ง. ผิด:** ใช้สำหรับคั่นระหว่างชิ้นงาน

### เฉลย ข้อ 8: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** เพื่ออำนวยฟังก์ชันกวาดดึงหน้าจอ (Pull-to-Refresh) เราต้องใช้ `refreshing` (สเตตคอยจำสถานะการดึงประมวลผล) ควบคู่กับเหตุการณ์ `onRefresh` (ฟังก์ชันสั่งยิงเรียกเน็ตเวิร์กอัปเดตใหม่)
  * **ข. ผิด:** เป็นคู่สำหรับระบุงานโหลดเพิ่มตอนท้ายจอ (Infinite Scroll)
  * **ค. และ ง. ผิด:** ไม่ใช่คำสั่งที่อยู่ในขอบข่ายงานดึงหมุนรีเฟรชหน้าต่าง

### เฉลย ข้อ 9: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** พฤติกรรมไม่ใช่การเคลียร์ปุ่ม ล้างสเตต หรือย้ายหัวข้อขยับขอบจอ
  * **ข. ถูก:** เหตุการณ์ `onEndReached` จะถูกส่งสัญญาณเมื่อผู้ใช้นิ้วเลื่อนตารางลงมาจนเข้าเกณฑ์ขอบล่าง เพื่อส่งสัญญาณทำ Callback ไปเรียกหน้าถัดไป (เช่น หน้า 2, 3) มาประกอบลงสเตตเดิมเพื่อสร้างลิสต์แบบต่อเนื่องยาวไม่มีที่สิ้นสุด

### เฉลย ข้อ 10: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. และ ข. ผิด:** ScrollView เปลือง RAM และไม่มี Sticky Headers ออโต้ ส่วน FlatList ออกแบบมาสำหรับแถวเรียงเส้นตรง ไม่ได้สนับสนุนการแยกกลุ่มและป้าย Sticky ประจำเซกชันโดยตรง
  * **ค. ถูก:** `SectionList` ดีไซน์ขึ้นมาแยกย่อยข้อมูลกลุ่มประเภทหัวข้อ (Sections) โดยความสามารถหลักคือมีระบบตรึงแถบหัวกลุ่มค้างไว้ (Sticky Section Headers) บนขอบจอยามเลื่อนผ่าน ซึ่งอำนวยความสวยงามสไตล์ระดับเนทีฟยอดนิยม
  * **ง. ผิด:** มีหน้าที่ป้องกันเนื้อหาชนขอบบาก ไม่เกี่ยวกับการคำนวณลิสต์กลุ่ม
