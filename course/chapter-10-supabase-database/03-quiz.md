# แบบทดสอบท้ายบทเรียน บทที่ 10: การใช้งานคลาวด์ดาต้าเบสด้วย Supabase (Quiz)

แบบทดสอบนี้มีวัตถุประสงค์เพื่อประเมินความรู้และความเข้าใจเกี่ยวกับแนวคิดระบบหลังบ้านสำเร็จรูป (BaaS), การลงทะเบียนเชื่อมต่อ Supabase Client ในสภาพแวดล้อม Expo SDK 54, การเขียนคำสั่งสืบค้นและคัดกรองข้อมูล (SELECT), การเพิ่มแถวข้อมูล (INSERT), การแก้ไขค่าข้อมูล (UPDATE), การลบข้อมูล (DELETE) และสุนทรียภาพในการจัดการข้อผิดพลาดความปลอดภัย

---

## คำถามประจำบทเรียน (10 ข้อ)

### ข้อ 1: ข้อใดคือบทบาทหน้าที่ที่ถูกต้องที่สุดของสถาปัตยกรรม Backend-as-a-Service (BaaS) เช่น Supabase?
ก. ทำการอัปเกรดความเร็วในการประมวลผล CPU และ RAM บนสมาร์ทโฟนของผู้ใช้งานโดยตรง  
ข. จัดสรรระบบบริการหลังบ้านสำเร็จรูป เช่น ฐานข้อมูล ระบบยืนยันตัวตน และคลังเก็บไฟล์บนระบบคลาวด์ เพื่อให้แอปมือถือเรียกใช้งานได้ทันทีโดยไม่ต้องเขียนเซิร์ฟเวอร์เอง  
ค. ทำหน้าที่เขียนสไตล์ชีทสำหรับจัดวาง Flexbox ในแบบออโต้  
ง. บีบอัดขนาดโค้ดของแอปพลิเคชันให้เหลือเพียงหน้าจอเดียวชั่วคราว  

### ข้อ 2: ฐานข้อมูลหลักที่รันอยู่เบื้องหลังและคอยจัดสรรแถวตารางข้อมูลในคลาวด์ของ Supabase คือฐานข้อมูลประเภทใด?
ก. MongoDB (NoSQL)  
ข. MySQL (Relational)  
ค. PostgreSQL (Relational)  
ง. Firebase Realtime Database (JSON)  

### ข้อ 3: การดึงคีย์ Anon Key จากระบบของ Supabase มาใช้เชื่อมต่อนั้น Anon Key มีลักษณะความปลอดภัยอย่างไร?
ก. เป็นคีย์ลับสูงสุดระดับ Root ห้ามใครพบเห็นเด็ดขาดเนื่องจากใช้ลบระบบทั้งหมดได้  
ข. เป็นคีย์สาธารณะ (Public API Key) สำหรับส่งไปพร้อม API เพื่อยืนยันสิทธิ์เข้าใช้ขั้นต้นของลูกค้าร่วมกับการตั้งเงื่อนไขสิทธิ์ RLS (Row Level Security)  
ค. เป็นรหัสผ่านสำหรับเข้าสู่โหมดนักพัฒนาบนแอนดรอยด์  
ง. เป็นรหัสยืนยันตนสำหรับการอัปเดตแอปขึ้นไปบน Google Play Store  

### ข้อ 4: ใน Expo SDK 54 เมื่อต้องการกำหนดค่ารหัสโครงการและคีย์ Anon Key ไว้ในไฟล์ `.env` เพื่อไม่ให้โค้ดรั่วไหล ข้อใดคือรูปแบบการตั้งชื่อตัวแปรสภาพแวดล้อม (Environment Variables) ที่ถูกต้อง?
ก. `SUPABASE_URL=...`  
ข. `SECRET_SUPABASE_URL=...`  
ค. `EXPO_PUBLIC_SUPABASE_URL=...`  
ง. `REACT_APP_SUPABASE_URL=...`  

### ข้อ 5: เหตุใดเราจึงจำเป็นต้องติดตั้งและสั่งนำเข้า `import 'react-native-url-polyfill/auto';` ไว้ที่ส่วนบนสุดของไฟล์กำหนดค่า Supabase Client?
ก. เพื่อบีบอัดขนาดไฟล์ภาพของโลโก้ให้เล็กลง  
ข. เพื่อชดเชยและแก้ไขปัญหาที่ React Native รันไทม์ไม่มีคลาสคำสั่งจัดการ URL (URL API Class) ในระดับเนทีฟ ซึ่งทำให้ไลบรารีของ Supabase ส่งรีเควสไม่ได้  
ค. เพื่อสั่งเปิดสัญญาณบลูทูธของมือถือเชื่อมเข้าคลาวด์  
ง. เพื่อสลับภาษาในแบบฟอร์มให้กลายเป็นภาษาไทยอัตโนมัติ  

### ข้อ 6: คำสั่งสืบค้นข้อความ SQL-like ใดใน Supabase JS Client ที่ใช้สำหรับระบุชื่อตารางเป้าหมายที่ต้องการเปิดติดต่อข้อมูล?
ก. `.select()`  
ข. `.from()`  
ค. `.insert()`  
ง. `.eq()`  

### ข้อ 7: หากต้องการดึงข้อมูลทุกคอลัมน์จากตารางชื่อ "tasks" โดยเรียงลำดับแถวเอาโพสต์ที่มีอายุการบันทึกล่าสุดขึ้นก่อน ข้อใดเขียนคำสั่งได้ถูกต้องและมีประสิทธิภาพที่สุด?
ก. `await supabase.from('tasks').select('*').order('created_at', { ascending: false })`  
ข. `await supabase.from('tasks').select('all').eq('id', 1)`  
ค. `await supabase.from('tasks').insert({ order: 'created_at' })`  
ง. `await supabase.from('tasks').delete().order('created_at')`  

### ข้อ 8: การเพิ่มข้อมูลแถวใหม่เข้าไปในตารางด้วยเมธอด `.insert()` ของ Supabase ข้อมูลดังกล่าวต้องมีโครงสร้างของอาร์กิวเมนต์เป็นประเภทใด?
ก. ข้อความสตริงระบุคีย์ดิบ ๆ  
ข. อาร์เรย์ของออบเจกต์ (Array of Objects) เช่น `[{ key: value }]`  
ค. ตัวแปรประเภทตัวเลขทศนิยมเดี่ยว  
ง. ฟังก์ชันประเภท Callback  

### ข้อ 9: พฤติกรรมการทำงานของระบบความปลอดภัย RLS (Row Level Security) ของ Supabase ข้อใดอธิบายได้ถูกต้อง?
ก. ทำหน้าที่บังคับล็อกหน้าจอโทรศัพท์มือถือหากกรอกรหัสผ่านผิดเกิน 3 ครั้ง  
ข. เป็นกฎความปลอดภัยบนระบบคลาวด์เพื่ออนุญาตหรือปิดกั้นให้เฉพาะผู้ใช้ที่มีสิทธิ์เข้าถึงหรือแก้ไขเฉพาะแถวข้อมูลที่ตนเองเป็นเจ้าของเท่านั้น  
ค. ใช้จำกัดความสูงความกว้างของฟอร์ม TextInput  
ง. เป็นฟังก์ชันลบข้อมูลขยะออกจากหน่วยความจำ RAM ของมือถือโดยออโต้  

### ข้อ 10: หากต้องการแก้ไขสถานะการทำงานในคอลัมน์ `is_completed` ให้กลายเป็นค่า `true` เฉพาะแถวงานที่มีรหัสไอดี ID เท่ากับ 5 ข้อใดเขียนคำสั่งระบุเงื่อนไขได้ถูกต้องและเจาะจงเป้าหมายที่สุด?
ก. `await supabase.from('tasks').update({ is_completed: true }).eq('id', 5)`  
ข. `await supabase.from('tasks').insert({ is_completed: true }).eq('id', 5)`  
ค. `await supabase.from('tasks').delete().eq('id', 5)`  
ง. `await supabase.from('tasks').select('is_completed').eq('id', 5)`

## เฉลยคำตอบและคำอธิบายโดยละเอียด

### เฉลย ข้อ 1: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ไม่สามารถยุ่งเกี่ยวหรือยกระดับฮาร์ดแวร์จริงอย่าง CPU/RAM ของเครื่องปลายทางได้
  * **ข. ถูก:** BaaS (เช่น Supabase) ถูกดีไซน์มาเพื่อประหยัดแรงผู้พัฒนาแอปโดยการจัดการหลังบ้าน เช่น Database และ Auth ให้สำเร็จรูปบนระบบ Cloud
  * **ค. และ ง. ผิด:** ไม่เกี่ยวข้องกับ Flexbox และไม่มีฟังก์ชันช่วยย่อบีบอัดหน้าจอแอป

### เฉลย ข้อ 2: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** Supabase เลือกใช้งานฐานข้อมูลประเภท Relational SQL และใช้ PostgreSQL เป็นแกนหลักขับเคลื่อนเท่านั้น
  * **ค. ถูก:** โครงสร้างดาต้าเบสเบื้องหลังของ Supabase คือ PostgreSQL 100% ทำให้สนับสนุนฟีเจอร์อย่าง SQL, Constraints และความสัมพันธ์ข้อมูลสมบูรณ์

### เฉลย ข้อ 3: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** คีย์สูงสุดคือ Service Role Key ซึ่งห้ามนำมาแปะลงในแอปมือถือเด็ดขาดเพราะข้ามระบบความปลอดภัยได้หมด
  * **ข. ถูก:** Anon Key เป็นคีย์ระดับสาธารณะที่อนุญาตให้ลูกค้ายิง API ค้นหาตารางได้ โดยคีย์นี้จะทำงานร่วมกับระบบ RLS (Row Level Security) เพื่อจำกัดความปลอดภัยของตารางข้อมูล
  * **ค. และ ง. ผิด:** ไม่ใช่รหัสสำหรับ Android หรือสิทธิ์ในการส่งแอปขึ้น Play Store

### เฉลย ข้อ 4: ตอบ ค.
* **เหตุผลประกอบ:**
  * **ก. ข. และ ง. ผิด:** ตัวแปรอื่นจะไม่ได้รับการดึงเข้าระบบจาวาสคริปต์อัตโนมัติในฝั่ง Expo Client
  * **ค. ถูก:** ใน Expo SDK 54 ตัวแปรในไฟล์ `.env` ที่แอปจะอ่านเข้าไปรันใช้งานได้ในโค้ดดิ้งฝั่งไคลเอนต์ จำเป็นต้องมีคำนำหน้า (Prefix) เป็น **`EXPO_PUBLIC_`** เสมอ

### เฉลย ข้อ 5: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ไม่เกี่ยวข้องกับการย่อขนาดไฟล์รูปภาพใด ๆ
  * **ข. ถูก:** รันไทม์ของ React Native ต่างจากเว็บบราวเซอร์ตรงที่ไม่มีคลาส `URL` หรือ `URLSearchParams` ดั้งเดิม การนำเข้าตัว polyfill จะชดเชยช่องว่างนี้ เพื่อให้ Supabase JS Client ส่ง URL รีเควสเชื่อมต่อได้สมบูรณ์โดยแอปไม่ระเบิด
  * **ค. และ ง. ผิด:** ไม่สามารถควบคุมการเปิดบลูทูธเครื่อง หรือแก้ไขฟอร์มภาษาไทยได้โดยออโต้

### เฉลย ข้อ 6: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** `.select()` คุมการค้นหาคอลัมน์  
  * **ข. ถูก:** `.from('ชื่อตาราง')` บ่งชี้เฉพาะถึงแหล่งข้อมูลตารางเป้าหมายที่ลงทะเบียนไว้ใน Supabase
  * **ค. ผิด:** `.insert()` คุมการสร้างแถวข้อมูล  
  * **ง. ผิด:** `.eq()` ทำหน้าที่กรองค่าเงื่อนไข

### เฉลย ข้อ 7: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** โครงสร้างที่สมบูรณ์คือระบุตารางด้วย `.from()`, เลือกคอลัมน์ทั้งหมดด้วย `.select('*')` และสั่งเรียงข้อมูลตามเวลาด้วย `.order('created_at', { ascending: false })`
  * **ข. ค. และ ง. ผิด:** รูปแบบอื่นระบุคำสั่งไม่ถูกประเภท หรือมีพารามิเตอร์ผิดจุดประสงค์

### เฉลย ข้อ 8: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ค. และ ง. ผิด:** เมธอดของ insert ปฏิเสธค่าข้อความเพลน ๆ หรือค่าทศนิยมเดี่ยว และไม่ใช่ฟังก์ชัน
  * **ข. ถูก:** พารามิเตอร์ของ `.insert()` จะต้องรับค่าเป็น JSON Array เสมอ แม้ว่าเราจะยิงเพื่อสร้างเพียงแถวเดียว ก็ยังต้องส่งเป็น Array ที่มีออบเจกต์ชิ้นเดียวนอนอยู่ด้านใน

### เฉลย ข้อ 9: ตอบ ข.
* **เหตุผลประกอบ:**
  * **ก. ผิด:** ไม่ใช่ความปลอดภัยในการจำกัดรหัสผ่านบนฮาร์ดแวร์หน้าจอมือถือ
  * **ข. ถูก:** Row Level Security (RLS) ของ PostgreSQL/Supabase ทำหน้าที่เช็คว่าแถวนี้เขียนค่านำเข้าโดยผู้ใช้คนไหน และใช้บล็อกไม่ให้คนนอกเขียนทับหรืออ่านข้อมูลข้ามบัญชีของคนอื่น
  * **ค. และ ง. ผิด:** สไตล์ TextInput คุมด้วย CSS และ RAM คุมด้วยรันไทม์ระบบไม่ใช่ RLS

### เฉลย ข้อ 10: ตอบ ก.
* **เหตุผลประกอบ:**
  * **ก. ถูก:** เมธอด `.update()` จะยอมรับฟิลด์ใหม่ และต้องสั่งกรองระบุเฉพาะตัวด้วยเมธอด `.eq('id', 5)` เพื่อให้แก้เฉพาะไอดี 5
  * **ข. ผิด:** insert ใช้สร้างแถวใหม่ ไม่ใช่แก้ไขค่า  
  * **ค. ผิด:** delete เป็นคำสั่งถอนลบข้อมูลไม่ใช่การเปลี่ยนสถานะ  
  * **ง. ผิด:** select เป็นเพียงการเรียกรับค่าเดิมมาแสดง
