# บทที่ 10: การใช้งานคลาวด์ดาต้าเบสด้วย Supabase (Overview)

ในบทเรียนนี้ ผู้เรียนจะได้ศึกษาแนวคิดการทำระบบหลังบ้านสำเร็จรูป (Backend-as-a-Service หรือ BaaS) โดยเน้นใช้งานคลาวด์ดาต้าเบสด้วย **Supabase** ซึ่งทำงานร่วมกับฐานข้อมูลเชิงสัมพันธ์ระดับโลกอย่าง **PostgreSQL** ผู้เรียนจะได้เรียนรู้กระบวนการเชื่อมต่อไคลเอนต์ในระบบสภาพแวดล้อมความปลอดภัยด้วยไฟล์ `.env` ที่มีคำนำหน้า `EXPO_PUBLIC_` ในระบบของ Expo SDK 54, โครงสร้างโค้ดการทำ CRUD (Create, Read, Update, Delete) ผ่าน Supabase JS Client ร่วมกับ `react-native-url-polyfill` และกลไกการรักษาความปลอดภัยของตารางข้อมูลด้วยกฎ Row Level Security (RLS)

## 📂 สารบัญเอกสารประจำบทเรียน (Chapter Documents)
คุณสามารถเลือกอ่านและเข้าถึงไฟล์เอกสารต่าง ๆ ในโฟลเดอร์บทเรียนนี้ได้โดยตรง:

1. 📋 **[แผนการสอนประจำบท (01-lesson-plan.md)](./01-lesson-plan.md)**  
   หัวข้อการสอน วัตถุประสงค์เชิงพฤติกรรม สื่อประกอบการสอน และหัวข้อการวัดประเมินทักษะคลาวด์ดาต้าเบส
2. 📖 **[เนื้อหาหลักการเรียนรู้ (02-content.md)](./02-content.md)**  
   เนื้อหาหลักทางทฤษฎีอธิบายสถาปัตยกรรม BaaS, การตั้งค่า Supabase Client ร่วมกับ Polyfill, การเขียนคำสั่งสืบค้นและดึงข้อมูล (SELECT), การเพิ่มข้อมูล (INSERT), การแก้ไขค่าข้อมูล (UPDATE), การลบแถวตารางข้อมูล (DELETE) และแนวคิดความปลอดภัย Row Level Security (RLS)
3. 📝 **[แบบทดสอบท้ายบทเรียน (03-quiz.md)](./03-quiz.md)**  
   แบบทดสอบปรนัยทบทวนความเข้าใจ 10 ข้อ ภาษาไทยเจาะลึกความรู้เรื่องระบบหลังบ้าน ความปลอดภัย RLS คำสั่ง SQL-like ของ Supabase API และการตั้งค่าตัวแปรสภาพแวดล้อม พร้อมเฉลยและคำอธิบายอย่างละเอียดทุกตัวเลือก
4. 💻 **[คู่มือปฏิบัติการขั้นพื้นฐาน (04-lab.md)](./04-lab.md)**  
   โจทย์ปฏิบัติการสร้างแอปพลิเคชัน **"สมุดจดสินค้าที่ต้องการและรายการโปรดออนไลน์ (Supabase Wishlist Manager)"** เพื่อฝึกฝนสร้างตารางข้อมูล wishlist บนระบบหลังบ้านจริง ตั้งค่า API Keys และพัฒนา UI ซิงก์ข้อมูล CRUD บนหน้าจอของ React Native พร้อมโค้ดเฉลยฉบับสมบูรณ์ (Complete Solution)

## 🛠️ สรุปโปรเจคปฏิบัติการท้ายบทเรียน (Lab Project Summary)
ในปฏิบัติการของบทนี้ ผู้เรียนจะได้ฝึกฝนการเขียนระบบติดต่อคลาวด์ดาต้าเบสด้วยฟีเจอร์หลักดังนี้:
* **Supabase Client Initializer:** การติดตั้ง `@supabase/supabase-js` และแก้ไขข้อจำกัดการทำ URL Request ใน React Native ด้วย `react-native-url-polyfill`
* **Realtime Select / Fetch:** การดึงรายการ Wishlist ทั้งหมดเรียงตามวันเวลาบันทึกใหม่ล่าสุดมาเรนเดอร์ในคอมโพเนนต์ `FlatList`
* **Insert Operation:** แบบฟอร์มเพิ่มรายการใหม่ที่ตรวจสอบความถูกต้องก่อนกดบันทึกขึ้นดาต้าเบส และเสียบแถวใหม่ต่อท้ายสเตตได้ทันที
* **Toggle & Delete Actions:** การผูกปุ่มไอคอนหัวใจสลับสถานะความชอบสินค้า (`is_favorite`) และไอคอนถังขยะยิงคำสั่งลบข้อมูลแถวตาม ID ควบคู่กับการคัดกรอง `.eq()`
