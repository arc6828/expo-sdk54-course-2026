# บทที่ 10: การใช้งานคลาวด์ดาต้าเบสด้วย Supabase (Lesson Plan)

## หัวข้อเนื้อหาประจำบท
- แนวคิดระบบหลังบ้านสำเร็จรูป (Backend-as-a-Service - BaaS) และการเชื่อมต่อฐานข้อมูลแบบ PostgreSQL ของ Supabase
- ขั้นตอนการลงทะเบียน สร้างโปรเจกต์ และดึงค่ากำหนดโครงข่าย (Supabase URL, Anonymous Key)
- การติดตั้งไลบรารี `@supabase/supabase-js` สำหรับเชื่อมโยงระบบใน Expo SDK 54
- การสร้างและประกาศใช้โมดูล Helper สำหรับเรียกใช้งาน Supabase Client ในแอปพลิเคชัน
- คำสั่งดึงข้อมูลด้วยเมธอด `.select()` ร่วมกับการใช้ตัวกรอง (`.eq()`, `.order()`, และ `.ilike()`)
- คำสั่งการจัดการข้อมูลข้ามระบบ: การเพิ่มข้อมูล (`.insert()`), การแก้ไขข้อมูล (`.update()`) และการลบข้อมูล (`.delete()`)
- ระบบควบคุมความปลอดภัยและการดักเช็คค่าตอบกลับความผิดพลาด (Response Error handling) ของ API

## วัตถุประสงค์เชิงพฤติกรรม
1. ผู้เรียนสามารถอธิบายสถาปัตยกรรมจัดสรรหลังบ้าน BaaS และความแตกต่างเมื่อเทียบกับฐานข้อมูลธรรมดาได้ถูกต้อง
2. ผู้เรียนสามารถติดตั้ง ตั้งค่า และลงทะเบียน Supabase Client ในโปรเจกต์ Expo ได้สำเร็จ
3. ผู้เรียนสามารถเขียนคำสั่งภาษา SQL จำลองผ่าน JS Client ดึงข้อมูลขึ้นมาแสดงผลเป็นรายการตารางงานได้สำเร็จ
4. ผู้เรียนสามารถเขียนคำสั่งเพิ่ม ลบ และอัปเดตสถานะงานของตารางบน Supabase Cloud ได้ถูกต้อง
5. ผู้เรียนสามารถวิเคราะห์สถานะข้อผิดพลาดของข้อมูล (Error objects) และเขียนส่งสัญญาณเตือนอินเตอร์เฟซได้ถูกต้อง

## วิธีสอนและกิจกรรมการเรียนการสอนประจำบท
1. **การบรรยายสถาปัตยกรรม BaaS และคลาวด์เบสหลังบ้าน (BaaS & Cloud Database Lecture):** อธิบายหลักการของระบบหลังบ้านแบบ Serverless และเปรียบเทียบระหว่างสถาปัตยกรรมแบบทำเซิร์ฟเวอร์เอง (Custom Backends) กับการประยุกต์ใช้คลาวด์สำเร็จรูปอย่าง Supabase
2. **การทำ Live Coding Demonstration หัวข้อ Real-time Shopping Task List:** ผู้สอนสาธิตการทำแอปจดลิสต์รายการซื้อของซิงก์ตรงหลังบ้าน Supabase ที่ผู้ใช้สามารถจิ้มขีดฆ่าสินค้าที่ซื้อแล้ว (Update) หรือพิมพ์เพิ่มรายการใหม่ได้ทันที
3. **กิจกรรมแฮกเกอร์เขียนคำสั่งเงื่อนไข (Conditional Queries Challenge):** ให้ผู้เรียนทดลองเขียนตัวกรองดักรับข้อมูลให้แสดงผลเฉพาะสินค้าที่ยังไม่ได้ซื้อ และมีปุ่มกดสับเปลี่ยนลำดับแสดงราคาแพงสุดขึ้นก่อน
4. **กิจกรรมป้องกันความปลอดภัยฟอร์มข้อมูล (Database Constraint Protection):** ให้กลุ่มผู้เรียนช่วยกันพัฒนาเงื่อนไขสกัดกั้นการบันทึกข้อมูล หากช่องรับตัวหนังสือไม่มีค่า เพื่อป้องกันการยิงแถวว่างเปล่าเข้าไปรบกวนฐานข้อมูล

## สื่อการเรียนการสอน
1. เอกสารสไลด์เรื่อง "Supabase Cloud Database Integration in React Native Applications"
2. หน้าแผงควบคุมหลักของโครงการ Supabase (Supabase Dashboard) สำหรับตรวจเช็คแถวตารางข้อมูลแบบเรียลไทม์
3. ตัวอย่างการเซ็ตค่าสิทธิ์ RLS (Row Level Security) ขั้นต้น
4. ชุดคำสั่งตัวอย่างการเขียน Wrapper Helper สำหรับดึงข้อมูล Supabase Client ในแอปพลิเคชัน

## การวัดผลและการประเมินผล

### 1. การวัดผล
1.1 ประเมินระดับความรู้เชิงทฤษฎีจากการทำคะแนนแบบทดสอบปรนัยท้ายบทเรียน 10 ข้อ
1.2 ตรวจสอบความสมบูรณ์และถูกต้องในการเรียกใช้ Supabase client และคำสั่ง SQL-like methods (select, insert, etc.)
1.3 ตรวจสอบขั้นตอนการจัดการข้อยกเว้นและตัวบ่งบอกความคืบหน้า (Loading) ในโค้ดใบงานปฏิบัติการ

### 2. การประเมินผล
2.1 ผู้เรียนทำคะแนนแบบทดสอบปรนัยท้ายบทเรียนได้ไม่น้อยกว่าร้อยละ 80 (อย่างน้อย 8 จาก 10 ข้อ)
2.2 ผู้เรียนสามารถสร้างโมดูลสมุดรายชื่อสินค้าที่ซิงก์ตรงขึ้นระบบคลาวด์ Supabase โดยแสดงข้อมูลถูกประเภทและรันได้ไม่มีข้อผิดพลาด (ผ่าน/ไม่ผ่าน)
2.3 สังเกตพฤติกรรมการรักษาความปลอดภัยของ Anon Key (ไม่เปิดเผยคีย์สาธารณะ) และความละเอียดรอบคอบในการจัดชนิดข้อมูลอินเทอร์เฟซ
